# Feature Specification: 홈 작품 카드 개선 + 마이페이지 테마 토글 수정

**Feature Branch**: `worktree-038-home-cards-theme-toggle` (워크트리 격리, develop `f497951` 기준)

**Created**: 2026-06-24

**Status**: Draft

**Input**: User description: "홈 대시보드 작품 카드 개선 + 마이페이지 테마 토글 버그 수정"

## 개요

본 명세는 서로 독립적인 두 사용성 개선을 묶는다.

1. **홈 작품 카드 개선** — 메인 홈(`/`)에서 "이어서 쓰기" 강조 카드를 제외한 나머지 작품 카드의 개수를 제한하고, 각 카드가 보여주는 정보를 풍부하게 한다.
2. **마이페이지 테마 토글 수정** — 마이페이지 설정의 라이트/다크 모드 선택이 화면에 반영되지 않는 회귀 버그를 고친다.

두 작업은 다른 화면·다른 데이터 경로를 다루므로 독립적으로 개발·테스트·배포할 수 있다.

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 홈 작품 카드 정보 강화 및 개수 제한 (Priority: P1)

작가가 메인 홈에 들어오면, "이어서 쓰기"로 강조된 가장 최근 작품 1개 아래에 **최근 작업한 작품 최대 2개**가 카드로 보인다. 각 카드는 제목, 그 작품이 속한 시리즈명, 마지막으로 작성한 내용 일부, 최종 수정일, 목표 분량 게이지를 한눈에 보여준다. 카드에 마우스를 올리면 작품 생성일과 그 작품에 들인 총 집필 시간이 추가로 나타난다. 상위 2개 밖의 작품이 있으면 카드 영역 아래 "더 보기" 진입점을 통해 작품 보관함으로 이동해 전체를 열람한다.

**Why this priority**: 현재 홈은 작품이 늘어날수록 카드가 무한정 쌓여(append) 홈이 길어지고, 각 카드의 정보도 빈약하다. 개수 제한 + 정보 강화는 "재진입 허브"로서 홈의 핵심 가치(지금 무엇을 이어서 쓸지 빠르게 판단)를 직접 끌어올린다.

**Independent Test**: 작품 4개 이상을 가진 계정으로 홈에 진입해, "이어서 쓰기" 외 카드가 정확히 2개만 보이고 각 카드에 제목·시리즈·마지막 작성 내용·최종 수정일이 표시되며, 호버 시 생성일·총 집필 시간이 나타나는지로 검증 가능하다.

**Acceptance Scenarios**:

1. **Given** 활성 작품이 4개 있는 작가가, **When** 홈에 진입하면, **Then** "이어서 쓰기" 카드 1개 + 그 아래 최근 작업한 작품 카드가 최대 2개만 보이고, 카드 영역 아래 "더 보기" 진입점이 표시된다.
2. **Given** 홈의 작품 카드 하나가, **When** 화면에 표시되면, **Then** 제목, 시리즈명, 마지막 작성 내용 일부, 최종 수정일, 목표 분량 게이지가 보인다.
3. **Given** 미분류(시리즈에 속하지 않은) 작품 카드가, **When** 표시되면, **Then** 시리즈 자리에 "미분류"로 표시된다.
4. **Given** 홈의 작품 카드에, **When** 마우스를 올리면, **Then** 작품 생성일과 그 작품의 총 집필 시간이 추가로 나타난다.
5. **Given** 활성 작품이 "이어서 쓰기" 대상 1개뿐인 작가가, **When** 홈에 진입하면, **Then** 아래 작품 카드 영역은 비어 있고(0개) 오류 없이 표시된다(이때 "더 보기" 진입점도 표시하지 않는다).
6. **Given** 홈의 "더 보기" 진입점을, **When** 작가가 클릭하면, **Then** 작품 보관함(전체 작품 목록)으로 이동한다.

### User Story 2 - 마이페이지 테마 토글이 화면에 즉시 반영 (Priority: P2)

작가가 마이페이지 설정에서 라이트 모드 또는 다크 모드를 선택하면, 선택 즉시 화면 전체의 테마가 그에 맞게 바뀌고, 이후 재진입·새로고침에도 그 선택이 유지된다.

**Why this priority**: 이전(통합 `/settings`)에는 동작하던 기능이 마이페이지 재구성 과정에서 회귀했다. 기능 회귀이므로 우선순위가 높으나, US1과 독립적이고 영향 화면이 설정 한 곳이라 P2로 둔다.

**Independent Test**: 마이페이지 설정에서 다크 모드를 선택했을 때 화면이 즉시 다크로 바뀌고, 라이트로 되돌리면 즉시 라이트로 바뀌며, 새로고침 후에도 마지막 선택이 유지되는지로 검증 가능하다.

**Acceptance Scenarios**:

1. **Given** 라이트 모드 상태의 작가가, **When** 마이페이지 설정에서 다크 모드를 선택하면, **Then** 화면 테마가 즉시 다크로 전환된다(추가 새로고침 불필요).
2. **Given** 다크 모드 상태의 작가가, **When** 라이트 모드를 선택하면, **Then** 화면 테마가 즉시 라이트로 전환된다.
3. **Given** 작가가 테마를 선택한 뒤, **When** 페이지를 새로고침하거나 다른 기기에서 재로그인하면, **Then** 마지막으로 선택한 테마가 그대로 적용된다.

---

### Edge Cases

- **작품이 정확히 3개일 때**: "이어서 쓰기" 1개 + 카드 2개로 모든 작품이 홈에 표시된다(잘림 없음).
- **작품이 0개일 때(신규 가입)**: "이어서 쓰기" 영역과 작품 카드 영역 모두 빈 상태 안내로 처리되며 오류가 없어야 한다(기존 동작 보존).
- **시리즈명이 매우 길 때**: 카드 레이아웃이 깨지지 않도록 시리즈명이 적절히 잘리거나 줄바꿈 처리된다.
- **마지막 작성 내용이 비어 있을 때(본문 없는 작품)**: 마지막 작성 내용 자리가 비거나 적절한 대체 문구로 표시되어 카드가 깨지지 않는다.
- **총 집필 시간이 0일 때(한 번도 집필 세션이 없는 작품)**: 호버 시 "0분" 등 일관된 형식으로 표시된다.
- **테마가 '시스템 설정 따름' 같은 제3의 값을 지원하는 경우**: 본 수정 범위에 그 값이 포함되는지 여부를 명확히 한다(아래 Assumptions 참조).

## Requirements *(mandatory)*

### Functional Requirements

#### 홈 작품 카드 (US1)

- **FR-001**: 홈은 "이어서 쓰기" 강조 카드(가장 최근 수정한 작품)를 제외한 나머지 작품 카드를 **최대 2개**까지만 표시해야 한다.
- **FR-002**: 표시되는 2개 카드는 **최종 수정일(최근 작업 순)** 기준 상위 2개여야 한다(기존 정렬 기준 유지 — 최종 수정일 내림차순, 동률 시 작품 id 큰 쪽).
- **FR-003**: 각 작품 카드는 다음 정보를 표시해야 한다 — 작품 **제목**, **시리즈명**, **마지막 작성 내용 일부**(기존 표시 방식 유지: 마지막 작성 평문을 1~2줄로 자름), **최종 수정일**, **목표 분량 게이지**(기존 게이지 유지).
- **FR-004**: 시리즈에 속하지 않은(미분류) 작품은 시리즈 자리에 "미분류"로 표시해야 한다.
- **FR-005**: 작품 카드에 마우스를 올리면(호버) **작품 생성일**과 **총 집필 시간**이 추가로 표시되어야 한다.
- **FR-006**: 마지막 작성 내용·총 집필 시간 등 값이 없거나 0인 경우에도 카드 레이아웃이 깨지지 않고 일관된 대체 표현으로 표시되어야 한다.
- **FR-007**: "이어서 쓰기" 카드의 동작(가장 최근 작품으로 이어쓰기 진입)은 기존과 동일하게 보존되어야 한다.
- **FR-011**: 상위 2개 밖의 작품이 1개 이상 존재하면, 작품 카드 영역 아래에 "더 보기"(전체 작품 보기) 진입점을 표시하고, 클릭 시 작품 보관함(`/library`, 전체 작품 목록)으로 이동해야 한다. 상위 2개로 모든 작품이 표시되는 경우(작품 ≤ 3개)에는 "더 보기"를 표시하지 않는다.

#### 테마 토글 (US2)

- **FR-008**: 마이페이지 설정에서 테마(라이트/다크)를 선택하면, 추가 새로고침 없이 화면 테마가 즉시 그 선택대로 전환되어야 한다.
- **FR-009**: 선택한 테마는 서버에 저장되어, 새로고침·재로그인·다른 기기에서도 유지되어야 한다(기존 설정 영속 동작 보존).
- **FR-010**: 테마 선택 UI는 현재 적용 중인 테마를 정확히 반영(선택 표시)해야 한다.

### Key Entities *(데이터 관련)*

- **작품 카드(ProjectCard)**: 홈에 표시되는 작품 요약. 속성 — 제목, 소속 시리즈명, 마지막 작성 내용 일부, 최종 수정일, 작품 생성일, 총 집필 시간. (현재 응답에는 시리즈 **식별자**만 있고 **시리즈명**은 없어, 시리즈명 표시를 위한 보강이 필요하다.)
- **시리즈(Category)**: 작품이 속하는 분류. 카드에는 그 **이름**이 표시된다. 미분류(소속 없음) 상태가 존재한다.
- **사용자 설정(테마)**: 라이트/다크 선택값. 서버에 영속되며 화면 테마 적용의 원천이다.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 작품 수와 무관하게 홈의 "이어서 쓰기" 아래 작품 카드는 항상 2개 이하로 표시된다(작품 10개 계정에서도 2개).
- **SC-002**: 홈의 각 작품 카드에서 제목·시리즈·마지막 작성 내용·최종 수정일 4개 정보가 한 번의 시선으로 모두 확인된다(클릭·스크롤 불필요).
- **SC-003**: 작품 카드 호버 시 1초 이내에 생성일·총 집필 시간이 나타난다.
- **SC-004**: 마이페이지에서 테마를 변경하면 1초 이내에 화면 전체 테마가 전환되며, 새로고침 후에도 100% 유지된다.
- **SC-005**: 두 개선 후 기존 홈 진입·이어쓰기·설정 영속 동작에 회귀가 없다(기존 자동화 테스트 GREEN 유지).

## Assumptions

- **나머지 작품 접근 (Q1 확정)**: 홈은 최근 작업 작품 2개만 표시하고, 상위 2개 밖 작품은 카드 영역 아래 **"더 보기" 진입점**을 통해 기존 **작품 보관함(`/library`)**으로 이동해 전체 열람한다. 홈은 "최근 작업한 소수로 빠르게 재진입"하는 허브로 한정한다.
- **"마지막 작성 내용"의 단위 (Q2 확정)**: 현재 응답은 작품 본문의 마지막 작성 텍스트(`lastSentenceSource`, 평문)를 제공하며, 홈 카드는 이미 이를 1~2줄로 잘라 표시 중이다. 본 작업은 이 **기존 표시 방식을 그대로 유지**한다(신규 추가 아님).
- **목표 분량 게이지 (확정)**: 현재 홈 카드의 목표 분량 게이지는 **유지**하며, 그 위에 시리즈·최종 수정일·호버(생성일·총 집필 시간) 정보를 더한다.
- **시리즈명 보강 방식**: 시리즈명은 작품 카드 응답에 시리즈 이름을 추가(기존 필드 깨지 않는 additive 보강)하는 방식으로 제공한다고 가정한다. 기존 데이터·다른 응답에 영향이 없어야 한다.
- **호버 정보의 모바일 동작**: 호버(생성일·총 집필 시간)는 마우스가 있는 데스크톱 기준 인터랙션이다. 터치 환경에서의 동등 표현은 본 v1 범위 밖으로 가정한다(기존 `/library` 말풍선 패턴과 동일 수준).
- **테마 값 범위**: 본 수정은 현재 설정 UI가 제공하는 테마 선택지(라이트/다크)에 한정한다. '시스템 설정 따름' 등 추가 모드가 현재 UI에 없다면 신설하지 않는다(회귀 수정에 집중).
- **정렬 기준 불변**: 카드 정렬 기준(최종 수정일 내림차순, 동률 시 작품 id)은 기존을 유지한다. 본 작업은 표시 개수·정보·테마 적용만 다룬다.
- **데이터 가용성**: 제목·마지막 작성 내용·최종 수정일·생성일·총 집필 시간은 현재 작품 카드 응답에 이미 존재한다(별도 신규 조회 경로 불필요). 시리즈명만 보강 대상이다.

## Clarifications (해소됨)

- **Q1 (스코프) → 확정**: 홈 카드는 최대 2개로 제한하고, 상위 2개 밖 작품은 카드 영역 아래 **"더 보기" 진입점 → 작품 보관함(`/library`)**으로 접근한다. (FR-011)
- **Q2 (표시 내용) → 확정**: "마지막 작성 내용"은 **기존 표시 방식(마지막 작성 평문 1~2줄)을 그대로 유지**한다(신규 작업 아님). (FR-003)
- **목표 분량 게이지 → 확정**: 기존 게이지를 **유지**한다. (FR-003)
