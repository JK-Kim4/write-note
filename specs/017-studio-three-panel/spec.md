# Feature Specification: 집필실 3단 (Studio 3-panel)

**Feature Branch**: `feat/studio-three-panel`

**Created**: 2026-06-10

**Status**: Draft

**Input**: 확정 설계 `docs/superpowers/specs/2026-06-10-studio-three-panel-design.md` + 핸드오프 `docs/handoff/2026-06-10-scope-b-studio-three-panel-handoff.md`

## 개요

집필 중인 작가가 **"한 작업실에 모든 맥락"**(PRODUCT.md 디자인 원칙 2)을 누리도록, 집필 surface(원고) 양옆에 **장면 아웃라인**과 **인물 노트**를 두어 흩어진 맥락을 글 쓰는 자리로 모은다. Scrivener의 복잡함이 아니라 iA Writer의 절제로 — 평소엔 물러나고(접기), 필요할 때 곁에.

집필실 `/projects/[id]/write`의 본문 영역을 3열로 재구성한다: **[아웃라인 ~240px | 원고 1fr | 우측 패널 ~320px]**. 좌·우 패널은 각각 접을 수 있고, 우측 패널은 인물 노트(상단) + 곁쪽지(하단) 세로 스택이다.

**비목표**: 글쓰기 효율 기능, 장면 단위 문서 분할(Scrivener 바인더), AI 보조, 음악/분위기(Ambience).

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 장면 아웃라인으로 원고를 조망하고 점프 (Priority: P1)

작가가 긴 원고를 쓰는 도중, 좌측 아웃라인 패널에서 본문에 단 제목(heading)들이 목차로 자동 정리된 것을 본다. 목차 항목을 클릭하면 원고의 해당 위치로 스크롤 점프하고, 스크롤하는 동안 현재 보고 있는 섹션이 목차에서 하이라이트된다. 별도 입력 없이 본문 제목만으로 목차가 생기고 사라진다.

**Why this priority**: 이 기능이 본 작업의 양보 불가 핵심이다 — "긴 원고에서 길을 잃지 않고 맥락을 조망"하는 것이 3단 레이아웃을 도입하는 가장 distinctive한 이유다. 백엔드 변경 없이 본문에서 순수 파생하므로 독립적으로 완결된다.

**Independent Test**: 본문에 H1/H2 제목을 여러 개 단 원고를 열어 좌측 아웃라인에 제목들이 순서·레벨대로 나타나는지, 항목 클릭 시 원고가 해당 위치로 점프하는지, 스크롤 시 현재 섹션이 하이라이트되는지, 제목을 추가/수정/삭제하면 목차가 갱신되는지로 검증. 제목이 0개면 안내 문구가 보인다.

**Acceptance Scenarios**:

1. **Given** 본문에 "1장", "2장" 등 H1/H2 제목이 순서대로 있는 원고, **When** 집필실을 연다, **Then** 좌측 아웃라인 패널에 제목들이 본문 등장 순서대로, 레벨(H1/H2)에 따른 들여쓰기로 표시된다.
2. **Given** 아웃라인에 항목이 표시된 상태, **When** "2장" 항목을 클릭한다, **Then** 원고가 "2장" 제목 위치로 스크롤 점프한다.
3. **Given** 원고를 위아래로 스크롤, **When** 특정 섹션이 화면에 들어온다, **Then** 아웃라인에서 해당 항목이 현재 섹션으로 하이라이트된다(색만이 아니라 굵기/배경 병행).
4. **Given** 집필 중인 원고, **When** 새 제목을 달거나 기존 제목 텍스트를 고친다, **Then** 잠시 후(디바운스) 아웃라인 목차가 그 변경을 반영한다.
5. **Given** 제목이 하나도 없는 원고, **When** 아웃라인 패널을 본다, **Then** "장면에 큰 제목을 달면 여기 목차가 생겨요." 안내가 보인다.

### User Story 2 - 곁에 둔 인물 노트 보기와 빠른 추가 (Priority: P2)

작가가 글을 쓰다 등장인물의 설정을 확인하고 싶을 때, 우측 인물 노트 패널에서 이 작품의 등장인물 목록(이름·한 줄 설명)을 본다. 각 인물의 상세 노트는 펼쳐 볼 수 있다. 글 쓰는 흐름을 끊지 않고 떠오른 새 인물을 이름(+한 줄)으로 즉시 추가한다. 상세 편집·삭제·재정렬이 필요하면 기존 등장인물 화면으로 이동한다.

**Why this priority**: "맥락이 새지 않음"을 인물 축에서 실현한다. 기존 등장인물 데이터를 집필 자리로 끌어와 재진입 비용을 줄인다. 기존 API 재사용으로 독립 완결되며, 아웃라인(P1)과 무관하게 단독 가치를 가진다.

**Independent Test**: 등장인물이 있는 작품의 집필실을 열어 인물 목록이 이름·한 줄 설명으로 표시되는지, 상세 노트를 펼침/접힘 토글할 수 있는지, 인라인 입력으로 이름을 넣어 추가하면 목록에 즉시 반영되는지, 등장인물 화면으로 가는 링크가 동작하는지로 검증. 인물이 0명이면 안내 + 빠른 추가 입력이 보인다.

**Acceptance Scenarios**:

1. **Given** 등장인물이 등록된 작품, **When** 집필실을 연다, **Then** 우측 인물 노트 패널에 인물들이 이름·한 줄 설명으로 표시된다.
2. **Given** 인물 항목, **When** 상세 펼침을 토글한다, **Then** 해당 인물의 상세 노트가 펼쳐지거나 접힌다.
3. **Given** 인물 노트 패널의 빠른 추가 입력, **When** 이름(과 선택적 한 줄)을 입력하고 확정한다, **Then** 새 인물이 생성되고 목록에 반영된다.
4. **Given** 인물이 한 명도 없는 작품, **When** 인물 노트 패널을 본다, **Then** "곁에 둘 인물을 추가." 안내와 빠른 추가 입력이 보인다.
5. **Given** 인물 노트 패널, **When** 상세 편집/삭제/재정렬 링크를 누른다, **Then** 기존 등장인물 화면(`/projects/[id]/characters`)으로 이동한다.

### User Story 3 - 패널 접기로 몰입과 조망을 전환 (Priority: P3)

작가가 깊게 몰입해 쓰고 싶을 땐 좌·우 패널을 접어 원고만 남기고, 맥락을 살펴야 할 땐 다시 펼친다. 우측 패널 안에서 인물 노트와 곁쪽지는 각각 따로 접을 수 있다. 곁쪽지(연결 메모)는 기존 동작 그대로 우측 스택 하단에 자리한다. 화면이 좁아지면 패널이 단계적으로 물러난다.

**Why this priority**: "평소엔 물러나고 필요할 때 곁에"라는 절제(PRODUCT.md 원칙 1·2)를 구현하는 마감 단계. P1·P2 패널이 존재해야 의미가 있으므로 마지막 우선순위. 기존 곁쪽지를 우측 스택으로 재배치하는 통합도 여기서 마무리한다.

**Independent Test**: 좌·우 패널 토글로 패널이 접히고/펼쳐지는지, 둘 다 접으면 원고만 남는지, 우측 패널 안에서 인물·곁쪽지 섹션을 개별 접을 수 있는지, 곁쪽지가 기존과 동일하게 동작하는지(연결 메모 목록·고정·해제), 좁은 폭에서 아웃라인이 먼저 접히고 그다음 우측 패널이 물러나는지로 검증.

**Acceptance Scenarios**:

1. **Given** 3단이 모두 펼쳐진 집필실, **When** 좌측 토글을 누른다, **Then** 아웃라인 패널이 접히고 원고 영역이 넓어진다.
2. **Given** 3단이 펼쳐진 상태, **When** 좌·우 토글을 모두 접는다, **Then** 원고만 남아 몰입 상태가 된다.
3. **Given** 우측 패널이 펼쳐진 상태, **When** 인물 노트 섹션 헤더의 접기를 누른다, **Then** 인물 섹션만 접히고 곁쪽지 섹션은 유지된다.
4. **Given** 우측 스택 하단의 곁쪽지, **When** 연결 메모를 고정/해제한다, **Then** 기존과 동일하게 동작한다(동작 불변).
5. **Given** 화면 폭을 좁힌다, **When** 폭이 임계 이하로 줄어든다, **Then** 먼저 아웃라인이 접히고, 더 좁아지면 우측 패널이 물러난다(모바일 폭에서는 오버레이/시트로 원고 우선).

---

### Edge Cases

- 제목 텍스트가 비었거나 공백만 있는 heading: 아웃라인에서 빈 항목을 어떻게 표기할지(빈 텍스트 placeholder 또는 무시) — 본문 등장 순서·위치는 유지.
- 같은 제목 텍스트가 여러 번 등장: 각각 별도 항목으로, 본문 위치로 구분해 점프가 정확해야 한다.
- 매우 긴 원고(수백 개 heading): 아웃라인 렌더·재파생이 입력 체감을 해치지 않아야 한다(디바운스).
- 빠른 추가 중 이름이 비어 있음: 추가가 거부되거나 비활성. 추가 실패(네트워크/서버 오류) 시 사용자에게 알리고 입력값은 보존.
- 한국어 IME 조합 중 본문 입력: 조합 중에는 부모 갱신을 억제하는 기존 가드를 깨지 않아야 하며, 아웃라인 재파생이 조합을 방해하지 않는다.
- `prefers-reduced-motion`이 켜진 환경: 아웃라인 점프는 부드러운 스크롤 대신 즉시 이동한다.

## Requirements *(mandatory)*

### Functional Requirements

**레이아웃 / 패널 (US3 포함)**

- **FR-001**: 집필실 본문 영역은 3열 레이아웃 [아웃라인(좌) | 원고(중) | 우측 패널]로 구성되어야 한다. 원고가 가변 폭, 아웃라인 ~240px, 우측 패널 ~320px을 기준으로 한다.
- **FR-002**: 좌측 아웃라인과 우측 패널은 각각 독립적으로 접기/펼치기가 가능해야 하며, 둘 다 접으면 원고만 남는다.
- **FR-003**: 우측 패널은 인물 노트(상단) + 곁쪽지(하단)의 세로 스택이며, 각 섹션은 개별 접기가 가능해야 한다.
- **FR-004**: 화면 폭이 좁아질 때 아웃라인이 먼저 접히고, 더 좁아지면 우측 패널이 물러나야 한다. 모바일 폭에서는 패널을 오버레이/시트로 띄워 원고를 우선한다. 유동(fluid) 타이포그래피는 쓰지 않는다.

**아웃라인 (US1)**

- **FR-005**: 시스템은 현재 원고 본문에서 제목(level 1·2) 노드를 본문 등장 순서대로 추출해 목차 항목(레벨, 텍스트, 본문 위치)으로 만들어야 한다.
- **FR-006**: 목차 항목 클릭 시 원고의 해당 위치로 스크롤 점프해야 한다.
- **FR-007**: 원고 스크롤 위치에 따라 현재 보고 있는 섹션을 아웃라인에서 하이라이트해야 하며, 하이라이트는 색만이 아니라 굵기/배경을 병행해 구분한다.
- **FR-008**: 본문(제목)이 변경되면 목차를 재파생해야 하며, 입력 체감을 해치지 않도록 디바운스한다.
- **FR-009**: 제목이 0개일 때 아웃라인은 "장면에 큰 제목을 달면 여기 목차가 생겨요." 안내를 표시해야 한다.
- **FR-010**: 아웃라인 파생은 백엔드/데이터 모델 변경 없이 본문에서 순수하게 파생해야 한다(저장·전송 없음).

**인물 노트 (US2)**

- **FR-011**: 인물 노트 패널은 현재 작품의 등장인물 목록을 이름·한 줄 설명으로 표시해야 한다(기존 등장인물 데이터 재사용).
- **FR-012**: 각 인물의 상세 노트를 펼침/접힘으로 볼 수 있어야 한다.
- **FR-013**: 인라인 입력으로 이름(과 선택적 한 줄)을 넣어 새 인물을 빠르게 추가할 수 있어야 하며, 추가 후 목록에 반영되어야 한다.
- **FR-014**: 상세 수정·삭제·재정렬은 본 패널에서 구현하지 않고 기존 등장인물 화면(`/projects/[id]/characters`)으로 링크해야 한다.
- **FR-015**: 인물이 0명일 때 "곁에 둘 인물을 추가." 안내와 빠른 추가 입력을 표시해야 한다.
- **FR-016**: 이름이 빈 빠른 추가는 거부/비활성되어야 하고, 추가 실패 시 사용자에게 알리며 입력값을 보존해야 한다.

**곁쪽지 (US3)**

- **FR-017**: 곁쪽지(연결 메모 목록·고정·해제)는 기존 동작을 그대로 유지하며 우측 스택 하단에 배치되어야 한다(회귀 0).

**접근성 / 한국어 (전 범위)**

- **FR-018**: 패널 텍스트 대비는 ≥4.5:1을 만족해야 한다(기존 토큰 계승).
- **FR-019**: 패널 토글과 아웃라인 항목은 키보드로 포커스·조작 가능해야 하며, 현재 섹션은 `aria-current`, 접기 상태는 `aria-expanded`로 노출해야 한다.
- **FR-020**: `prefers-reduced-motion`이 켜지면 아웃라인 점프는 즉시 이동으로 대체해야 한다.
- **FR-021**: UI와 본문은 한국어 1차이며, 한국어 제목 텍스트의 heading 파생이 정상 동작해야 한다.

**제품 원칙 (영구 제약)**

- **FR-022**: AI 보조(Assistant)와 음악/분위기(Ambience) 기능을 추가하지 않는다.

### Key Entities *(include if feature involves data)*

- **목차 항목 (OutlineItem)**: 원고 본문의 한 제목을 가리키는 파생 표시값. 속성 — 레벨(1·2), 텍스트, 본문 내 위치(점프 대상). 저장되지 않으며 본문에서 매번 파생된다.
- **등장인물 (Character)**: 기존 시스템의 인물 엔티티(이름·한 줄 설명·상세 노트 등)를 **재사용**. 본 기능은 새 필드·새 저장소를 만들지 않고 읽기·빠른 생성만 수행한다.
- **연결 메모 (곁쪽지)**: 기존 메모 연결 시스템을 **재사용**. 본 기능에서 동작 변경 없음.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 제목이 있는 원고를 열면 사용자가 추가 동작 없이 좌측에서 전체 장면 구조를 한눈에 보고, 원하는 장면으로 한 번의 클릭으로 이동할 수 있다.
- **SC-002**: 등장인물을 확인하려고 집필실을 떠나 다른 화면으로 이동할 필요가 없다 — 인물 목록·설명을 같은 화면에서 본다.
- **SC-003**: 새 인물 추가가 집필 흐름을 끊지 않는다 — 같은 화면에서 이름 한 줄로 추가가 끝난다.
- **SC-004**: 작가가 몰입하고 싶을 때 한 번의 토글로 양옆 패널을 모두 치워 원고만 남길 수 있다.
- **SC-005**: 기존 집필 경험(본문 작성, 한국어 IME 조합, 자동저장, 페이지 분할, 곁쪽지)에 회귀가 없다.
- **SC-006**: 키보드만으로 패널 토글·아웃라인 이동이 가능하고, 모션 줄이기 설정에서 점프가 즉시 이동으로 동작한다(WCAG 2.1 AA 충족).

## Assumptions

- 집필실은 인증된 사용자가 자신의 작품을 여는 화면이며, 기존 인증 게이트(`useAuthGuard`)·문서 세션·작업 세션 흐름을 그대로 따른다.
- 아웃라인은 본문 제목(level 1·2)만을 대상으로 한다. level 3 이하 제목, 장면별 설명 메타, 항목 순서 변경(드래그)은 본 범위 밖(후속 spec).
- 등장인물의 "한 줄 설명"과 "상세 노트"는 기존 등장인물 엔티티가 이미 제공하는 필드를 매핑해 표시하며, 본 기능에서 새 필드를 추가하지 않는다.
- 곁쪽지(연결 메모)는 기존 컴포넌트를 그대로 재사용하며 동작을 바꾸지 않는다.
- 백엔드 변경은 없다 — 아웃라인은 클라이언트 순수 파생, 인물은 기존 등장인물 API, 곁쪽지는 기존 메모 시스템.
- 본 작업은 `feat/studio-three-panel` 브랜치(Scope A 웜 리스킨 위 스택)에서 진행한다.
- 무관 부채(`documents.test.ts`의 version 타입 typecheck 에러, 016 잔여)는 본 작업에서 건드리지 않는다.
