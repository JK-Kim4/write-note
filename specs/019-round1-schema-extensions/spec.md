**Status**: Draft

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 곁쪽지 버리기 + 되돌리기 (Priority: P1)

작가가 책상(미분류 곁쪽지 보관 화면)에서 더 이상 필요 없는 곁쪽지를 버린다. 버린 직후 "되돌리기" 안내가 잠시 떠서, 실수로 버렸을 때 한 번의 클릭으로 곁쪽지를 — 작품 연결까지 그대로 — 복구할 수 있다. desktop 앱에서 이미 검증된 흐름의 web 1:1 이식이다.

**Why this priority**: 현재 web 은 삭제 수단 자체가 없어 책상이 잡동사니로 쌓이기만 한다(015 때 의도적 보류). 또한 삭제가 생기는 순간 "실수 삭제 = 영구 유실" 위험이 함께 생기므로 되돌리기가 반드시 동반돼야 한다. 데이터 유실 방지가 걸려 있어 최우선.

**Independent Test**: 책상에서 곁쪽지 1장을 버리고 → 목록에서 사라짐 확인 → 되돌리기 클릭 → 목록 복귀 + 기존 작품 연결 보존 확인. 다른 두 스토리 없이 단독으로 검증 가능.

**Acceptance Scenarios**:

1. **Given** 책상에 곁쪽지가 있을 때, **When** 곁쪽지의 버리기 버튼을 누르면, **Then** 그 곁쪽지가 목록에서 즉시 사라지고 "되돌리기" 액션이 포함된 안내(토스트)가 나타난다
2. **Given** 버리기 직후 되돌리기 안내가 떠 있을 때, **When** 되돌리기를 누르면, **Then** 곁쪽지가 목록에 복귀하고 버리기 전의 작품 연결·고정 상태가 그대로 보존돼 있다
3. **Given** 되돌리기 안내를 닫거나 무시했을 때, **Then** 곁쪽지는 화면에서 계속 보이지 않지만 데이터는 영구 보관된다(자동 정리 없음)
4. **Given** 작품에 연결된 곁쪽지를 버렸을 때, **Then** 그 작품의 서랍(작품별 곁쪽지 목록)과 재진입 카드 후보에서도 보이지 않는다
5. **Given** 작품에 연결됐던 곁쪽지를 되돌렸을 때, **Then** 그 작품의 서랍에 다시 나타난다

### User Story 2 - 환경설정 다기기 동기화 (Priority: P2)

작가가 한 기기에서 맞춰 둔 환경설정 — 테마(라이트/다크/시스템), 작성 모드(원고지/에디터), 원고지 크기(200/400/1000자) — 이 계정에 묶여 서버에 저장되어, 다른 기기에서 로그인해도 같은 작업 환경이 그대로 펼쳐진다.

**Why this priority**: 지금은 설정이 브라우저(기기) 로컬에만 저장돼 기기를 옮기면 매번 다시 맞춰야 한다. 글쓰기 환경의 영속이라는 제품 본질("컨텍스트가 안 죽는 작업공간")에 직결되지만, 데이터 유실 위험은 없으므로 US1 다음.

**Independent Test**: 기기(또는 브라우저 프로필) A 에서 다크 테마로 변경 → 기기 B 에서 같은 계정으로 로그인 → 다크 테마가 적용돼 있는지 확인. 단독 검증 가능.

**Acceptance Scenarios**:

1. **Given** 기기 A 에서 테마를 다크로 변경했을 때, **When** 기기 B 에서 같은 계정으로 로그인하면, **Then** 기기 B 도 다크 테마로 표시된다 (작성 모드·원고지 크기도 동일)
2. **Given** 서버에 저장된 설정이 아직 없는 사용자(기존 사용자 최초 진입 포함)일 때, **Then** 현재 기기의 로컬 설정값이 그대로 유지되고, 이후 변경분부터 서버에 저장된다
3. **Given** 서버에 일시적으로 연결할 수 없을 때(오프라인·서버 오류), **Then** 설정 변경은 기기 로컬에서 즉시 동작하고 화면 사용이 차단되지 않는다
4. **Given** 다크 테마 사용자가 페이지를 새로고침했을 때, **Then** 밝은 화면이 번쩍이는 현상(FOUC) 없이 처음부터 다크로 그려진다

### User Story 3 - 등장인물 정보 확장 + 관리 화면 진입 (Priority: P3)

작가가 등장인물에게 이름·한 줄 설명·노트에 더해 **나이, 성별, 특징**을 기록한다. 나이는 "17세 가량", "불명" 같은 자유 표현을 허용하고, 성별은 선택지(남/여/기타, 비움 가능)로 고른다. 또한 좌측 네비게이션(Rail)에 등장인물 메뉴가 생겨 어느 화면에서든 인물 관리로 바로 들어갈 수 있다.

**Why this priority**: 인물 정보가 빈약하면 집필 중 인물 참조(집필실 인물 패널)의 가치가 떨어진다. 다만 기존 기능이 동작 중인 상태의 확장이라 유실 위험·환경 단절보다는 후순위.

**Independent Test**: 인물 관리 화면에서 새 인물에 나이·성별·특징을 채워 저장 → 목록·집필실 인물 패널에서 표시 확인. Rail 메뉴로 인물 관리 화면 진입 확인. 단독 검증 가능.

**Acceptance Scenarios**:

1. **Given** 등장인물 생성/편집 폼에서, **When** 나이("17세 가량")·성별(선택지에서 선택)·특징을 입력하고 저장하면, **Then** 목록과 상세에 입력한 값이 그대로 표시된다
2. **Given** 확장 이전에 만들어진 기존 인물을 열었을 때, **Then** 새 필드는 빈 값으로 표시되고 기존 정보(이름·한 줄 설명·노트)는 그대로이며, 새 필드만 채워 저장할 수 있다
3. **Given** 성별 입력에서, **When** 아무것도 선택하지 않고 저장하면, **Then** 비움(미설정)으로 정상 저장된다 — 구상 단계 인물 허용
4. **Given** 어떤 화면에 있든, **When** 좌측 네비게이션의 등장인물 메뉴를 누르면, **Then** 인물 관리 화면으로 진입한다 (목적지 규칙은 Assumptions 참조)

---

### Edge Cases

- 이미 버린 곁쪽지를 다시 버리거나, 버려지지 않은 곁쪽지를 되돌리려 하면? → 멱등 처리(상태 변화 없이 성공) 또는 명확한 오류 — 중복 클릭·탭 중복으로 발생 가능
- 존재하지 않거나 남의 곁쪽지를 되돌리려 하면? → 소유권 검증 후 거부(기존 삭제와 동일한 권한 규칙)
- 작품의 "고정(재진입 한 장)" 곁쪽지를 버리면? → 재진입 카드 후보에서 즉시 제외, 되돌리면 고정 상태까지 복귀
- 버린 곁쪽지가 검색·인물별 추림 등 모든 목록 표면에서 빠지는가? → 목록·검색·서랍·재진입 카드 전 표면에서 제외
- 두 기기에서 거의 동시에 설정을 바꾸면? → 마지막 저장이 이긴다(설정은 사용자 단일 주체의 낮은 빈도 변경이라 충돌 병합 불요)
- 서버 설정과 로컬 설정이 다른 채 접속하면? → 서버값이 기준(SoT), 로컬은 표시 가속용 사본으로 갱신
- 성별에 허용 목록 밖의 값이 들어오면? → 서버가 거부(허용 값: 남/여/기타/비움)
- 작품이 하나도 없는 계정이 Rail 등장인물 메뉴를 누르면? → 인물은 작품 종속이므로 작품 생성 유도 화면으로 안내 (Assumptions 의 목적지 규칙)

## Requirements *(mandatory)*

### Functional Requirements

**곁쪽지 버리기/되돌리기 (US1)**

- **FR-001**: 사용자는 책상 화면에서 곁쪽지를 버릴 수 있어야 한다. 버려진 곁쪽지는 모든 곁쪽지 노출 표면(책상 목록·검색·작품 서랍·인물별 추림·재진입 카드)에서 보이지 않아야 한다
- **FR-002**: 곁쪽지 버리기는 데이터를 즉시 파기하지 않고 "버려진 상태 표시"로 보존해야 하며, 보존 기한 없이 영구 보관한다(자동 정리 없음)
- **FR-003**: 사용자는 버린 곁쪽지를 되돌릴 수 있어야 하며, 되돌린 곁쪽지는 버리기 전의 작품 연결·인물 연결·고정 상태를 그대로 회복해야 한다
- **FR-004**: 버리기 직후 화면에는 되돌리기 액션을 포함한 일시적 안내가 표시되어야 하고, 목록에서는 즉시(서버 응답 대기 없이) 사라져야 한다
- **FR-005**: 버리기·되돌리기는 곁쪽지 소유자만 수행할 수 있어야 한다(기존 곁쪽지 권한 규칙과 동일)

**환경설정 서버 영속 (US2)**

- **FR-006**: 시스템은 사용자별 환경설정 — 테마(라이트/다크/시스템), 작성 모드(원고지/에디터), 원고지 크기(200/400/1000자) — 을 계정에 묶어 서버에 저장해야 한다
- **FR-007**: 사용자가 설정을 변경하면 서버 저장값이 갱신되고, 같은 계정의 다른 기기는 다음 진입(로그인·새로고침) 시 그 값을 반영해야 한다
- **FR-008**: 서버 저장값과 기기 로컬값이 다르면 서버값을 기준으로 한다. 단, 서버에 저장값이 없는 사용자는 현재 로컬값을 유지하고 이후 변경분부터 서버에 쌓는다(기존 사용자 설정 유실 금지)
- **FR-009**: 서버에 연결할 수 없는 동안에도 설정 변경·표시는 기기 로컬로 정상 동작해야 하며, 페이지 첫 그리기 시 테마 깜빡임(FOUC)이 없어야 한다
- **FR-010**: 설정 저장 구조는 이후 항목 추가(예: Round 2 용지 크기)를 스키마 파괴 없이 수용할 수 있어야 한다

**등장인물 확장 (US3)**

- **FR-011**: 등장인물에 나이(자유 텍스트 — "17세 가량"/"불명" 등 허용), 성별(고정 선택지: 남/여/기타, 비움 허용), 특징(자유 텍스트)을 기록·수정할 수 있어야 한다. "소개" 역할은 기존 한 줄 설명 필드가 계속 담당하며 신설하지 않는다
- **FR-012**: 시스템은 성별에 허용 선택지 외의 값을 거부해야 한다
- **FR-013**: 확장 이전에 생성된 등장인물은 새 필드가 빈 값인 상태로 기존과 동일하게 조회·수정 가능해야 한다(기존 데이터 무손상)
- **FR-014**: 인물 목록·상세·집필실 인물 패널 등 인물 정보를 보여주는 표면에서 새 필드 값을 확인할 수 있어야 한다
- **FR-015**: 좌측 네비게이션(Rail)에 등장인물 진입 메뉴가 있어야 한다. 단, 일부 화면에서 Rail 자체가 사라지는 기존 문제의 해소는 본 범위가 아니다(Round 2 B2/#50)

### Key Entities

- **곁쪽지(Memo)**: 작가가 빠르게 잡아둔 메모. 본 기능으로 "버려진 시각" 속성이 추가됨 — 값이 있으면 모든 노출 표면에서 숨겨지고, 비우면 복귀. 작품 연결·인물 연결·고정 상태는 버려져도 파기되지 않고 보존
- **사용자 설정(User Settings)**: 계정 단위 환경설정 묶음(테마·작성 모드·원고지 크기). 사용자당 하나, 항목 추가에 열린 구조. 서버가 기준(SoT)이고 기기 로컬 사본은 표시 가속용
- **등장인물(Character)**: 작품에 속한 인물. 기존 이름·한 줄 설명·노트·표시 순서에 나이(자유 텍스트)·성별(제한 선택지, 비움 허용)·특징(자유 텍스트)이 추가됨

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 책상에서 곁쪽지 버리기 → 되돌리기 완료까지 클릭 2회 이내, 되돌린 곁쪽지의 작품 연결·고정 상태 보존율 100%
- **SC-002**: 버려진 곁쪽지가 책상·검색·작품 서랍·재진입 카드 어느 표면에도 노출되지 않음(노출 0건)
- **SC-003**: 기기 A 에서 변경한 환경설정 3종이 기기 B 의 다음 진입 시 100% 동일하게 반영됨
- **SC-004**: 다크 테마 사용자의 페이지 새로고침 시 밝은 화면 번쩍임(FOUC) 발생 0회
- **SC-005**: 나이·성별·특징을 저장한 인물의 재조회 값 정합 100%, 확장 이전 인물의 기존 데이터 변동 0건
- **SC-006**: 어느 화면에서든 등장인물 관리 화면까지 클릭 2회 이내 진입 가능

## Assumptions

- **세 스토리는 상호 의존이 없다** — 각각 독립 구현·검증·dogfooding 가능해야 하며, 하나만 구현해도 그 자체로 가치가 있다
- **삭제 정책(사용자 확정 2026-06-11)**: 버려진 곁쪽지는 영구 보관, 자동 정리 스케줄러 없음 — desktop 과 1:1. 추후 필요 시 정리 기능은 별도 작업
- **설정 범위(사용자 확정 2026-06-11)**: 서버 영속 대상은 기존 환경설정 3종만. 자동저장 관련 신규 설정은 만들지 않는다(자동저장은 항상 켜짐 유지)
- **인물 필드(사용자 확정 2026-06-11)**: 신규는 나이·성별·특징 3개. "소개" 는 기존 한 줄 설명이 담당. 나이는 자유 텍스트, 성별은 남/여/기타 + 비움 허용
- **Rail 등장인물 메뉴 목적지(합리적 default)**: 마지막으로 작업한 작품의 인물 관리 화면으로 진입하고, 그 정보가 없으면 작품 벽(작품 목록)으로 안내해 작품 선택을 유도한다. 작품이 하나도 없으면 작품 생성 유도. — 이 규칙이 부적절하면 clarify 단계에서 조정
- **이식 원본**: 버리기/되돌리기의 상호작용 패턴(즉시 제거 + 되돌리기 안내)은 desktop 앱(MemoInboxScreen)에서 이미 사용자 검증됨 — web 은 동등 경험을 목표로 한다
- **DB 스키마 버전(사용자 확정 2026-06-11)**: 신규 마이그레이션은 V9(곁쪽지)·V10(설정)·V11(인물) 사용. 챕터 설계(Round 2.5)는 V12 이후 번호 사용
- **마이그레이션 적용 범위**: 본 작업에서는 로컬 dev DB 에만 적용한다. 운영(Supabase) 적용은 Round 4 에서 일괄 수행하며 외부 인프라 쓰기는 사용자 컨펌 영역
- **동시 설정 변경**: 설정은 단일 사용자의 낮은 빈도 변경이므로 마지막 쓰기 우선(last-write-wins)으로 충분하다 — 충돌 병합 메커니즘 불요
