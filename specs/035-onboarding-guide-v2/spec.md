# Feature Specification: 온보딩 가이드 고도화 (최초 사용자 온보딩 투어 v2)

**Feature Branch**: `035-onboarding-guide-v2`

**Created**: 2026-06-23

**Status**: Draft

**Input**: User description: 기존 최초 사용자 온보딩 투어(홈 3단계 스포트라이트)를 인트로 설명 카드 3개 + 메뉴 설명 3단계 + "더 보기" 분기 + (더보기 시) 작품 페이지에서 시리즈/작품 만드는 법 가이드로 고도화.

## 배경 *(컨텍스트)*

처음 가입한 작가는 "이 서비스가 무엇을 해주는지" 와 "어디서 무엇을 하는지" 를 빠르게 이해해야 첫 작품을 시작한다. 현재 온보딩은 홈에서 3개 요소를 짚는 짧은 스포트라이트뿐이라 **서비스의 핵심 개념(시리즈로 묶고 한 단위로 내보낸다)** 과 **메뉴 구조** 를 충분히 설명하지 못한다. 본 작업은 그 가이드를 더 풍부한 다단계 흐름으로 고도화한다.

- 용어: 작품을 묶는 단위는 앱 전체 명칭과 동일하게 **"시리즈"** 로 부른다.
- 대상: 온보딩을 아직 보지 않은 신규 사용자(기존 1회성 노출 정책 유지).
- 범위: 프론트엔드 화면 흐름만. 새 데이터·새 화면·새 권한 없음.

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 홈에서 서비스 개념과 메뉴를 이해한다 (Priority: P1)

처음 로그인한 작가에게 가이드가 자동으로 뜬다. 먼저 **서비스가 무엇인지 설명하는 카드 3장** 이 순서대로 보인다 — (1) 시리즈를 만들 수 있다 (2) 시리즈에 작품을 만들거나 담을 수 있다 (3) 시리즈를 하나의 작품 단위로 내보낼 수 있다. 이어서 **메뉴를 순서대로 짚어 설명** 한다 — 작품 → 메모 → 인물. 작가는 이 흐름을 보고 나면(또는 중간에 건너뛰면) 가이드가 끝나고 다시 뜨지 않는다.

**Why this priority**: 본 기능의 핵심. 신규 작가가 "무엇을 하는 서비스이고 어디서 하는지" 를 한 번에 이해하게 만드는 부분으로, 이 슬라이스만으로도 기존 짧은 투어 대비 가치가 크게 향상된다(독립 출시 가능한 MVP).

**Independent Test**: 온보딩 미완료 신규 사용자로 홈에 진입 → 인트로 카드 3장이 순서대로 표시되고 → 작품·메모·인물 메뉴가 차례로 강조·설명되며 → 끝내거나 건너뛰면 종료되고, 재진입 시 다시 뜨지 않는다.

**Acceptance Scenarios**:

1. **Given** 온보딩을 본 적 없는 작가가 홈에 진입, **When** 가이드가 자동 시작, **Then** 서비스 설명 카드 3장이 (시리즈 생성 → 작품 포함 → 단위 내보내기) 순서로 표시된다.
2. **Given** 인트로 카드를 다 본 뒤, **When** 계속 진행, **Then** 작품 → 메모 → 인물 메뉴가 순서대로 강조되고 각 설명이 표시된다.
3. **Given** 가이드 진행 중, **When** 작가가 끝내기(또는 건너뛰기/배경 클릭/ESC)를 함, **Then** 가이드가 종료되고 완료로 기록되어 **다음 진입 시 다시 뜨지 않는다**.
4. **Given** 가이드를 완료(또는 건너뛴) 작가, **When** 홈에 다시 진입, **Then** 가이드가 자동으로 뜨지 않는다.

---

### User Story 2 - 더 알고 싶은 작가는 시리즈·작품 만드는 법을 본다 (Priority: P2)

메뉴 설명이 끝나는 지점에서 작가에게 **"더 보기" 와 "바로 시작"** 두 선택지를 준다. "바로 시작" 을 고르면 가이드가 끝나고 곧장 서비스를 쓴다. "더 보기" 를 고르면 **작품 페이지로 이동** 해, 거기서 **시리즈 만드는 법** 과 **작품 만드는 법**(해당 버튼을 순서대로 강조하고 "여기서 만들어요" 설명)을 안내한다.

**Why this priority**: 핵심 흐름(US1) 위에 얹는 선택적 심화 안내. 원하는 작가만 더 깊이 안내받고, 원치 않으면 바로 사용하게 해 부담을 주지 않는다.

**Independent Test**: 메뉴 설명 종료 시 "더 보기/바로 시작" 선택지가 뜨고 → "바로 시작" 시 즉시 종료, "더 보기" 시 작품 페이지로 이동해 시리즈/작품 생성 버튼이 순서대로 강조·설명된다.

**Acceptance Scenarios**:

1. **Given** 메뉴 설명(인물)까지 끝난 상태, **When** 분기 안내가 표시됨, **Then** "더 보기" 와 "바로 시작" 두 선택지가 보인다.
2. **Given** 분기에서, **When** "바로 시작" 선택, **Then** 가이드가 즉시 종료되고 작가는 곧장 서비스를 사용한다.
3. **Given** 분기에서, **When** "더 보기" 선택, **Then** 작품 페이지로 이동하고, **시리즈 만들기 버튼 → 작품 만들기 버튼** 이 순서대로 강조되며 만드는 법 설명이 표시된다.
4. **Given** "더 보기" 로 진입한 라이브러리 가이드, **When** 작가가 중간에 이탈(다른 화면 이동 등), **Then** 이미 완료로 기록돼 있어 **다음 진입에 긴 인트로가 다시 뜨지 않는다**(이탈 내성).

---

### Edge Cases

- **분기 도중/라이브러리 가이드 중 이탈**: 완료 기록은 메뉴 설명 종료 시점에 이미 저장됨 → 재진입 시 처음부터 다시 뜨지 않는다. 라이브러리 가이드는 "보너스" 라 미완으로 남아도 무방.
- **작품 페이지에 안내 대상 버튼이 보장되는지**: 신규 작가의 작품 페이지(비어 있음)에도 "시리즈 만들기" · "작품 만들기" 진입 버튼이 표시되어 강조 대상이 존재해야 한다(실측 확인됨).
- **페이지 이동과 가이드 시작 타이밍**: "더 보기" 로 작품 페이지에 도착했을 때, 강조할 버튼이 화면에 준비된 뒤 라이브러리 가이드가 시작되어야 한다(빈 강조/깜빡임 없이).
- **중간 종료 경로 일관성**: 끝내기·건너뛰기·배경 클릭·ESC 어느 경로로 끝내도 동일하게 "완료" 처리된다.
- **이미 완료한 사용자**: 자동으로 뜨지 않으며, 다시 보려면 설정 초기화로만 재노출(별도 "다시 보기" 버튼은 범위 외, 기존 정책 유지).
- **한국어 문구**: 모든 카드·설명 문구는 한국어.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 온보딩 미완료 신규 사용자가 홈에 진입하면 가이드가 **자동으로 1회 시작** 되어야 한다.
- **FR-002**: 가이드는 **서비스 소개 카드 3장** 을 순서대로 보여야 한다 — (1) 시리즈 생성, (2) 시리즈에 작품 생성/포함, (3) 시리즈를 하나의 작품 단위로 내보내기. (화면 중앙 카드 형태, 특정 요소를 가리키지 않음)
- **FR-003**: 이어서 **메뉴를 순서대로 강조·설명** 해야 한다 — 작품 → 메모 → 인물.
- **FR-004**: 작품을 묶는 단위의 명칭은 앱 전체 UI와 동일하게 **"시리즈"** 로 표기해야 한다.
- **FR-005**: 메뉴 설명 종료 지점에서 **"더 보기" / "바로 시작" 2지선다** 를 제공해야 한다.
- **FR-006**: "바로 시작" 선택 시 가이드를 **즉시 종료** 하고 서비스 사용으로 넘어가야 한다.
- **FR-007**: "더 보기" 선택 시 **작품 페이지로 이동** 하여 **시리즈 만들기 버튼 → 작품 만들기 버튼** 을 순서대로 강조하고 만드는 법을 **설명형으로 안내**(실제 생성을 강제하지 않음)해야 한다.
- **FR-008**: 가이드 **완료 처리(다시 안 뜸)** 는 **인트로 + 메뉴 설명까지 본 시점**(또는 그 전 종료)에 저장되어야 한다. 라이브러리 가이드 중 이탈해도 다음 진입에 다시 뜨지 않아야 한다.
- **FR-009**: 끝내기·건너뛰기·배경 클릭·ESC 등 **모든 종료 경로** 가 동일하게 완료로 처리되어야 한다.
- **FR-010**: 완료한 사용자에게는 가이드가 **자동으로 다시 뜨지 않아야** 한다.
- **FR-011**: 작품 페이지의 강조 대상 버튼은 **화면에 준비된 뒤** 라이브러리 가이드가 시작되어 빈 강조/깜빡임이 없어야 한다.
- **FR-012**: 본 기능은 **새 데이터·새 서버 기능·새 권한·새 화면(라우트)을 추가하지 않는다**(기존 온보딩 완료 영속·기존 페이지 재사용). 모든 문구는 한국어.

### Key Entities

- 해당 없음 — 신규 데이터 모델 없음. 온보딩 완료 여부는 기존 사용자 설정(완료 플래그)을 재사용한다.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 온보딩 미완료 신규 사용자는 홈 진입 시 **가이드를 100% 자동으로** 접한다(인트로 3카드 → 메뉴 3설명).
- **SC-002**: 가이드를 완료하거나 건너뛴 사용자의 **재진입 시 자동 재노출이 0건** 이다.
- **SC-003**: "바로 시작" 을 선택한 사용자는 추가 단계 없이 **즉시 서비스 사용** 으로 진입한다(분기에서 멈춤 0).
- **SC-004**: "더 보기" 를 선택한 사용자는 작품 페이지에서 **시리즈·작품 만들기 위치를 안내받는다**(강조 대상 누락/빈 강조 0건).
- **SC-005**: 라이브러리 가이드 중 이탈한 사용자도 다음 진입에 **긴 인트로가 다시 뜨지 않는다**(이탈로 인한 재노출 0).
- **SC-006**: 기존 메뉴·작품 페이지의 **동작 회귀가 0건**(가이드 마커 추가가 기존 화면 동작을 바꾸지 않음).

## Assumptions

- 온보딩 1회성 노출 정책(완료 후 자동 재노출 없음)과 완료 영속 메커니즘은 기존 것을 재사용한다.
- "다시 보기"(완료 후 수동 재실행) 버튼은 본 범위 밖이다(기존과 동일하게 설정 초기화로만 재노출).
- 신규 사용자의 작품 페이지에도 "시리즈 만들기" · "작품 만들기" 진입 버튼이 빈 상태에서 표시된다(실측 확인).
- 라이브러리 가이드는 설명형(버튼 위치 안내)이며 실제 시리즈/작품 생성을 강제하지 않는다 → 온보딩으로 인한 실데이터 생성이 없다.
- 배포 전 로컬 dogfooding 으로 전체 흐름을 검증한다(인증 화면이라 자동 테스트만으로 단정하지 않음).
