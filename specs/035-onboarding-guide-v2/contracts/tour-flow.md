# Contract: 투어 흐름 (단계·분기·완료 저장)

온보딩 v2 의 단계 흐름과 종료/완료 계약. 구현(driver.js)과 독립한 행위 정의 — 자동 테스트는 이 계약을 검증한다.

## 단계 순서

| # | 종류 | 타겟 | 문구(요지, 한국어) |
|---|---|---|---|
| 1 | 인트로(중앙) | 없음 | 시리즈를 만들 수 있어요 |
| 2 | 인트로(중앙) | 없음 | 시리즈에 작품을 만들거나 담을 수 있어요 |
| 3 | 인트로(중앙) | 없음 | 시리즈를 하나의 작품 단위로 내보낼 수 있어요 |
| 4 | 스포트라이트 | `[data-tour="nav-works"]` | 작품 메뉴 설명 |
| 5 | 스포트라이트 | `[data-tour="nav-memos"]` | 메모 메뉴 설명 |
| 6 | 스포트라이트 + 분기 | `[data-tour="nav-characters"]` | 인물 메뉴 설명 + "더 보기"/"바로 시작" |

> 2차(라이브러리) 단계는 별도 계약([dom-markers.md](./dom-markers.md) + 아래 분기) — `/library` 에서 시작.

## 분기 (step 6 종료)

- **"바로 시작"**: 완료 저장 → 투어 종료. 추가 이동 없음.
- **"더 보기"**: 완료 저장 → `sessionStorage["writenote.onboarding.stage.v1"]="library"` → `/library` 이동. (1차 투어는 종료)

## 완료 저장 계약 (FR-008)

- **트리거**: step 6 의 분기 선택(둘 다) / step 1~6 어디서든 끝내기·건너뛰기·ESC·배경 클릭.
- **동작**: `onboardingCompleted="true"` 를 서버 설정에 PUT(기존 설정 client) + 로컬 React Query 캐시 갱신.
- **보장**: 한 번 저장되면 이후 홈 자동 시작 안 함(FR-010). 2차 투어 이탈은 추가 저장 불필요(이미 저장됨).

## 2차(라이브러리) 흐름

| # | 종류 | 타겟 | 문구(요지) |
|---|---|---|---|
| L1 | 스포트라이트 | `[data-tour="new-series"]` | 여기서 시리즈를 만들어요 |
| L2 | 스포트라이트 | `[data-tour="new-work-root"]` | 여기서 작품을 만들어요 |

- 시작 조건: `/library` 마운트 + `stage==="library"` + 타겟 DOM 존재. 시작 시 stage 키 제거.
- 설명형(버튼 강조 + 안내). 실제 생성 강제 없음. 종료(완료/건너뛰기) 시 추가 서버 저장 없음.

## 자동 테스트 대상(행위)

- 미완료 → 홈에서 시작 / 완료 → 미시작.
- step 진행 순서(1~6) + 분기에서 "바로 시작"=종료·"더 보기"=핸드오프 set + router.push("/library") 호출.
- 어느 종료 경로든 putSettings({onboardingCompleted:"true"}) 1회 호출.
- /library: stage 있으면 2차 시작·키 제거, 없으면 미시작.
- (시각 스포트라이트·중앙 카드·이동 타이밍 = dogfooding)
