# Data Model: 온보딩 가이드 고도화 (Phase 1)

## 신규 데이터 모델: 없음

새 엔티티·테이블·서버 필드 0. 온보딩 완료 여부는 **기존 사용자 설정** `onboardingCompleted`(허용값 `"true"`, 서버 영속)를 재사용한다.

## 재사용/임시 상태

| 상태 | 위치 | 용도 | 수명 |
|---|---|---|---|
| `onboardingCompleted` | 서버 사용자 설정(기존) | 가이드 완료 여부(자동 재노출 차단) | 영속(기기 무관). `"true"` 면 미시작 |
| `writenote.onboarding.stage.v1` | 클라이언트 `sessionStorage` | "더 보기" → /library 2차 투어 핸드오프 | 임시. 2차 투어 시작 시 즉시 제거(1회성) |

## 투어 상태기계 (클라이언트 흐름)

```
[미완료 사용자 홈 진입]
      │ (onboardingCompleted !== "true")
      ▼
  인트로 카드 1 → 2 → 3            (중앙 popover, 타겟 없음)
      ▼
  메뉴 스포트라이트: 작품 → 메모 → 인물
      ▼
  ── 분기 ──────────────────────────────
   "바로 시작"            "더 보기"
      │                     │
   [완료 저장]          [완료 저장] + stage=library + /library 이동
   투어 종료               │
                           ▼
                   [/library 2차 투어]
                   타겟 준비 대기 → 시리즈 버튼 → 작품 버튼 강조·설명 → 종료
                   (stage 키 제거)

※ 끝내기/건너뛰기/ESC/배경 = 어느 단계든 [완료 저장] + 종료
※ [완료 저장] 은 인트로+메뉴 종료/분기 시점에 1회(이탈 내성). 2차 투어는 완료와 독립.
```

## 검증 규칙 (FR 매핑)

- FR-001/010: `onboardingCompleted !== "true"` 일 때만 자동 시작.
- FR-008: 완료 저장은 인트로+메뉴 종료(또는 그 전 종료)·분기 시점. 2차 투어 이탈은 재노출 유발 안 함.
- FR-002/003/004: 인트로 3카드(시리즈 용어)·메뉴 3(작품/메모/인물).
- FR-005/006/007: 분기 2지선다 / 바로시작=즉시종료 / 더보기=이동+버튼 강조(설명형).
- FR-011: 2차 투어는 타겟 DOM 준비 후 시작.
