## 핵심 결정 (확정) — PRD v0.1 전제 대체

본 spec 작성 중 PRD v0.1 의 전제(§3·§6)가 실제 코드베이스와 어긋남을 발견했고(메모는 사용자 글로벌 + 작품 다대다 저장이며 `title` 없이 `body` 단일), 사용자 결정으로 아래와 같이 재정의했다. 이 결정이 본 spec 의 기준이며 PRD 의 충돌 전제는 폐기한다.

1. **보드 = 독립 객체.** 보드는 작품의 부속이 아니라 사용자(`user_id`) 소유의 독립 엔티티다. 작가는 **작품/시리즈와 무관하게 언제든 보드를 직접 생성**(이름 짓고·삭제) 하며, 매핑은 별개의 선택적 행위다(자동 생성 아님).
2. **보드 ↔ 작품/시리즈 매핑(선택, 0~1 : 0~1).** 보드는 **시리즈(Category, "책"; 0~1개) 그리고/또는 작품(Project, "장"; 0~1개)** 에 매핑할 수 있다. 미매핑 **독립 보드** 도 정상 상태다. 반대로 **한 작품·한 시리즈에 매핑되는 보드도 최대 1개**다(대상당 보드 0~1). 즉 보드↔작품, 보드↔시리즈 모두 0~1 : 0~1 관계다.
3. **노드 = 보드 전용 신규 객체 (기존 메모와 별개).** 보드 위의 "플롯 노드" 는 쪽지 책상/캡처 메모(`memos`) 와 **완전히 분리된 새 엔티티**다. 자체 텍스트 본문·위치·연결을 가진다. 기존 캡처 메모는 보드에 자동 유입되지 않으며(마이그레이션 없음), 작가가 보드에서 노드를 직접 만든다.
4. **노드 제거 ≠ 메모 삭제.** 노드는 보드 전용 객체이므로 노드 제거·보드 삭제는 보드 도메인에만 영향을 주고 캡처 메모에는 영향이 없다.
5. **위치는 노드에 직접 저장.** 노드↔보드가 1:N(한 노드는 한 보드에만 속함) 이므로 위치/겹침 순서를 노드에 직접 둔다(별도 배치 테이블 불필요).

> **용어 분리**: "노드(=플롯 노드, 보드 위 카드)" 와 "메모(=쪽지 책상의 캡처 메모)" 는 서로 다른 객체다. "시리즈" 는 기존 Category 엔티티("책" 단위), "작품" 은 기존 Project 엔티티("장"/개별 작품) 를 가리킨다. 보드 위 항목은 항상 "노드" 로 부른다.

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 보드를 만들어 노드를 배치하고, 그 배치가 유지된다 (Priority: P1)

작가가 (작품을 먼저 만들 필요 없이) 새 플롯 보드를 만든다. 빈 캔버스에서 노드(카드) 를 직접 만들어 본문을 적고, 드래그해 원하는 위치에 두며 이야기 구조를 공간적으로 배치한다. 보드를 닫았다 다시 열어도 노드의 위치·본문과 마지막으로 보던 화면(줌·이동 상태) 이 그대로 복원된다.

**Why this priority**: "작품/시리즈와 무관하게 보드에서 스토리를 잡는다" 는 작가의 핵심 흐름이자 본 기능의 양보 불가 핵심이다. 보드 생성 + 노드 배치 + 영속·복원만으로도 작가는 플롯을 코르크보드처럼 짜는 가치를 얻으며, 매핑·연결 없이 독립 보드 하나로 출시 가능한 MVP 다.

**Independent Test**: 새 보드를 만들어 노드 몇 개를 만들고 본문을 넣어 드래그로 옮긴 뒤, 줌/이동을 하고 보드를 떠났다 재진입했을 때 노드의 위치·본문·화면 상태가 동일하게 복원되는지로 단독 검증 가능(작품 매핑·연결 없이).

**Acceptance Scenarios**:

1. **Given** 작품이 하나도 없거나 무관한 상태, **When** 새 보드 만들기를 하면, **Then** 어떤 작품/시리즈에도 매핑되지 않은 독립 보드가 만들어지고 빈 캔버스로 열린다.
2. **Given** 보드가 열린 상태, **When** 빈 공간에서 새 노드 만들기를 하면, **Then** 그 위치에 노드가 생성되어 본문을 입력할 수 있고 영속된다.
3. **Given** 노드가 선택된 상태, **When** 본문을 수정하면, **Then** 변경이 저장되고 재진입 시 반영된다.
4. **Given** 보드에 노드가 표시된 상태, **When** 노드를 드래그해 옮기고 손을 떼면, **Then** 그 위치가 저장되고(드래그 중에는 저장하지 않음), 재진입 시 같은 위치로 복원된다.
5. **Given** 여러 노드를 한 번에 선택한 상태, **When** 함께 드래그해 옮기면, **Then** 옮긴 노드 전부의 새 위치가 한 번의 저장으로 영속된다.
6. **Given** 보드를 줌/이동(팬) 한 상태, **When** 조작을 멈추면, **Then** 마지막 화면 상태가 저장되고 재진입 시 같은 줌·위치로 복원된다.
7. **Given** 위치/본문 저장 요청이 실패, **When** 변경을 마치면, **Then** 화면은 즉시 변경을 보여주되(낙관적 반영) 실패 시 직전 상태로 되돌아가고 작가에게 실패가 드러난다.

### User Story 2 - 노드를 연결해 관계를 표현하고, 연결을 따라 탐색한다 (Priority: P2)

작가가 한 노드의 연결점에서 다른 노드로 선을 그어 두 노드 사이의 관계(예: 사건의 인과, 복선↔회수) 를 명시적으로 표현한다. 노드를 선택하면 그 노드로 **들어오는 연결과 나가는 연결(= 연결된 노드 목록/백링크)** 을 본다. 연결은 영속되어 재진입 시 복원된다.

**Why this priority**: 배치(US1) 가 "어디에 있는가" 라면 연결은 "무엇이 무엇과 이어지는가" 로, 플롯의 구조(탐정 보드의 핀-실) 를 드러내는 두 번째 축이다. 배치 위에 독립적으로 얹을 수 있어 P2.

**Independent Test**: 두 노드 사이에 선을 긋고, 한 노드를 선택해 연결 목록에 상대 노드가 들어오는/나가는 방향으로 표시되는지, 연결 삭제 후 사라지는지, 재진입 후 복원되는지로 단독 검증 가능.

**Acceptance Scenarios**:

1. **Given** 노드 A·B 가 보드에 있는 상태, **When** A 의 연결점에서 B 로 선을 그으면, **Then** A→B 연결이 생성·저장되고 화면에 선으로 표시된다.
2. **Given** A→B 연결이 있는 상태, **When** 노드 A 를 선택하면 A 의 "나가는 연결" 에 B 가, B 를 선택하면 B 의 "들어오는 연결" 에 A 가 표시된다.
3. **Given** A→B 연결이 있는 상태, **When** 그 연결을 삭제하면, **Then** 화면·저장에서 사라지고 재진입 시 복원되지 않는다.
4. **Given** A→B 연결이 이미 있는 상태, **When** 같은 방향의 동일 연결을 다시 그으면, **Then** 중복 연결이 생기지 않는다.
5. **Given** A→B 연결이 있는 상태, **When** 노드 B 가 제거되면, **Then** B 에 걸린 연결도 함께 정리되어 끊긴 선(고아 엣지) 이 남지 않는다.

### User Story 3 - 보드를 작품/시리즈에 매핑하고 보드를 관리한다 (Priority: P3)

작가가 (먼저 만들어 둔) 보드를 나중에 작품(Project) 또는 시리즈(Category) 에 매핑해 작업 맥락에 연결한다. 매핑을 해제할 수도 있다. 한 작품/시리즈에서 그곳에 매핑된 보드들을 찾아 열 수 있다. 보드의 이름을 바꾸거나, 더 이상 필요 없는 보드를 삭제해 정리한다.

**Why this priority**: 보드+노드+연결(US1·2) 로 핵심 가치는 확보되므로 작업 맥락 연결(매핑) 과 보드 관리는 통합 편의로 P3. 다만 "보드 먼저 만들고 나중에 작품에 매핑" 흐름의 완성에 필요.

**Independent Test**: 독립 보드를 작품에 매핑한 뒤 그 작품에서 보드가 보이는지, 매핑 해제 시 사라지는지(보드 자체는 잔존), 시리즈 매핑도 동일한지, 보드 삭제 시 그 노드·연결이 함께 사라지고 캡처 메모는 그대로인지로 단독 검증 가능.

**Acceptance Scenarios**:

1. **Given** 미매핑 독립 보드, **When** 그 보드를 한 작품에 매핑하면, **Then** 그 작품에서 보드 목록에 나타나고 보드는 그 작품에 연결된다.
2. **Given** 작품에 매핑된 보드, **When** 매핑을 해제하면, **Then** 작품에서 사라지되 보드(노드·연결) 자체는 보존되어 다시 열 수 있다.
3. **Given** 보드, **When** 시리즈(Category) 에 매핑하면, **Then** 그 시리즈 맥락에서 보드를 찾아 열 수 있다(작품 매핑과 독립적으로 동작).
4. **Given** 이미 보드가 매핑된 작품, **When** 그 작품에 다른 보드를 매핑하려 하면, **Then** 거부된다(작품당 보드 최대 1개 — 기존 매핑을 먼저 해제해야 함). 시리즈도 동일.
5. **Given** 노드·연결이 있는 보드, **When** 보드를 삭제하면, **Then** 그 보드의 노드·연결이 함께 사라지고, 쪽지 책상의 캡처 메모는 영향받지 않는다.
6. **Given** 보드, **When** 이름을 변경하면, **Then** 변경된 이름이 저장·복원된다.

---

### Edge Cases

- 자기 자신으로의 연결(A→A) 시도 → 허용하지 않는다.
- 동일 두 노드 사이 반대 방향 연결(A→B 와 B→A) → 방향이 다르므로 별개 연결로 허용.
- 보드가 작품과 시리즈에 동시에 매핑된 경우(예: 작품 매핑의 그 작품이 보드 시리즈 매핑과 다른 시리즈 소속) → 두 매핑은 독립으로 취급하며 v1 은 둘 사이 정합성을 강제하지 않는다(필요 시 후속 정책).
- 노드/보드 삭제 시 → 보드 도메인(노드·연결) 에만 영향, 캡처 메모 0 영향.
- 매핑된 작품/시리즈가 삭제되는 경우 → 보드는 보존되고 그 매핑만 해제된다(보드는 독립 객체).
- 저장(위치/뷰포트/본문/연결/매핑) 실패·네트워크 끊김 → 낙관적 반영 후 실패 시 직전 상태로 롤백, 작가에게 드러냄.
- 대량 노드(수백 개) 가 한 보드에 있어도 열기·드래그·줌이 막힘 없이 동작.
- 본문이 매우 긴 노드 → 카드는 본문을 말줄임/요약해 일정 크기로 표시.
- 타 사용자의 보드/작품/시리즈에는 접근할 수 없어야 한다(본인 소유만).

## Requirements *(mandatory)*

### Functional Requirements

**보드 생성·열기·캔버스 (US1)**

- **FR-001**: 작가는 작품 없이도 새 보드를 직접 만들 수 있어야 하며, 새 보드는 어떤 작품/시리즈에도 매핑되지 않은 독립 상태로 시작한다.
- **FR-002**: 보드는 생성한 작가(`user_id`) 소유이며, 시스템은 본인 소유 보드만 노출·조작 가능하게 해야 한다(타 사용자 접근 차단).
- **FR-003**: 시스템은 작가가 보드를 열면 그 보드의 노드 목록 + 노드 간 연결 + 마지막 화면 상태(줌·이동) 를 함께 불러와 한 화면으로 표시해야 한다.
- **FR-004**: 시스템은 무한 캔버스에서 줌·이동(팬) 을 지원해야 한다.
- **FR-005**: 시스템은 노드가 0개인 보드도 오류 없이 빈 캔버스로 정상 표시해야 한다.

**노드 생성·편집·배치·영속 (US1)**

- **FR-006**: 작가는 보드의 빈 공간에서 새 노드를 만들 수 있어야 하며, 노드는 생성 시점에 위치를 부여받아 영속되어야 한다.
- **FR-007**: 작가는 노드의 텍스트 본문을 작성·편집할 수 있어야 하고, 변경이 저장·복원되어야 한다.
- **FR-008**: 작가는 노드를 드래그해 위치를 옮길 수 있어야 하며, 위치 저장은 **드래그 종료 시점에만** 일어나고 드래그 중에는 저장하지 않는다.
- **FR-009**: 작가는 여러 노드를 선택해 함께 이동할 수 있어야 하고, 이동한 노드들의 새 위치가 한 번의 저장 동작으로 영속되어야 한다(단건 N회 저장 아님).
- **FR-010**: 시스템은 노드 위치를 줌·화면 이동과 무관한 캔버스 절대 좌표로 저장하며, 음수·소수 좌표를 허용해야 한다.
- **FR-011**: 시스템은 노드의 겹침 순서를 보존해야 한다.
- **FR-012**: 시스템은 마지막 화면 상태(줌 배율·이동 위치) 를 보드 단위로 저장하고, 조작이 멈춘 뒤 잠시 모아서 1회 저장해야 한다.
- **FR-013**: 시스템은 재진입 시 노드의 위치·본문·연결·마지막 화면 상태를 저장된 그대로 복원해야 한다.
- **FR-014**: 모든 위치·본문·연결·화면 상태·매핑 변경은 화면에 즉시(낙관적) 반영하고, 저장 실패 시 직전 상태로 되돌리며 실패를 작가에게 드러내야 한다.
- **FR-015**: 노드 카드는 노드 본문의 앞부분을 말줄임/요약해 일정 크기로 표시해야 한다.
- **FR-016**: 작가는 노드를 제거할 수 있어야 하며, 제거는 보드(노드·연결) 에만 영향을 주고 쪽지 책상의 캡처 메모에는 영향을 주지 않아야 한다.

**연결(엣지) / 백링크 (US2)**

- **FR-017**: 작가는 한 노드의 연결점에서 다른 노드로 선을 그어 방향 있는 연결을 만들 수 있어야 하며, 연결은 영속·복원되어야 한다.
- **FR-018**: 작가는 기존 연결을 삭제할 수 있어야 한다.
- **FR-019**: 시스템은 같은 방향의 동일한 두 노드 사이 연결이 중복 생성되지 않도록 해야 한다.
- **FR-020**: 시스템은 자기 자신으로의 연결을 허용하지 않아야 한다.
- **FR-021**: 시스템은 연결을 같은 보드 안의 노드들 사이로 한정해야 한다(보드 횡단 연결 없음).
- **FR-022**: 작가가 노드를 선택하면, 그 노드로 들어오는 연결과 나가는 연결(= 연결된 노드 목록/백링크) 을 방향이 구분되게 표시해야 한다.
- **FR-023**: 노드가 제거되면 그 노드에 걸린 연결도 함께 정리되어 끊긴 연결이 남지 않아야 한다.

**보드 ↔ 작품/시리즈 매핑 · 보드 관리 (US3)**

- **FR-024**: 작가는 보드를 작품(Project) 에 매핑하거나 매핑을 해제할 수 있어야 한다(보드당 작품 0~1개).
- **FR-025**: 작가는 보드를 시리즈(Category) 에 매핑하거나 매핑을 해제할 수 있어야 한다(보드당 시리즈 0~1개). 작품 매핑과 독립적으로 동작한다.
- **FR-026**: 시스템은 한 작품·한 시리즈에 매핑되는 보드를 최대 1개로 제한해야 하며(대상당 0~1), 작가가 작품/시리즈에서 그 매핑된 보드를 찾아 열 수 있게 해야 한다. 이미 보드가 매핑된 대상에 다른 보드를 매핑하려는 시도는 거부된다(기존 매핑 해제 선행).
- **FR-027**: 매핑된 작품/시리즈가 삭제되어도 보드(노드·연결) 는 보존되고 해당 매핑만 해제되어야 한다.
- **FR-028**: 작가는 보드의 이름을 지정·변경할 수 있어야 하고, 변경이 저장·복원되어야 한다.
- **FR-029**: 작가는 보드를 삭제할 수 있어야 하며, 삭제 시 그 보드의 노드·연결이 함께 정리되고 캡처 메모에는 영향이 없어야 한다.

### Key Entities *(include if feature involves data)*

- **플롯 보드(Board)**: 사용자(`user_id`) 소유의 독립 캔버스. 이름, 마지막 화면 상태(줌·이동) 를 가진다. 선택적으로 시리즈(Category, 0~1) 그리고/또는 작품(Project, 0~1) 에 매핑하며, 반대로 작품/시리즈당 보드도 최대 1개(보드↔작품, 보드↔시리즈 모두 0~1 : 0~1). 쪽지 책상 메모 도메인과 독립.
- **플롯 노드(Node)**: 보드 위의 카드. **기존 캡처 메모(`memos`) 와 별개의 신규 객체.** 자체 텍스트 본문, 캔버스 절대 좌표(위치), 겹침 순서를 가진다. 정확히 한 보드에 속함(1:N). v1 은 단일 유형.
- **연결(Edge)**: 같은 보드 안 두 노드 사이의 방향 있는 관계. 출발 노드·도착 노드를 가리킨다. 같은 방향 동일 쌍은 유일, 자기참조 불가. v1 은 관계 유형 라벨 없음.
- **(기존) 시리즈(Category) / 작품(Project)**: 보드 매핑의 대상. 본 기능이 변경하지 않으며 매핑 관계만 추가로 참조한다.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 작가는 작품을 만들지 않고도 새 보드를 만들어 노드를 배치하고, 재진입 시 노드의 위치·본문·화면 상태가 **100% 동일하게** 복원된 것을 확인할 수 있다.
- **SC-002**: 드래그로 노드를 옮기는 동안 화면 움직임이 즉각적이며, 위치 저장은 손을 뗀 뒤에만 일어나 드래그 중 불필요한 저장이 **0회** 발생한다.
- **SC-003**: 한 보드에 노드가 **수백 개(예: 300개)** 있어도 보드 열기·드래그·줌이 막힘 없이 동작한다.
- **SC-004**: 두 노드를 연결한 뒤 한 노드를 선택하면, 연결된 상대 노드가 들어오는/나가는 방향으로 **정확히** 표시된다.
- **SC-005**: 독립 보드를 작품에 매핑하면 그 작품에서 즉시 보이고, 매핑 해제 시 작품에서 사라지되 보드는 보존되어 다시 열 수 있다(보드 데이터 유실 0).
- **SC-006**: 위치·본문·연결·화면 상태·매핑 저장이 실패한 경우, 화면은 직전 상태로 되돌아가고 작가가 실패를 인지할 수 있다.
- **SC-007**: 플롯 보드 기능 도입·사용(노드/보드 생성·삭제 포함) 이 기존 쪽지 책상/캡처 메모 데이터·동작에 **0건** 영향을 준다.

## Assumptions

사용자 확정 사항(보드=독립 객체 / 보드↔시리즈·작품 매핑 / 노드=보드 전용 신규 객체 / 스토리=기존 시리즈) 외, PRD §11 미결정은 Phase 1·비목표에 정합한 안전 기본값으로 해소:

- **매핑 카디널리티(확정)**: 보드↔작품, 보드↔시리즈 모두 **0~1 : 0~1**. 즉 보드당 작품 0~1 + 시리즈 0~1, 작품당 보드 0~1, 시리즈당 보드 0~1. 두 매핑(작품/시리즈)은 독립이며 정합성 미강제. (한 작품/시리즈에 여러 보드, 한 보드를 여러 대상에 공유하는 N:M 은 v1 밖.)
- **단일 사용자 v1 (PRD §11.6 → §2 비목표)**: 실시간 공동 편집 비목표. 화면 상태(뷰포트) 는 보드 단위 저장.
- **단일 노드 유형 v1 (PRD §11.4)**: 유형 구분·시각 차별화는 Phase 2.
- **시퀀스/타임라인 뷰 제외 (PRD §11.5)**: Phase 3.
- **연결 유형(relation_type) 제외 (PRD §5.3)**: v1 은 라벨 없는 단순 방향 연결.
- **고정 크기 카드 v1 (PRD §11.7)**: 리사이즈 미지원, 본문 말줄임.
- **실행취소(undo/redo) 제외 (PRD §11.8)**: v1 밖.
- **인라인 `[[ ]]` 위키링크 제외 (§2 비목표)**: Phase 2.
- **노드 본문은 평문 텍스트 v1**: 구조화 포맷·리치 에디터는 후속.
- **캡처 메모 연동 제외 v1**: 기존 캡처 메모를 보드로 가져오기/연결은 v1 밖. 추후 별도 기능으로 검토.
- **기존 시리즈/작품 도메인 무변경**: 매핑 관계만 추가하며 Category/Project 의 기존 동작은 바꾸지 않는다.

## Dependencies

- 기존 인증(JWT) — 보드는 로그인한 작가 본인 소유로만 다룬다.
- 작품(Project) 도메인 — 보드 매핑 대상(0~1) 및 소유자(`user_id`) 정합.
- 시리즈(Category) 도메인 — 보드 매핑 대상(0~1).
- 기존 쪽지 책상/캡처 메모(`memos`) 도메인과는 **독립** — 본 기능은 그 도메인을 변경하지 않으며, 노드는 별개 신규 객체다.
