# Quickstart — 공유하기 (046) dogfooding

> 자동 게이트(Testcontainers IT + vitest)는 라운드별 GREEN. 본 체크리스트 = **인증 뒤·시각 dogfooding(§19 별도 수동 게이트)**. 로컬 3종 기동 필요([[local-dogfooding-needs-backend]]): DB(docker)→BE(bootRun)→FE(pnpm dev). 마이그레이션 로컬 적용은 **사용자 컨펌 후**.

## R1 — 공유 + 스냅샷 + 공개 읽기 (코어)
- [ ] 작품 공유 링크 생성 → 추측불가 토큰 URL 수령, 내 공유 목록 표시.
- [ ] 시크릿창(비로그인)에서 그 URL → 본문 읽기 전용 표시(편집 불가).
- [ ] 작가가 원문 수정 후 같은 URL 재열람 → 공유본 불변(스냅샷).
- [ ] 링크 끄기 → 그 URL 접근 시 "더 이상 볼 수 없는 링크".
- [ ] 잘못된 토큰 → 동일 안내(대상 존재 비노출).
- [ ] 공유 페이지 `<meta name="robots" content="noindex">` 확인.

## R2 — 위치 지정 댓글 (작가 전용 비공개)
- [ ] 로그인 회원 A 가 텍스트 구간 드래그 → 댓글 작성 → 저장·표시.
- [ ] 비로그인 → 댓글 입력 수단 없음(읽기만).
- [ ] 다른 회원 B 가 같은 공유본 열람 → A 의 댓글 안 보임(B 자기 것만).
- [ ] 작가 인박스 → A·B 댓글 전체 보임, 클릭 시 해당 구간 이동.
- [ ] A 가 자기 댓글 삭제 가능, B 댓글 삭제 불가(403).

## R3 — 시리즈 공유 + 삭제 수명주기
- [ ] 작품 여러 개 시리즈 → 일부만 공개 선택 → 공유 페이지에 선택분만.
- [ ] 시리즈에 새 작품 추가 → 공유 페이지 미자동노출(명시 추가 전).
- [ ] 공유한 작품 삭제 → 링크 비활성 + 작가 인박스에 댓글 보존(title_snapshot 표시).

## R4/R5 — FE
- [ ] 공유 관리 UI: 생성(작품/시리즈)·공개작품 선택·끄기·목록.
- [ ] 공개 페이지: 비로그인 읽기·noindex·회원 댓글 작성(구간 하이라이트)·자기 댓글만.
- [ ] 라이트/다크 + 한국어 본문 1문단 표시 정상.
- [ ] 게이트: `cd frontend && pnpm lint && pnpm typecheck && pnpm test && pnpm build`.

## 통과 단정 규율 (§25)
전 항목을 사용자가 확인하기 전 "통과" 단정 금지. 일부 항목 통과 ≠ 전체.
