## 핵심 결정 (확정) — worksheet / 038 spec 화해

본 spec 은 보드 UX 작업지시서(`docs/board/board-ux-worksheet.md`) **TASK-1(카드 만들기)** 의 미구현 잔여를 회수한다. 작업 방식 브레인스토밍에서 아래를 사용자 결정으로 확정했고(2026-06-27), 이 결정이 본 spec 의 기준이다.

1. **빈 카드 잔존 허용(저장 시점).** 카드는 생성 시점에 **즉시 저장**되며, **본문이 비어 있어도 저장·잔존**한다(폐기/정리하지 않음). 이는 worksheet TASK-1 완료기준 4개 중 "본문 없이 벗어나면 카드가 남지 않는다" 를 **대체**한다(나머지 3개 기준은 유지). 근거: 현행 "+ 카드" 의 즉시 저장 동작을 유지해 구현을 단순·저위험으로 두고, 작가가 "일단 빈 카드를 흩뿌리고 나중에 채우는" 흐름을 허용.
2. **세 생성 경로 동일 결과.** 상단 "+ 카드" 버튼 / 캔버스 빈 곳 더블클릭 / 빈 보드 안내 버튼은 **모두 같은 결과**를 낸다 — 카드를 즉시 생성하고 **생성 직후 자동으로 본문 편집에 진입**(추가 클릭 없이 바로 타이핑).
3. **빈 보드는 빈 캔버스를 노출하지 않는다.** 카드가 0개인 보드는 빈 격자 캔버스 대신 **중앙 안내**(문구 + "+ 카드 만들기" 버튼)를 보여, 무한 캔버스의 막막함을 제거한다.
4. **038 spec FR-005 정정.** `specs/038-memo-plot-board/spec.md` FR-005("노드 0개 보드도 빈 캔버스로 정상 표시")는 worksheet TASK-1("빈 캔버스 절대 노출 금지")과 정면 모순이었다. 본 spec 이 그 요구를 **대체**하며, 038 FR-005 문구는 worksheet 정합으로 직접 정정한다(룰 28 문서 모순 화해).

> **용어**: "카드(Card)" = 보드 위의 한 장(코드/DB/화면 통일 유비쿼터스 언어, 과거 "메모/노드" 대체). 카드는 **`body` 단일 본문**을 가진다(제목/내용 분리 없음 — worksheet 의 "제목칸" 은 `body` 로 매핑). "보드(Board)" = 캔버스 한 장. "더블클릭 줌" = 캔버스를 더블클릭하면 확대되는 React Flow 기본 동작.

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 빈 보드에서 첫 카드를 막힘없이 시작한다 (Priority: P1)

작가가 카드가 하나도 없는 보드를 연다. 빈 격자 캔버스(막막함)가 아니라 화면 중앙에 안내("여기에 첫 카드를 적어보세요")와 "+ 카드 만들기" 버튼이 보인다. 버튼을 누르면 카드가 생기고 **곧바로 본문을 타이핑**할 수 있다.

**Why this priority**: 무한 캔버스 도구가 첫 사용자에게 막막한 가장 큰 지점이 "빈 화면에서 무엇을 어떻게 시작하나" 다. 빈 보드 안내 + 한 동작 생성·편집은 보드를 "DB 설정"이 아니라 "책상에 카드를 툭 던지는" 느낌으로 만드는 핵심 레버이며, 단독으로도 즉시 가치를 준다.

**Independent Test**: 카드가 0개인 보드를 열어 (a) 빈 격자 캔버스 대신 중앙 안내가 보이는지, (b) "+ 카드 만들기" 버튼을 누르면 카드가 생기고 추가 클릭 없이 본문을 타이핑할 수 있는지, (c) 카드가 생긴 뒤에는 안내가 사라지고 캔버스가 보이는지로 단독 검증 가능.

**Acceptance Scenarios**:

1. **Given** 카드가 0개인 보드, **When** 보드를 열면, **Then** 빈 격자 캔버스 대신 중앙에 안내 문구와 "+ 카드 만들기" 버튼이 표시된다.
2. **Given** 빈 보드 안내가 보이는 상태, **When** "+ 카드 만들기" 를 누르면, **Then** 카드가 생성되고 곧바로 본문 편집 상태가 되어 추가 클릭 없이 타이핑할 수 있다.
3. **Given** 빈 보드 안내에서 카드를 하나 만든 직후, **When** 본문을 비운 채 다른 곳을 누르면, **Then** 그 카드는 사라지지 않고 보드에 남는다(빈 카드 보존).
4. **Given** 보드에 카드가 1개 이상 있는 상태, **When** 보드를 열면, **Then** 안내는 표시되지 않고 캔버스(카드들)가 보인다.

---

### User Story 2 - 캔버스 빈 곳을 더블클릭해 원하는 자리에 카드를 만든다 (Priority: P2)

카드가 있는 보드에서 작가가 캔버스의 빈 공간을 더블클릭하면, **더블클릭한 바로 그 위치**에 카드가 생기고 곧바로 본문을 타이핑할 수 있다(숙련자용 빠른 길).

**Why this priority**: 위치를 의도해 카드를 두는 "빠른 길"은 만들기-배치를 한 동작으로 합쳐 마찰을 더 줄인다. P1(빈 보드 시작) 위에 독립적으로 얹히는 가속 경로라 P2.

**Independent Test**: 카드가 있는 보드에서 빈 곳을 더블클릭해 (a) 그 위치에 카드가 생기는지, (b) 즉시 타이핑 가능한지, (c) 카드/컨트롤 위 더블클릭은 카드 생성이 아니라 기존 동작(카드 본문 편집)인지, (d) 더블클릭으로 캔버스가 확대(줌)되지 않는지로 단독 검증 가능.

**Acceptance Scenarios**:

1. **Given** 카드가 있는 보드, **When** 캔버스의 빈 곳을 더블클릭하면, **Then** 그 위치에 카드가 생성되고 곧바로 본문 편집 상태가 된다.
2. **Given** 카드가 있는 보드, **When** 기존 카드 위를 더블클릭하면, **Then** 새 카드가 생기지 않고 그 카드의 본문 편집이 열린다(기존 동작 보존).
3. **Given** 보드 캔버스, **When** 빈 곳을 더블클릭하면, **Then** 캔버스가 확대(줌)되지 않는다(더블클릭은 카드 생성 용도).
4. **Given** 빈 곳 더블클릭으로 카드를 만든 직후, **When** 본문을 비운 채 벗어나면, **Then** 카드는 보드에 남는다(빈 카드 보존).

---

### Edge Cases

- **카드/컨트롤 위 더블클릭** → 빈 곳(캔버스 pane)에서만 카드 생성. 카드 위는 그 카드 편집(기존), 상단 버튼·패널 등 컨트롤 위는 각자 동작.
- **잇기 빈 곳 drop 과의 구분** → 잇기 선을 끌어 빈 곳에 "놓는"(드래그-릴리스) 기존 "여기에 새 카드를 만들어 이을까요?" 흐름은 더블클릭과 별개 제스처라 충돌하지 않는다(무변경).
- **생성 직후 자동 편집 진입 타이밍** → 자동 편집은 카드가 서버에 확정(실제 식별자 부여)된 뒤 진입해, 진입 전 입력한 글자가 유실되지 않게 한다.
- **생성 실패** → 낙관적으로 추가된 카드를 화면에서 제거하고 실패를 알린다(기존 롤백 패턴).
- **참조 패널(좁은 폭)** → 보드 캔버스가 렌더되는 집필 참조 패널에서도 빈 보드 안내·빈 곳 더블클릭 생성이 동일하게 동작한다.
- **줌 수단** → 더블클릭 줌을 끄는 대신 마우스 휠·트랙패드 핀치·"한눈에 보기"(전체 맞춤)로 줌을 제공한다(기존에 이미 동작하던 수단).

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 카드가 0개인 보드를 열 때 **빈 격자 캔버스를 노출하지 않고** 화면 중앙에 안내(문구 + 카드 만들기 버튼)를 표시해야 한다.
- **FR-002**: 빈 보드 안내의 카드 만들기 버튼은 카드를 생성하고 **생성 직후 자동으로 본문 편집에 진입**해야 한다(추가 클릭 없이 타이핑 가능).
- **FR-003**: 보드에 카드가 1개 이상이면 시스템은 안내를 표시하지 않고 캔버스를 표시해야 한다.
- **FR-004**: 작가는 캔버스의 빈 곳(카드·컨트롤이 아닌 영역)을 더블클릭해 **그 위치에** 카드를 만들 수 있어야 하며, 생성 직후 자동으로 본문 편집에 진입해야 한다.
- **FR-005**: 상단 "+ 카드" 버튼 · 빈 곳 더블클릭 · 빈 보드 안내 버튼 **세 경로는 동일한 결과**(카드 생성 + 생성 직후 자동 편집 진입)를 내야 한다.
- **FR-006**: 카드는 생성 시점에 **즉시 저장**되며, 본문이 비어 있어도 저장·잔존해야 한다(본문 없이 벗어나도 삭제하지 않음).
- **FR-007**: 카드 생성이 실패하면 시스템은 낙관적으로 추가된 카드를 화면에서 제거하고 실패를 작가에게 드러내야 한다.
- **FR-008**: 시스템은 빈 곳 더블클릭을 카드 생성에 사용하기 위해 **캔버스 더블클릭 줌을 비활성화**해야 하며, 줌은 마우스 휠·트랙패드 핀치·"한눈에 보기"로 제공해야 한다.
- **FR-009**: 기존 동작 — 카드 더블클릭 본문 편집, 잇기(드래그/빈 곳 drop/클릭-클릭), 카드 드래그 배치, 선택 해제, 이웃 하이라이트, 종류 부여, 보드↔작품/시리즈 매핑 — 은 변경 없이 보존되어야 한다.
- **FR-010**: 본 동작은 보드 캔버스가 렌더되는 모든 화면(보드 상세 페이지 · 집필 참조 패널)에서 동일하게 적용되어야 한다.
- **FR-011**: `specs/038-memo-plot-board/spec.md` 의 FR-005("노드 0개 보드도 빈 캔버스로 정상 표시")는 본 spec(FR-001)으로 대체되며, 038 문구를 worksheet 정합으로 정정한다.

### Key Entities *(include if feature involves data)*

- **카드(Card)**: 보드 위의 한 장. `body`(단일 본문 텍스트), 종류(선택, 인물/장소/사건/테마/무지정), 캔버스 좌표(위치), 겹침 순서를 가진다. **본 기능은 카드 모델·저장 방식을 변경하지 않으며**(신규 필드·엔티티 0), 생성·편집의 **진입 경로와 시점(UX)** 만 다룬다.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 카드가 0개인 보드를 열면 **100%** 빈 격자 캔버스가 아니라 중앙 안내가 보인다.
- **SC-002**: 세 생성 경로(버튼·더블클릭·빈 보드 버튼) 모두에서, 카드 생성 후 **추가 클릭 없이** 곧바로 본문을 타이핑할 수 있다.
- **SC-003**: 빈 곳 더블클릭은 더블클릭한 **그 위치**(시각적으로 일치)에 카드를 만든다.
- **SC-004**: 본문을 비운 채 벗어나도 카드가 보존되어, 재진입 시 그대로 남아 있다(빈 카드 보존, 데이터 유실 0).
- **SC-005**: 기존 보드 동작(잇기·드래그 배치·종류 부여·매핑·카드 더블클릭 편집)의 회귀가 **0건**이다.
- **SC-006**: 빈 보드 안내·빈 곳 더블클릭 생성이 보드 상세 페이지와 집필 참조 패널 **두 곳 모두**에서 동작한다.

## Assumptions

- **카드 = `body` 단일 모델**(제목/내용 분리 없음). worksheet 의 "제목칸 자동 포커스" 는 본문(`body`) 편집 자동 진입으로 매핑한다.
- **백엔드 변경 0.** 기존 카드 생성 경로(이미 존재하는 카드 생성 엔드포인트)와 본문 편집 저장을 재사용한다. 신규 마이그레이션·에러코드·엔티티 없음.
- **빈 카드 잔존**은 사용자 결정(2026-06-27)으로, worksheet TASK-1 완료기준 "본문 없이 벗어나면 카드 안 남음" 을 대체한다.
- **자동 편집 진입은 카드 서버 확정 후** 일어난다(진입 전 키 입력 유실 방지). 생성 직후~확정 사이의 짧은 지연이 체감되면 dogfooding 에서 재검토한다.
- **집필 참조 패널도 동일 보드 캔버스**를 재사용하므로 별도 분기 없이 동일 동작이 적용된다.
- **시각·포커스 거동(더블클릭 줌 비활성, 자동 포커스)은 자동 단위 테스트로 보장되지 않는 영역**이라 dogfooding 을 게이트로 둔다(룰 14). 순수/결정 로직(좌표 변환·빈 곳 판별·자동 편집 신호 소비)은 단위 테스트로 보호한다.

## Dependencies

- 기존 보드 도메인(카드 생성·본문 편집·낙관적 반영·롤백) — 본 기능은 그 위에 진입 UX 만 얹는다.
- 보드 캔버스 라이브러리(무한 캔버스) — 빈 곳 판별·좌표 변환·더블클릭 줌 비활성을 그 API 로 구현(구현 단계 상세).
