# Quickstart & Dogfooding 검증: 보드 카드 만들기 UX 보완

**전제(authed 로컬 풀스택, 메모리 [[local-dogfooding-needs-backend]])**: DB(docker) → 백엔드(bootRun) → 프론트(`cd frontend && pnpm dev`) 3개 기동 후 로그인.

## 자동 게이트 (구현 중)

```bash
cd frontend && pnpm lint && pnpm typecheck && pnpm test && pnpm build
```

- `pnpm build` = RSC 서버/클라이언트 경계 검출(신규 `BoardEmptyGuide`가 `"use client"`인지).
- 단위 테스트: 순수 로직(빈 곳 판별·autoEdit consume·빈 보드 판정)만 — 캔버스 상호작용은 dogfooding.

## Dogfooding 체크리스트 (룰 25 — 전항 사용자 확인 후에만 통과 단정)

### US1 — 빈 보드 첫 카드
- [ ] 카드 0개 보드 진입 시 **빈 격자 캔버스가 아니라** 중앙 안내("여기에 첫 카드를 적어보세요" + "+ 카드 만들기")가 보인다 (FR-001).
- [ ] "+ 카드 만들기" 누르면 카드 생기고 **추가 클릭 없이 바로 타이핑** 가능 (FR-002·005).
- [ ] 카드 1개 생기면 안내 사라지고 캔버스(카드) 보인다 (FR-003).
- [ ] 빈 본문인 채 다른 곳 클릭 → 카드 **남아 있다**(삭제 안 됨) (FR-006).

### US2 — 빈 곳 더블클릭 생성
- [ ] 카드 있는 보드의 **빈 곳 더블클릭 → 그 위치**에 카드 생기고 바로 타이핑 (FR-004, SC-003).
- [ ] **기존 카드 위 더블클릭** → 새 카드 안 생기고 그 카드 본문 편집 열림(기존 보존) (FR-009).
- [ ] 빈 곳 더블클릭으로 **캔버스가 확대(줌)되지 않는다** (FR-008).

### 줌 수단 보존
- [ ] 마우스 휠로 줌인/줌아웃 된다.
- [ ] (트랙패드) 핀치로 줌 된다.
- [ ] "한눈에 보기"로 전체 맞춤 줌 된다.

### 두 화면 동일 적용 (FR-010, SC-006)
- [ ] 보드 상세 페이지(`/boards/[id]`)에서 위 동작.
- [ ] 집필 참조 패널(집필 화면 우측 슬라이드오버)에서 빈 보드 안내·빈 곳 더블클릭 생성 동작.

### 회귀 (FR-009, SC-005)
- [ ] 잇기(드래그/빈 곳 drop "만들어 이을까요?"/클릭-클릭) 정상.
- [ ] 카드 드래그 배치·선택 해제·이웃 하이라이트 정상.
- [ ] 종류 부여(인물/장소/사건/테마)·보드↔작품/시리즈 매핑 정상.

### 시각 (라이트/다크, 한국어)
- [ ] 빈 보드 안내가 라이트/다크 모두에서 읽힌다(보드는 colorMode=light 고정 — 안내 오버레이 대비 확인).

## 재진입 영속 (SC-004)
- [ ] 빈 카드 만든 뒤 보드 떠났다 재진입 → 빈 카드 그대로 있다.
