# Feature Specification: 출판 방식 선택 기반 에디터 레이아웃 (종이 출판 / 웹 출판) + 종이 출판 판형

**Feature Branch**: `031-publish-layout-modes`

**Created**: 2026-06-21

**Status**: Draft

**Input**: User description: brainstorming 으로 확정한 "작품 생성 시 출판 방식(종이/웹) 선택 → 페이지 분할 추가/제거" + "종이 출판 시 출판 판형(신국판 등) 도입"

## 배경 *(context)*

소설비는 작가용 글쓰기 작업공간이고, "실제 종이 한 장씩 나뉘는 진짜 페이지 분할"이 핵심 기능이다. 그러나 두 가지 한계가 있다.

1. **작업공간 크기 선택지가 ISO 종이 규격(A4/A3/A2/B4)뿐** — 실제 출판은 ISO 규격이 아니라 "판형"(신국판·국판 등)을 쓴다. 작가는 "내 책이 신국판이다"라고 생각하지 "A5다"라고 생각하지 않는다.
2. **모든 작가에게 페이지 분할을 강요** — 웹소설 작가는 판형·조판·페이지 분할이 무의미하다. 그들은 "편당 5,000자"처럼 글자수 단위로 일한다.

따라서 작가 유형에 따라 작업공간을 분기한다: 작품 생성 시 출판 방식을 고르게 하고, 그 선택에 따라 페이지 분할을 켜고(종이 출판) 끈다(웹 출판). 종이 출판을 고른 작가에게는 출판 판형 선택과 실제 출판물에 가까운 분량 근사를 제공한다.

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 작품 생성 시 출판 방식 선택 (Priority: P1)

작가가 새 작품을 만들 때, "종이 출판"과 "웹 출판" 중 하나를 **반드시 직접 선택**한다. 미리 선택된 기본값은 없으며, 고르지 않으면 작품을 만들 수 없다. 선택에 따라 집필실의 형태가 갈린다 — 종이 출판은 종이 한 장씩 나뉘는 페이지 분할 집필실, 웹 출판은 페이지 구분 없는 연속 집필실.

**Why this priority**: 이 선택이 나머지 모든 기능(판형 노출 여부, 페이지 분할 on/off, 분량 지표)의 분기점이다. 이 데이터(작품별 출판 방식)와 선택 흐름 없이는 다른 어떤 스토리도 성립하지 않는다. 단독으로도 "작가가 자기 작업 방식을 고른다"는 가치를 전달한다.

**Independent Test**: 새 작품 생성 화면에서 출판 방식을 고르지 않고 생성 시도 → 막힘. 종이/웹 각각 선택 후 생성 → 작품이 해당 방식으로 저장되고 집필실이 그 형태로 열리는지 확인.

**Acceptance Scenarios**:

1. **Given** 작가가 새 작품 생성 화면에 있고 출판 방식을 아직 고르지 않음, **When** 작품 생성을 시도, **Then** 출판 방식을 먼저 고르라는 안내와 함께 생성이 진행되지 않는다.
2. **Given** 작가가 "종이 출판"을 선택, **When** 작품을 생성하고 집필실에 진입, **Then** 종이 한 장씩 나뉘는 페이지 분할 집필실이 열린다.
3. **Given** 작가가 "웹 출판"을 선택, **When** 작품을 생성하고 집필실에 진입, **Then** 페이지 구분 없는 연속 집필실이 열리고 판형·조판 관련 UI가 보이지 않는다.
4. **Given** 본 기능 도입 이전에 만들어진 기존 작품, **When** 작가가 그 작품을 연다, **Then** 기존과 동일하게 페이지 분할(종이 출판) 형태로 열린다(작동 보존).

---

### User Story 2 - 종이 출판: 출판 판형 선택과 실측 분량 (Priority: P2)

종이 출판을 고른 작가가 작품의 **출판 판형**을 고른다. ISO 규격(A4/A3/A2/B4)에 더해 소설 단행본 판형 4종 — 신국판·국판·46판·문고판 — 이 선택지에 추가된다. 판형을 고르면 집필실의 종이가 그 판형 크기·비율로 바뀌고, 한 화면 페이지가 **실제 그 판형 책 한 페이지의 분량에 가깝게** 표시된다(예: 신국판 한 페이지 ≈ 200자 원고지 3.5매).

**Why this priority**: 종이 출판 작가에게 "실제 출판물 기준으로 글을 쓴다"는 핵심 가치를 준다. US1(출판 방식 선택) 위에서만 의미가 있으므로 P2. 단독 테스트 가능(종이 작품에서 판형을 바꿔 페이지 크기·분량 변화 확인).

**Independent Test**: 종이 출판 작품에서 판형을 신국판으로 변경 → 종이 크기/비율이 신국판으로 바뀌고, 일정 분량을 입력했을 때 페이지당 글자수가 실제 신국판 단행본 분량 범위에 드는지 확인. 판형 변경 시 텍스트 손실이 없는지 확인.

**Acceptance Scenarios**:

1. **Given** 종이 출판 작품의 집필실, **When** 작가가 판형 선택을 연다, **Then** ISO 4종과 출판 판형 4종(신국판/국판/46판/문고판)이 함께 제공된다.
2. **Given** 판형을 신국판으로 선택, **When** 본문을 한 페이지 가득 입력, **Then** 그 한 페이지의 글자수가 신국판 단행본 1페이지 통용 분량(약 200자 원고지 3.3~3.7매) 범위에 든다.
3. **Given** 작가가 판형을 바꿈, **When** 변경이 적용됨, **Then** 작성한 본문 텍스트는 그대로 유지되고 페이지 나뉨(분할 위치)만 새 판형에 맞게 다시 계산된다.
4. **Given** 판형이 출판 표준 글자 크기라 화면 글자가 작아질 수 있음, **When** 작가가 집필실을 본다, **Then** 화면 확대(zoom) 등으로 읽고 쓰기에 충분한 가독성이 유지된다.

---

### User Story 3 - 웹 출판: 연속 작업공간과 글자수 지표 (Priority: P2)

웹 출판을 고른 작가가 **페이지 구분 없는 연속 작업공간**에서 글을 쓴다. 종이·판형·페이지 분할이 전혀 없고, 분량은 **글자수**로 표시된다. 웹소설 플랫폼에 연재하듯, 한 화면에서 끊김 없이 길게 쓰는 경험을 제공한다.

**Why this priority**: 웹소설 작가라는 새 사용자층을 끌어들이는 핵심 가치. US1 위에서만 의미가 있어 P2. 단독 테스트 가능(웹 작품에서 페이지 분할이 없고 글자수가 보이는지 확인).

**Independent Test**: 웹 출판 작품 집필실에서 긴 본문을 입력 → 페이지로 나뉘지 않고 연속으로 흐르는지, 글자수가 입력에 따라 갱신되는지 확인.

**Acceptance Scenarios**:

1. **Given** 웹 출판 작품의 집필실, **When** 작가가 본문을 길게 입력, **Then** 내용이 페이지로 나뉘지 않고 하나의 연속 흐름으로 표시된다.
2. **Given** 웹 출판 집필실, **When** 작가가 분량을 확인, **Then** 페이지 수가 아니라 글자수로 분량이 표시된다.
3. **Given** 웹 출판 작품, **When** 작가가 집필실을 본다, **Then** 판형·종이 크기·원고지 같은 종이 출판 전용 UI가 전혀 나타나지 않는다.

---

### User Story 4 - 출판 방식 전환 (Priority: P3)

작가가 작품을 만든 뒤에도 작품 설정에서 출판 방식을 **종이 ↔ 웹 양방향으로 전환**할 수 있다. 웹소설로 쓰던 작품을 단행본으로 내거나, 그 반대도 가능하다. 전환해도 작성한 본문 텍스트는 사라지지 않고 보이는 형태(페이지 분할 유무)만 바뀐다.

**Why this priority**: 흔한 실제 경로(웹소설 → 단행본화)를 지원하지만, US1~US3 가 동작한 뒤에 얹는 편의 기능이라 P3.

**Independent Test**: 본문이 있는 웹 출판 작품을 종이 출판으로 전환 → 텍스트가 그대로 남은 채 페이지 분할 집필실로 바뀌는지, 다시 웹으로 되돌렸을 때도 텍스트가 보존되는지 확인.

**Acceptance Scenarios**:

1. **Given** 본문이 작성된 웹 출판 작품, **When** 작가가 설정에서 종이 출판으로 전환, **Then** 본문 텍스트는 그대로 유지되고 집필실이 페이지 분할 형태로 바뀌며 판형 선택이 노출된다.
2. **Given** 본문이 작성된 종이 출판 작품, **When** 작가가 웹 출판으로 전환, **Then** 본문 텍스트는 그대로 유지되고 페이지 분할이 사라지며 분량 지표가 글자수로 바뀐다.

---

### Edge Cases

- **빈 작품 전환**: 본문이 한 글자도 없는 작품을 전환 → 빈 상태 그대로 형태만 전환(오류 없음).
- **매우 긴 웹 작품을 종이로 전환**: 페이지 분할 재계산이 큰 분량에서도 작동하고 텍스트 손실이 없어야 한다.
- **기존 ISO 규격 작품**: 본 기능 이전에 A4/A3 등으로 저장된 작품은 종이 출판으로 보존되며 기존 판형(ISO) 값이 유지된다.
- **판형 변경 직후 페이지 수 변화**: 작은 판형(문고판)으로 바꾸면 같은 본문의 페이지 수가 늘어난다 — 이는 정상이며 텍스트 손실이 아니다.
- **웹 출판에서 분량 표시**: 글자수는 공백 포함/제외를 구분해 보여준다(둘 중 무엇을 우선 표기할지는 Assumptions 참조).

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 작가는 새 작품을 만들 때 출판 방식을 "종이 출판" 또는 "웹 출판" 중 하나로 **반드시 직접 선택**해야 한다. 시스템은 출판 방식이 선택되지 않은 작품 생성을 거부해야 한다(미리 선택된 기본값 없음).
- **FR-002**: 시스템은 작품별로 선택된 출판 방식을 영속 저장해야 한다.
- **FR-003**: "종이 출판" 작품의 집필실은 종이 한 장씩 나뉘는 페이지 분할을 제공해야 한다.
- **FR-004**: "웹 출판" 작품의 집필실은 페이지 분할 없이 연속된 하나의 흐름으로 본문을 표시해야 한다.
- **FR-005**: "종이 출판" 작품에서 작가는 판형을 선택할 수 있어야 하며, 선택지는 기존 ISO 규격 4종(A4/A3/A2/B4)과 출판 판형 4종(신국판 152×225mm / 국판 148×210mm / 46판 128×188mm / 문고판 105×148mm)을 포함해야 한다.
- **FR-006**: 판형을 선택하면 집필실 종이의 크기·비율이 그 판형의 실제 치수에 맞게 바뀌어야 한다.
- **FR-007**: 종이 출판에서 한 화면 페이지의 분량은 실제 그 판형 단행본 한 페이지 분량에 근사해야 한다. 검증 기준 앵커는 "신국판 1페이지 ≈ 200자 원고지 3.5매(약 700~800자)"이다.
- **FR-008**: 종이 출판의 분량 지표는 페이지 수와 200자 원고지 환산 매수로 표시되어야 한다.
- **FR-009**: 웹 출판의 분량 지표는 글자수(공백 포함/제외 구분)로 표시되어야 한다.
- **FR-010**: "웹 출판" 작품에서는 판형·종이 크기·원고지 등 종이 출판 전용 UI가 노출되지 않아야 한다.
- **FR-011**: 작가는 작품 생성 후에도 출판 방식을 종이 ↔ 웹 양방향으로 전환할 수 있어야 한다.
- **FR-012**: 출판 방식 전환 시 작성된 본문 텍스트는 손실 없이 보존되어야 하며, 변경되는 것은 표시 형태(페이지 분할 유무)와 분량 지표뿐이어야 한다.
- **FR-013**: 본 기능 도입 이전에 생성된 모든 작품은 "종이 출판"으로 취급되어 기존 페이지 분할 작동이 보존되어야 한다.
- **FR-014**: 종이 출판에서 판형이 출판 표준 글자 크기를 사용해 화면 글자가 작아지더라도, 작가가 읽고 쓰기에 충분한 화면 가독성(예: 화면 확대)이 유지되어야 한다.

### Key Entities *(include if feature involves data)*

- **작품(Project)**: 작가의 글쓰기 단위. 새로 **출판 방식(종이/웹)** 속성을 가진다. 기존 **판형(종이 크기)** 속성은 종이 출판일 때만 의미를 가진다. 본문 텍스트 내용은 출판 방식·판형과 독립적으로 보존된다(전환 시 무손실의 근거).
- **판형 프리셋**: 종이 출판에서 선택 가능한 종이 규격 정의. 각 판형은 실제 치수(가로×세로 mm)와 분량 근사를 위한 조판 기준(여백 등)을 가진다. 신국판을 분량 검증 앵커로 삼고, 나머지 판형은 면적비로 분량을 근사한다.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 작가가 새 작품 생성 시 출판 방식(종이/웹)을 100% 직접 선택하게 되며, 출판 방식 없이 생성된 작품은 0건이다.
- **SC-002**: 종이 출판에서 신국판 한 페이지의 본문 분량이 200자 원고지 3.3~3.7매 범위에 든다(실측 앵커 검증).
- **SC-003**: 웹 출판 작품에서 본문이 페이지로 나뉘지 않고 연속으로 표시되며, 분량이 글자수로 표시된다.
- **SC-004**: 출판 방식을 전환(종이↔웹)하거나 판형을 변경했을 때 본문 텍스트 손실이 0이다(전환 전후 텍스트 동일).
- **SC-005**: 본 기능 도입 이전에 생성된 작품 전체가 전환 작업 없이 기존과 동일한 페이지 분할 형태로 계속 열린다.
- **SC-006**: 웹소설 작가가 판형·조판을 한 번도 마주치지 않고 글쓰기를 시작할 수 있다(웹 출판 집필실에 종이 출판 전용 UI 노출 0건).

## Assumptions

- **현재 에디터 기반 재사용**: 집필실은 자체 커스텀 에디터(`CustomEditor`)를 쓰며 페이지 좌표계가 이미 실측 mm 기반이다. 따라서 종이 출판 판형 추가는 판형 치수·조판 기준 주입으로 달성하고, 웹 출판은 페이지 분할을 우회하는 연속 표시 경로를 새로 둔다. 양식화 모델(`pageLayout.ts`)은 레거시로 본 기능의 기준이 아니다.
- **텍스트 모델과 표시의 분리**: 본문 텍스트 모델은 페이지 분할·판형과 독립적이라, 출판 방식 전환·판형 변경 시 텍스트 무손실이 성립한다(FR-012의 근거).
- **조판 근사 방식**: 폰트 크기·행간은 4개 출판 판형 공통(출판 표준 ~10pt, 행간 약 1.8배)으로 두고, 판형별 크기·여백 차이로 분량 차이가 자연 발생하게 한다. 신국판 외 판형의 "페이지당 정확한 자수×행수"는 신뢰할 출처가 약해 직접 프리셋하지 않고 면적비로 근사한다.
- **글자수 표기 우선순위**: 웹 출판 분량은 공백 제외 글자수를 우선 표기하되 공백 포함 글자수도 함께 보여주는 것을 기본으로 한다(웹소설 플랫폼 관행).
- **기존 ISO 규격 유지**: 기존 ISO 4종은 제거하지 않고 출판 판형 4종과 병행한다(기존 작품·데이터 보존).
- **챕터 구조 유지**: 두 출판 방식 모두 기존 챕터(작품 1:N 본문) 구조를 그대로 사용한다.

## 비범위 (Out of Scope, v1 제외)

- 작가가 폰트 크기·여백·행간을 세밀하게 수동 조절하는 기능(v1은 판형별 표준 프리셋만 제공).
- 판형별 "정확한 자수×행수" 직접 프리셋(근거가 약해 면적비 근사로 대체).
- 내보내기(DOCX/HWPX/PDF)의 판형별 실측 반영(현재도 판형 값을 내보내기에 반영하지 않음) — 후속 과제.
- 웹 출판 작품의 전용 내보내기·연재 플랫폼 연동.

## Dependencies

- 기존 작품(Project) 데이터와 판형(종이 크기) 저장 구조를 확장한다(기존 데이터 마이그레이션 포함). 데이터 스토어 쓰기(마이그레이션 적용)는 사용자 컨펌 후 수행한다.
- 기존 자동저장·버전 토큰·충돌 처리(016) 및 챕터 구조(022)를 그대로 재사용한다.

## 구현 라운드 분해 (참고 — 상세는 plan 단계)

- **R2**: 웹 출판 연속 표시 경로 (US3) — 현재 페이지 분할 전제 코드를 우회하는 부분이 가장 불확실하므로 작은 PoC 선행 권장
- **R3**: 출판 판형 4종 + 실측 조판 + 화면 확대 (US2, 종이 출판 한정)
- **R4**: 분량 지표 (종이=페이지/원고지 매수, 웹=글자수) (US2·US3 마감)
