## User Scenarios & Testing *(mandatory)*

### User Story 1 - 홈에서 최근작 이어서 쓰기 (Priority: P1)

며칠 만에 돌아온 작가가 앱을 열면(`/`), 인사와 오늘 날짜 아래에 **가장 최근에 집필한 작품 1편**이 가장 큰 타일로 놓여 있다. 타일에는 제목, 본문의 마지막 문장(따옴표 인용), 작가가 적어둔 "다음 장면" 한 줄, 그리고 "N시간 전 저장 · N자"가 보인다(누적 총시간은 v4에서 집필 리듬 카드로 이동). [이어서 쓰기] 한 번이면 그 작품의 집필 화면으로 들어간다. 기존 작품 벽은 `/library`로 이동하고, 좌측 네비게이션(Rail)에 "홈" 항목이 생긴다.

**Why this priority**: 제품 1차 가치 = 재진입 비용 0. 홈이 "지난번 어디까지 썼지"를 즉시 복원해 주는 것이 본 기능의 양보 불가 핵심이며, 이 스토리 하나만으로 대시보드가 성립한다(MVP).

**Independent Test**: 작품 2편 이상 + 본문이 있는 계정으로 `/` 진입 → 최근 저장 작품의 제목·마지막 문장·다음 장면·메타(저장 시각/글자 수)가 보이고, 클릭 1번으로 해당 집필 화면 도달. `/library`에서 기존 작품 벽 전 기능 동작.

**Acceptance Scenarios**:

1. **Given** 본문이 있는 작품 2편(서로 다른 시점에 저장), **When** `/` 진입, **Then** 본문 저장 시각이 더 최신인 작품이 이어서 쓰기 타일에 나타나고 제목·마지막 문장·다음 장면·"N시간 전 저장 · N자"가 표시된다
2. **Given** 이어서 쓰기 타일, **When** 타일(또는 [이어서 쓰기] 버튼) 클릭, **Then** 그 작품의 집필 화면으로 이동한다
3. **Given** 로그인 직후 또는 집필 "작업 종료" 직후, **When** 홈으로 돌아옴, **Then** 대시보드가 표시된다(기존 `/` 의존 흐름이 끊기지 않는다)
4. **Given** 어떤 화면이든, **When** Rail의 "홈" 클릭, **Then** `/` 대시보드로, "작품" 클릭 시 `/library` 작품 벽으로 이동한다
5. **Given** 미인증 상태, **When** `/` 진입, **Then** 기존과 동일하게 로그인 화면으로 보내진다

### User Story 2 - 카드 집계 데이터 공급 (백엔드) (Priority: P1)

시스템이 작가의 활성 작품 전체를 카드 표시에 필요한 집계(글자 수·본문 저장 시각·누적 작업시간)와 함께 한 번에 제공한다. 화면(대시보드·작품 벽)은 이 집계 위에 마지막 문장만 본문에서 파생해 얹는다.

**Why this priority**: US1·US4·US5의 데이터 토대 — 이 집계 없이는 이어서 쓰기 타일의 메타와 최근작 선정(저장 시각 기준)이 성립하지 않는다. US1과 함께 MVP를 구성한다.

**Independent Test**: 본문·작업 세션이 있는 작품 2편 계정으로 카드 집계 조회 → 각 작품에 글자 수·본문 저장 시각·누적 작업시간(종료된 세션 합)이 정확히 동봉된다. 기존 작품 목록 조회는 변화 없다.

**Acceptance Scenarios**:

1. **Given** 본문 1,000자·종료 세션 2개(합 30분)인 작품, **When** 카드 집계 조회, **Then** 그 작품 카드에 글자 수 1,000·누적 30분·본문 저장 시각이 정확히 담긴다
2. **Given** 본문을 한 번도 쓰지 않은 새 작품, **When** 카드 집계 조회, **Then** 글자 수 0·누적 0으로 응답한다(오류 아님)
3. **Given** 진행 중(미종료) 세션이 있는 작품, **When** 카드 집계 조회, **Then** 진행 중 세션은 누적에 포함되지 않는다(기존 누적 규약과 동일)
4. **Given** 다른 사용자의 작품, **When** 카드 집계 조회, **Then** 내 작품만 반환된다
5. **Given** 미인증 요청, **When** 카드 집계 조회, **Then** 기존과 동일한 인증 오류로 거절된다

### User Story 3 - 집필 리듬 카드 (Priority: P2) — v4 그래프化

작가가 우측 사이드 카드에서 (a) 이번 주 합계와 **요일별 막대 7개**(오늘 강조, 0분은 가는 선), (b) **작품별 쌓인 시간** 가로 막대(최다 작품 강조)와 총합을 본다. 목표·등급·배지는 없다. 시스템은 기간 지정 작업시간 합계를 제공하고, 화면이 사용자 시간대 기준 요일 경계를 계산해 요일별로 묻는다. 카드 하단은 좌측 칼럼 바닥선과 정렬된다.

**Why this priority**: 완화된 원칙 4의 직접 목적 — "다시 돌아와 이어 쓰게 만드는" 연속 작업 유도 신호. 재진입 동선(P1) 위에 얹는 리듬 표시.

**Independent Test**: 이번 주 종료 세션이 있는 계정으로 `/` 진입 → 리듬 카드에 합계·요일 막대·작품별 누적 막대가 표시되고, 세션 없는 계정에서는 "기록 없음"+가는 선. 기간 합계 조회 직접 호출 시 범위 내 종료 세션 합이 정확하다.

**Acceptance Scenarios**:

1. **Given** 이번 주 요일별 종료 세션(예: 월 2시간·수 1시간 20분), **When** `/` 진입, **Then** 리듬 카드에 합계 "3시간 20분"과 요일 막대(해당 요일만 높이, 오늘 강조)가 표시된다
2. **Given** 이번 주 종료 세션 0건, **When** `/` 진입, **Then** 카드는 유지되되 합계 자리에 "기록 없음", 막대는 전부 가는 선이다
2-1. **Given** 누적 시간이 있는 작품 2편, **When** `/` 진입, **Then** 작품별 가로 막대가 누적 내림차순으로 총합과 함께 표시된다(누적 0 작품은 행 생략)
3. **Given** 기간 `from`~`to`와 그 경계 밖 세션들, **When** 기간 합계 조회, **Then** 시작 시각이 범위 안인 종료 세션만 합산된다(주 경계에 걸친 세션은 시작 시각 기준 귀속)
4. **Given** `from`이 `to`보다 늦은 잘못된 요청, **When** 기간 합계 조회, **Then** 검증 오류로 거절된다

### User Story 4 - 다른 작품 빠른 진입과 새 작품 시작 (Priority: P2)

작가가 최근작 말고 다른 작품을 이어 쓰고 싶을 때, 좌측 칼럼의 미니 카드 2열(제목 + 마지막 문장 2줄)로 보고 한 번에 그 작품의 집필 화면으로 들어간다. 섹션 헤더의 "모든 작품 보기 →"로 작품 벽에 간다(v4 — "+ 새 작품" 카드는 제거, 새 작품은 벽에서).

**Why this priority**: 다작 작가의 재진입 경로 단축. P1 없이도 `/library`로 우회 가능하므로 P2.

**Independent Test**: 작품 3편 계정으로 `/` 진입 → 최근작 제외 2편이 미니 카드 2열로 보이고, 카드 클릭 → 해당 집필 화면, "모든 작품 보기" → `/library` 도달.

**Acceptance Scenarios**:

1. **Given** 작품 3편, **When** `/` 진입, **Then** 이어서 쓰기 대상을 제외한 2편이 본문 저장 시각 내림차순 미니 카드로 표시된다(제목 + 마지막 문장 최대 2줄)
2. **Given** 미니 카드, **When** 클릭, **Then** 그 작품의 집필 화면으로 이동한다
3. **Given** "모든 작품 보기 →" 링크, **When** 클릭, **Then** 작품 벽(`/library`)에 도달한다

---

### User Story 5 - 최근 곁쪽지 다시 만나기 (Priority: P3)

작가가 홈 하단 전폭 줄에서 최근 곁쪽지 **3장**(쪽지 톤, 본문 4줄 클램프 + 상대 날짜)과 **"+ 새 곁쪽지"** 점선 카드를 만난다. 카드는 메모 화면으로, 새 곁쪽지는 빠른 메모 입력(잉크 한 방울과 동일 모달)을 연다. 섹션 헤더에 "모든 곁쪽지 보기 →".

**Why this priority**: "죽은 메모" 통증 완화 — 재진입 시점의 맥락 보강. 핵심 동선 이후.

**Independent Test**: 곁쪽지 4장 계정으로 `/` 진입 → 최신 3장 + "+ 새 곁쪽지"가 보이고, 곁쪽지 클릭 → 메모 화면, 새 곁쪽지 클릭 → 빠른 메모 모달.

**Acceptance Scenarios**:

1. **Given** 곁쪽지 4장(서로 다른 캡처 시점), **When** `/` 진입, **Then** 캡처 시각 최신 3장이 본문과 상대 날짜로 표시되고 "+ 새 곁쪽지" 카드가 함께 보인다
2-1. **Given** "+ 새 곁쪽지" 카드, **When** 클릭, **Then** 빠른 메모 입력 모달이 열린다
2. **Given** 곁쪽지 카드, **When** 클릭, **Then** 메모 화면(`/memos`)으로 이동한다

---

### User Story 6 - 작품 벽 마지막 문장 회복 (Priority: P2)

작가가 작품 벽(`/library`)을 열면 각 작품 카드의 얼굴에 실제 본문의 마지막 문장이 보인다. (현재는 데이터 연결 격차로 본문이 있어도 전부 "아직 첫 문장을 기다리는 중"으로 표시되는 상태 — 본 작업의 데이터 경로 정비가 이 격차를 함께 해소한다.)

**Why this priority**: 015 설계가 의도한 기존 화면의 표시 회복. 대시보드와 같은 데이터 경로를 쓰므로 본 작업에서 함께 해결하는 것이 가장 저렴하다.

**Independent Test**: 본문이 있는 작품으로 `/library` 진입 → 카드 얼굴에 실제 마지막 문장 표시(placeholder 문구 아님).

**Acceptance Scenarios**:

1. **Given** 본문이 있는 작품, **When** `/library` 진입, **Then** 그 작품 카드에 본문에서 파생한 마지막 문장이 표시된다
2. **Given** 본문이 빈 작품, **When** `/library` 진입, **Then** 기존과 동일하게 "아직 첫 문장을 기다리는 중"이 표시된다

---

### Edge Cases

- **작품 0편**: 이어서 쓰기·이번 주·작품 줄 대신 환영 블록 + "첫 작품 시작하기" → 새 작품 작성 흐름.
- **작품 1편뿐**: 좌측 미니 카드 영역을 생략한다(타일만) — 우측 리듬 카드는 유지(v4).
- **본문이 빈 최근작**: 마지막 문장 자리에 "아직 첫 문장을 기다리는 중"(작품 벽과 동일 카피).
- **다음 장면 미입력(빈 문자열)**: 그 줄을 숨긴다.
- **이번 주 0분**: 리듬 카드 합계 자리 "기록 없음" + 막대 전부 가는 선. **누적 전부 0**: 작품별 절 생략(v4 — 압박 없는 처리).
- **곁쪽지 0장**: 라벨 유지 + "아직 곁쪽지가 없어요" 한 줄.
- **본문 저장 시각 동률(드묾)**: 2차 기준으로 결정적 정렬 — 새로고침 간 일관.
- **카드·주간 집계 중 일부 조회 실패**: 작품 벽과 동일한 에러 상태(안내 + 재시도) — 반쪽 데이터로 그리지 않는다.
- **주 경계(월요일 0시) 직후 진입**: 이번 주 합 0 → "기록 없음"(정상). 경계에 걸친 세션은 시작 시각 기준 한 요일/주에만 귀속(이중 계산 없음).
- **로딩 중**: 작품 벽과 동일한 스켈레톤 패턴.

## Requirements *(mandatory)*

### Functional Requirements

**대시보드 화면**

- **FR-001**: `/` 진입 시 작가 홈(대시보드)이 표시되어야 한다 — 인사 한 줄(이름 없음)과 오늘 날짜("YYYY년 M월 D일 X요일", 한국어).
- **FR-002**: 대시보드는 본문 저장 시각이 가장 최신인 작품 1편을 "이어서 쓰기" 최대 타일로 표시해야 한다 — 제목, 본문 마지막 문장(인용 톤), "다음 장면" 한 줄, 마지막 저장 상대 시각, 글자 수, 누적 작업시간.
- **FR-003**: 이어서 쓰기 타일과 [이어서 쓰기] 버튼은 해당 작품의 집필 화면으로 이동해야 한다.
- **FR-004**: 대시보드 우측에 "집필 리듬" 카드를 표시해야 한다 — (a) 이번 주 합계 + 요일별 막대 7개(사용자 시간대 기준 월요일 시작, 오늘 강조, 0분=가는 선, 합 0이면 "기록 없음") (b) 작품별 누적 가로 막대(내림차순, 최다 강조, 0 작품 행 생략, 전부 0이면 절 생략) + 총합. 목표·등급·배지 금지. 카드 하단은 좌측 칼럼 바닥선과 정렬(v4).
- **FR-005**: 대시보드 좌측 칼럼에 최근작 제외 나머지 작품을 본문 저장 시각 내림차순 미니 카드 **2열**로 표시하고, 클릭 시 해당 집필 화면으로 이동해야 한다(v4).
- **FR-006**: "이어서 쓰기" 섹션 헤더의 "모든 작품 보기 →"는 작품 벽(`/library`)으로 이동해야 한다. 새 작품 시작은 작품 벽과 환영 블록 CTA(`/library?new=1`)가 담당한다(v4 — 대시보드 내 "+ 새 작품" 카드 제거).
- **FR-007**: 대시보드 하단 전폭에 캡처 시각 최신 곁쪽지 **3장**(본문 4줄 클램프 + 상대 날짜) + "+ 새 곁쪽지" 점선 카드(빠른 메모 모달)를 표시하고, 곁쪽지 클릭과 "모든 곁쪽지 보기 →"는 메모 화면으로 이동해야 한다(v4).
- **FR-008**: 대시보드는 읽기 전용이어야 한다 — 모든 동작은 다른 화면으로의 진입뿐이다.
- **FR-009**: 대시보드에 평가·압박 장치(연속 기록 streak, 목표 달성 게이지, 등급/배지 라벨, 자동 인용구)를 표시하지 않아야 한다. 시간 막대 그래프는 평가 없는 사실 표시로 허용한다(v4, PRODUCT.md 원칙 4 v2).

**데이터 공급 (백엔드)**

- **FR-010**: 시스템은 인증 사용자의 활성 작품 전체를 카드 집계와 함께 제공해야 한다 — 작품 기본 정보(제목·다음 장면 등)에 글자 수·본문 저장 시각·누적 작업시간(종료된 세션 합, 진행 중 제외)을 동봉. 본문이 없거나 세션이 없는 작품은 0으로 응답한다.
- **FR-011**: 시스템은 기간(`from`~`to`)을 지정한 작업시간 합계를 제공해야 한다 — 인증 사용자의 전체 작품 횡단, 시작 시각이 범위 안인 종료된 세션의 합. `from`이 `to`보다 늦으면 검증 오류로 거절한다.
- **FR-012**: 카드 집계·기간 합계는 요청 사용자의 데이터만 반환해야 하며, 미인증 요청은 기존과 동일한 인증 오류로 거절한다.
- **FR-013**: 기존 작품 목록 조회(페이지네이션 목록)는 변경되지 않아야 한다 — 기존 소비처(레거시 화면·기록 화면) 동작 보존.
- **FR-014**: 마지막 문장은 본문에서 파생되어 대시보드와 작품 벽 모두에 표시되어야 한다(현 placeholder 격차 해소 — 본문이 있는데 빈 문구가 나오는 경우 0건). 파생은 화면 측 책임이다(백엔드는 본문 텍스트 파생을 제공하지 않는다 — 사용자 결정).

**라우팅·네비게이션**

- **FR-015**: 기존 작품 벽은 `/library`에서 기존 기능 그대로(목록·새 작품·다음 장면 인라인 수정·삭제) 동작해야 한다.
- **FR-016**: 좌측 네비게이션(Rail)에 "홈" 항목이 신설되고(`/`), "작품" 항목은 `/library`를 가리켜야 한다. 메모·기록·문의·빠른 메모 캡처는 불변.
- **FR-017**: 기존 `/` 의존 동선은 끊기지 않아야 한다 — 로그인 직후, 집필 "작업 종료" 직후, 미인증 가드 리다이렉트는 모두 대시보드로 자연 연결되고, 집필 화면 오류 시 "작품 벽으로" 동선은 작품 벽(`/library`)으로 간다.
- **FR-018**: 대시보드는 인증된 사용자 전용이어야 한다(기존 인증 가드와 동일 동작).

**품질**

- **FR-019**: Edge Cases 절의 빈 상태·로딩·에러 동작이 각각 정의된 대로 표시되어야 한다.
- **FR-020**: 대시보드는 기존 작업실 외관(웜 톤·종이 질감·세리프 인용)을 계승하고 라이트/다크 모두 지원해야 한다.
- **FR-021**: 접근성 — 본문·보조 텍스트 대비 AA(≥4.5:1), 모든 진입 동작은 키보드로 도달·실행 가능, 모션 줄이기 설정 시 등장 애니메이션 제거.

### Key Entities

- **작품 카드**: 작품의 홈/벽 표시 단위 — 제목, 작가가 적은 "다음 장면", 글자 수, 본문 저장 시각, 누적 작업시간(이상 시스템 집계 제공) + 본문에서 파생한 마지막 문장(화면 파생). 누적 작업시간 = 종료된 세션 합(진행 중·폐기 제외).
- **기간 작업시간 합계**: 사용자의 전체 작품 횡단, 지정 기간(`from`~`to`) 안에 시작되어 종료된 세션들의 시간 합. "이번 주" 범위는 화면이 사용자 시간대 기준으로 계산해 지정한다.
- **곁쪽지**: 작가가 캡처한 메모 — 본문, 캡처 시각(상대 날짜로 표시), 연결 작품(있을 수 있음).

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 작가가 홈 진입 후 **클릭 1번**으로 최근작 집필 화면에 도달한다.
- **SC-002**: 홈 첫 화면(데스크탑 기준, 스크롤 없이)에서 최근작의 재진입 맥락 — 제목·마지막 문장·다음 장면 — 을 모두 확인할 수 있다.
- **SC-003**: 본문이 있는 작품에서 빈 placeholder 마지막 문장이 표시되는 경우가 대시보드·작품 벽 통틀어 **0건**이다.
- **SC-004**: 이번 주에 종료된 세션이 있으면 홈에서 이번 주 집필 시간이 정확히(분 단위) 표시되고, 없으면 표시 자체가 없다.
- **SC-005**: 카드 집계·기간 합계의 수치가 기존 화면(기록 화면의 누적·글자 수)과 동일 데이터 기준으로 일치한다.
- **SC-006**: 기존 화면(작품 벽·집필·메모·기록·로그인 흐름)과 기존 백엔드 계약의 자동화 테스트가 전부 통과한다(회귀 0).
- **SC-007**: 라이트/다크 × 모션 줄이기 설정에서 대비 AA와 키보드 진입이 검증된다.
- **SC-008**: 작품 0편·본문 0·곁쪽지 0·이번 주 0분의 빈 상태에서 깨진 영역 없이 정의된 동작이 표시된다.

## Assumptions

- **백엔드 확장 포함**(v3, 사용자 결정): 신규 제공 2종 — (a) 작품 카드 집계 (b) 기간 작업시간 합계. 둘 다 읽기 전용이며 기존 데이터(작품·문서·작업 세션)의 집계라 스키마 변경 없음. 기존 조회 계약은 불변.
- **마지막 문장은 화면 파생 유지**(사용자 결정): 본문 텍스트의 백엔드 파생(파싱/비정규화)은 베타 작품 수 대비 과투자로 기각. 화면이 작품별 본문을 추가 조회해 파생한다(베타 작품 소수 전제).
- **"이번 주" 정의**: 사용자 시간대 기준 월요일 0시 시작. 범위 계산은 화면 책임, 시스템은 기간 합계만 제공(시간대 무지).
- **인사말에 이름 없음**: 사용자 이름 데이터가 시스템에 없다(검증 완료). 이메일 일부 표시는 제품 정서에 어긋나 배제.
- **최근작 판단 기준은 본문 저장 시각**: 작품 메타 수정은 "최근에 집필함"으로 치지 않는다. 로컬 기억값(마지막 연 작품)은 기기 종속이라 쓰지 않는다.
- **게이미피케이션 시각화는 여전히 범위 밖**: streak·목표 게이지·주간 그래프는 원칙 4 완화 후에도 배제(압박 장치).
- **기존 무관 부채 불변**: 본 작업과 무관한 기존 실패(문서 테스트 타입 오류 1건, 집필 화면 lint 경고 1건)는 건드리지 않는다.
