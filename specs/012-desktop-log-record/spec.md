# Feature Specification: Desktop 기록(Log) — 작품별 진척·작업 시간·기록 메모

**Feature Branch**: `012-desktop-log-record`

**Created**: 2026-06-08

**Status**: Draft

**Input**: 설계 brief `docs/superpowers/specs/2026-06-08-desktop-log-record-design.ko.md` (2026-06-08 브레인스토밍 확정)

> 4개 화면(작품/집필/메모/**기록**) 중 비어 있던 기록 화면을 실데이터로 채운다. 작가가 재진입할 때 "어디까지 했지"를 다시 떠올리지 않도록, 작품별 진척 상태·작업 흔적을 한눈에 보여준다.

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 작품별 진척 상태를 한눈에 본다 (Priority: P1)

작가가 기록 화면을 열면, 자신의 모든 작품이 카드로 나열되고 각 카드에서 **목표 대비 진척률·가장 최근 수정 시점·본문의 마지막 문장**을 바로 확인한다. 오랜만에 돌아왔을 때 "이 작품 어디까지 썼더라"를 다시 떠올릴 필요 없이, 카드만 보고 재진입 지점을 잡는다.

**Why this priority**: 본 기능의 핵심 가치(재진입 컨텍스트 영속)를 가장 직접적으로 전달한다. 신규 데이터 적재 없이 기존에 저장된 작품·본문 정보만으로 즉시 가치를 주는, 가장 작은 dogfoodable 슬라이스다.

**Independent Test**: 목표 글자수가 설정된 작품과 미설정 작품을 각각 만들고 본문을 일부 작성한 뒤 기록 화면을 열어, 진척률(또는 "목표 미설정")·최근 수정일·마지막 문장이 정확히 표시되는지로 단독 검증 가능.

**Acceptance Scenarios**:

1. **Given** 목표 글자수 12,000자에 현재 본문이 7,440자인 작품, **When** 기록 화면을 연다, **Then** 그 작품 카드에 진척 62%와 목표 글자수가 표시된다.
2. **Given** 목표 글자수가 설정되지 않은 작품, **When** 기록 화면을 연다, **Then** 진척률 대신 "목표 미설정"이 표시된다.
3. **Given** 본문이 작성된 작품, **When** 기록 화면을 연다, **Then** 본문의 마지막 문장과 가장 최근 수정 날짜가 표시된다.
4. **Given** 작품이 하나도 없는 상태, **When** 기록 화면을 연다, **Then** 빈 상태 안내가 표시된다.

---

### User Story 2 - 작업을 마칠 때 기록 메모를 남기고 누적 기록을 본다 (Priority: P2)

작가가 집필을 마칠 때 **"작업 종료"** 동작에서 그날의 작업을 한 줄로 남긴다(예: "3장 도입부 톤 고민"). 이 기록은 빠른 메모(아이디어 캡처)와는 별개의 **작업 일지**로, 작품마다 쌓인다. 기록 화면 카드에는 가장 최근 기록 한 줄이 보이고, 펼치면 그 작품의 누적 기록 전체를 시간순으로 본다.

**Why this priority**: 재진입 시 "무슨 생각을 하다 멈췄는지"를 작가 자신의 언어로 남기는 가치. P1의 자동 파생값(마지막 문장)을 작가의 의도적 기록으로 보완한다. 신규 기록 저장과 누적 조회가 필요해 P1보다 뒤.

**Independent Test**: 집필을 마치며 기록 메모를 남기고, 같은 작품에 두 번째 기록을 더 남긴 뒤 기록 화면에서 최신 한 줄 표시와 펼침 시 누적 2건이 시간순으로 보이는지로 단독 검증 가능.

**Acceptance Scenarios**:

1. **Given** 집필 중인 작품, **When** "작업 종료"로 기록 메모 "3장 도입부 톤 고민"을 남기고 저장한다, **Then** 기록 화면 그 작품 카드에 최신 기록으로 "3장 도입부 톤 고민"이 표시된다.
2. **Given** 같은 작품에 기록이 2건 이상 쌓인 상태, **When** 카드의 펼침을 연다, **Then** 누적 기록 전체가 최신순으로 표시된다.
3. **Given** 기록을 한 번도 남기지 않은 작품, **When** 기록 화면을 연다, **Then** 기록 영역은 비어 있음으로 표시된다.
4. **Given** 기록 메모 입력 없이 작업 종료를 취소한다, **When** 모달을 닫는다, **Then** 기록은 추가되지 않고 집필 상태가 유지된다.

---

### User Story 3 - 작업 시간이 자동으로 추적되어 총 작업 시간을 본다 (Priority: P3)

작가가 작품의 집필 화면에 들어가면 작업이 시작된 것으로 보고, 화면을 벗어나거나 다른 작품으로 옮기거나 앱을 닫으면 작업이 끝난 것으로 본다. 이렇게 모인 시간이 작품마다 합산되어, 기록 화면 카드에 **총 작업 시간**으로 표시된다. 작가는 "이 작품에 내가 얼마나 시간을 들였나"를 별도 조작 없이 확인한다.

**Why this priority**: 들인 노력을 가시화하는 부가 가치. 자동 추적 생명주기(시작/종료 판정·예외 처리)가 필요해 구현 비용이 가장 크므로 P3. P1·P2 없이도 독립적으로 작동.

**Independent Test**: 한 작품 집필 화면에 들어가 잠시 머문 뒤 다른 화면으로 이동하고, 다시 들어갔다 나오는 동작을 반복한 후 기록 화면에서 총 작업 시간이 누적되는지로 단독 검증 가능.

**Acceptance Scenarios**:

1. **Given** 작품 집필 화면에 5분간 머문 뒤 다른 화면으로 이동, **When** 기록 화면을 연다, **Then** 그 작품의 총 작업 시간에 약 5분이 더해져 표시된다.
2. **Given** 집필 화면에 들어갔다 30초 미만 만에 나온 경우, **When** 기록 화면을 연다, **Then** 그 짧은 진입은 총 작업 시간에 합산되지 않는다.
3. **Given** 집필 도중 앱을 그대로 닫은 경우, **When** 앱을 다시 열고 기록 화면을 본다, **Then** 닫기 전까지의 시간이 합산되거나(정상 종료 감지) 비정상 종료로 판정되어 과대 합산 없이 처리된다.
4. **Given** 작업 시간 기록이 없는 작품, **When** 기록 화면을 연다, **Then** 총 작업 시간은 "기록 없음"(또는 0)으로 표시된다.

---

### Edge Cases

- **목표 초과**: 현재 글자수가 목표를 넘으면 진척률은 100%를 초과한 실제 수치로 표시한다(예: 112%). 작가가 목표를 넘겼음을 인지하게 한다.
- **빈 본문**: 본문이 비어 있으면 마지막 문장은 표시하지 않는다(빈 상태 신호).
- **작품 삭제**: 작품을 삭제하면 그 작품의 기록 메모·작업 시간 기록도 함께 사라진다(고아 데이터 없음).
- **미종료 작업**: "작업 종료"를 누르지 않고 화면을 벗어나거나 앱을 닫아도, 작업 시간은 그 이탈 시점으로 자동 종료된다.
- **비정상 종료**: 앱이 강제 종료되어 종료 시점을 알 수 없는 작업은 다음 실행 시 폐기되어, 자리를 비운 채 방치된 시간이 가짜로 합산되지 않는다.
- **짧은 진입**: 30초 미만의 진입은 작업으로 보지 않고 폐기한다. 단, 작가가 그 짧은 진입에서 의도적으로 기록 메모를 남긴 경우는 보존한다.

## Requirements *(mandatory)*

### Functional Requirements

**기록 화면 표시 (US1)**

- **FR-001**: 기록 화면은 작가의 모든 작품을 카드 목록으로 표시해야 한다(읽기 전용).
- **FR-002**: 각 작품 카드는 목표 글자수 대비 현재 글자수의 진척률(%)을 표시해야 한다. 현재 글자수는 본문의 공백 제외 문자 수로 한다.
- **FR-003**: 목표 글자수가 설정되지 않은 작품은 진척률 대신 "목표 미설정"을 표시해야 한다.
- **FR-004**: 각 작품 카드는 가장 최근 수정 날짜를 표시해야 한다.
- **FR-005**: 각 작품 카드는 본문의 마지막 문장을 간추려 표시해야 한다. 본문이 비어 있으면 표시하지 않는다.
- **FR-006**: 작품이 하나도 없으면 빈 상태 안내를 표시해야 한다.

**기록 메모 (US2)**

- **FR-007**: 작가는 집필을 마칠 때 "작업 종료" 동작에서 작품에 기록 메모(한 줄~짧은 글)를 남길 수 있어야 한다.
- **FR-008**: 기록 메모는 빠른 메모(아이디어 캡처) 기능과 별개로, 작품별 작업 일지로 누적 저장되어야 한다.
- **FR-009**: 각 작품 카드는 가장 최근 기록 메모 한 줄을 표시하고, 펼침을 통해 그 작품의 누적 기록 전체를 최신순으로 보여줘야 한다.
- **FR-010**: 기록 메모를 남기지 않고 작업 종료를 취소하면 기록은 추가되지 않아야 한다.
- **FR-011**: 작업 종료 시 기록 메모 저장과 작업 시간 종료는 함께 처리되어, 하나만 저장되고 다른 하나가 누락되는 일이 없어야 한다.

**작업 시간 추적 (US3)**

- **FR-012**: 작가가 작품의 집필 화면에 진입하면 그 작품의 작업이 시작된 것으로 기록해야 한다.
- **FR-013**: 작가가 집필 화면을 벗어나거나(다른 화면 이동), 다른 작품으로 전환하거나, 앱을 닫으면 진행 중이던 작업이 자동으로 종료되어야 한다.
- **FR-014**: 각 작품 카드는 그 작품에 들인 총 작업 시간을 표시해야 한다(종료된 작업의 합).
- **FR-015**: 30초 미만의 작업은 합산에서 제외해야 한다. 단, 그 짧은 작업에서 기록 메모를 남긴 경우는 보존한다.
- **FR-016**: 앱 비정상 종료로 종료 시점이 불명확한 작업은 다음 실행 시 폐기되어, 총 작업 시간이 과대 합산되지 않아야 한다.

**공통**

- **FR-017**: 작품을 삭제하면 그 작품의 기록 메모와 작업 시간 기록도 함께 삭제되어야 한다.
- **FR-018**: 기록 메모·작업 시간은 앱을 껐다 켜도 유지되어야 한다(로컬 영속).

### Key Entities *(include if feature involves data)*

- **기록 메모(작업 일지 항목)**: 한 작품에 속하며, 작가가 작업을 마칠 때 남긴 본문과 작성 시각을 가진다. 한 작품에 여러 개가 시간순으로 누적된다.
- **작업 세션**: 한 작품에 속하며, 작업 시작 시각과 종료 시각을 가진다. 종료 시각이 없으면 진행 중을 뜻한다. 작품별 합산이 총 작업 시간이 된다.
- **작품 / 본문(기존)**: 진척률(목표 글자수·현재 글자수)·최근 수정 날짜·마지막 문장의 출처. 본 기능은 기존 작품·본문 데이터를 읽어 파생 표시한다.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 작가는 기록 화면을 열어, 별도 클릭 없이 각 작품의 진척률·최근 수정일·마지막 문장을 카드에서 즉시 확인할 수 있다.
- **SC-002**: 진척률은 목표 글자수 대비 현재 글자수를 정확히 반영한다(예: 목표 12,000자·현재 7,440자 → 62%). 목표 미설정 작품은 진척률 자리에 "목표 미설정"이 나타난다.
- **SC-003**: 작가가 작업 종료에서 남긴 기록 메모는 그 즉시 기록 화면 해당 작품 카드의 최신 기록으로 반영되고, 펼침에서 누적 기록 전체가 최신순으로 보인다.
- **SC-004**: 집필 화면에 머문 실제 시간이 총 작업 시간에 누적되며, 30초 미만 진입과 비정상 종료로 인한 가짜 시간은 합산되지 않는다.
- **SC-005**: 작품을 삭제한 뒤 기록 화면을 다시 열면, 그 작품과 관련된 기록 메모·작업 시간이 남아 있지 않다.

## Assumptions

- 한 작품은 하나의 본문을 가진다(현재 데이터 구조 전제). 따라서 현재 글자수는 그 단일 본문의 글자 수로 한다.
- 진척률 100% 초과는 캡 없이 실제 수치로 표시한다(작가가 목표 초과를 인지하도록).
- 기록 메모는 본 기능 범위에서 추가·조회만 한다(편집·삭제는 후속 검토).
- "작업 시작"은 집필 화면 진입으로 자동 판정한다(별도 시작 버튼 없음). 작가가 명시적으로 끝맺는 경로는 "작업 종료" 버튼이며, 그 외 이탈은 자동 종료로 처리한다.
- 시간축 통계(이번 주/이번 달 글자수, 세션 횟수 그래프)와 본문 버전 추적(직전 대비 추가분), 무입력 idle 시간 캡은 본 기능 범위 밖이다.
- 로컬 우선 데스크탑 앱(단일 사용자, 단일 기기) 전제 — 동기화·다중 사용자는 범위 밖.
