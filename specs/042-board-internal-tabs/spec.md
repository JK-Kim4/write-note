## 1. 문제 / 의도
전역 보드 허브(041, `/boards`)는 "내 모든 보드 찾기"용이다. 그러나 작가가 **특정 작품/시리즈를 보는 맥락 안에서** "이 작품(시리즈)에 매달린 보드"만 빠르게 보고 만들 진입점이 없다. PRD §5.4 의 두 번째 진입점("작품/시리즈 내부 보드 탭 — 그 주체 보드만, 생성 시 owner 자동").

별도 작품/시리즈 상세 페이지가 없으므로(실측), 호스트는:
- **작품 상세 = 집필 화면**(`/works/[id]`) → 우측 보조 패널에 "보드" 탭.
- **시리즈 상세 = 작품 보관함 드릴인**(`/library?folder=`) → "시리즈 보드" 섹션.

## 2. 사용자 시나리오
1. 작가가 집필 화면 우측 패널에서 **보드 탭**을 연다 → 이 작품에 매달린 보드 목록이 보인다. "+ 새 보드"로 이름만 적어 만들면 이 작품 소속으로 생성되고 열린다.
2. 작가가 작품 보관함에서 시리즈를 열면(드릴인) **시리즈 보드 섹션**에 그 시리즈 보드가 보인다. "+ 새 보드"로 이 시리즈 소속 보드를 만든다.
3. 목록의 보드를 누르면 `/boards/{id}` 캔버스로 이동(전역/내부/집필참조 모두 같은 보드 실체).

## 3. 요구사항 (FR)
- **FR-1** 작품 집필 화면 보조 패널에 "보드" 탭이 있고, `ownerType=project, ownerId={projectId}` 보드만 최근순으로 보인다.
- **FR-2** 시리즈 드릴인에 "시리즈 보드" 섹션이 있고, `ownerType=category, ownerId={categoryId}` 보드만 보인다.
- **FR-3** 두 호스트 모두 "+ 새 보드"는 owner 를 자동 결정하고 **이름만** 입력받아 생성한다(전역 picker 없음).
- **FR-4** 생성 성공 시 그 보드(`/boards/{id}`)로 이동한다.
- **FR-5** 목록의 보드 클릭 시 `/boards/{id}` 로 이동한다.
- **FR-6** 보드가 없으면 빈 캔버스 노출 금지 — 안내 + 생성 진입만 보인다.
- **FR-7** 로딩/에러 상태를 표시한다(기존 보드 목록 패턴 정합).

## 4. 비범위
- 내부 탭에서의 이름변경/삭제/소속변경 — 전역 허브(`/boards`)가 담당(중복 회피). 내부 탭은 **목록·생성·열기**만.
- 집필 중 인라인 참조(읽기 분할 뷰) = 043.
