# Feature Specification: 플롯 보드 연결(Link) UI — 트랙 A

**Feature Branch**: `038-memo-plot-board` (트랙 A 연장 — 새 브랜치 미생성, 사용자 지시)

**Created**: 2026-06-25

**Status**: Draft

**Input**: 038 플롯 보드의 보류된 연결(Link) UI 재개. 백엔드 엣지 계약·FE API·React Query 훅·타입은 보존돼 있고, 캔버스 화면 결선만 dogfooding 중 제거됐다. brainstorming(2026-06-25) 확정: **무방향 선** + **이웃 하이라이트**(별도 패널 없음). 기준 문서 = `docs/board/board-ux-worksheet.md` TASK-2.

> **맥락**: 보드는 작가가 카드(인물·사건·설정 등)를 캔버스에 펼쳐 플롯을 설계하는 공간이다. 카드 만들기·편집·드래그 배치·뷰포트 복원은 이미 동작한다. 이번 트랙은 **카드를 서로 잇는 것**을 더한다. 원칙: "책상에 카드를 툭 던져놓는 느낌"이어야지 "DB를 설정하는 느낌"이면 안 된다. 첫 사용자는 매뉴얼을 안 읽고 화면 단서로 움직인다.

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 카드 잇기 (Priority: P1)

작가가 보드에서 두 카드를 이어 관계를 표현한다. 카드에 마우스를 올리면 가장자리에 연결점이 나타나고, 그 점을 끌어 다른 카드 위에 놓으면 두 카드가 선으로 이어진다. 끌다가 빈 곳에 놓으면 "여기에 새 카드를 만들어 이을까요?"를 물어, 승낙하면 새 카드를 그 자리에 만들고 함께 잇는다. 드래그가 어려우면 카드를 선택하고 "잇기"를 누른 뒤 대상 카드를 클릭하는 길도 있다.

**Why this priority**: 연결은 보드를 "낱장 카드 더미"에서 "이야기 구조"로 바꾸는 핵심 가치다. 이것 하나만으로도 보드가 플롯 설계 도구로 작동하기 시작한다. 트랙 A 전체가 이 스토리를 중심으로 존재한다.

**Independent Test**: 카드 2장이 있는 보드에서, 연결점 드래그 한 번으로 둘이 선으로 이어지고 보드를 다시 열어도 그 선이 남아 있으면 검증 완료.

**Acceptance Scenarios**:

1. **Given** 카드 2장이 있는 보드, **When** 한 카드에 hover해 나타난 연결점을 끌어 다른 카드 위에 놓는다, **Then** 두 카드가 선으로 이어지고 그 연결이 저장된다.
2. **Given** 드래그로 연결 중, **When** 커서가 놓을 수 있는 유효한 카드 위에 있다, **Then** 그 카드가 강조(초록)되고 연결선이 커서를 따라온다.
3. **Given** 연결점을 끌고 있는 상태, **When** 카드가 없는 빈 곳에 놓는다, **Then** "여기에 새 카드를 만들어 이을까요?" 확인이 뜨고, 승낙하면 그 자리에 새 카드가 생기며 출발 카드와 이어진다.
4. **Given** 카드 1장을 선택한 상태, **When** "잇기"를 누르고 안내("이을 카드를 선택하세요")를 따라 다른 카드를 클릭한다, **Then** 두 카드가 이어진다.
5. **Given** "잇기" 모드, **When** ESC를 누르거나 빈 곳을 클릭한다, **Then** 연결 시도가 취소되고 아무 연결도 생기지 않는다.
6. **Given** 이미 이어진 카드 A·B, **When** A에서 B로(또는 B에서 A로) 다시 이으려 한다, **Then** 새 연결이 생기지 않는다(중복 없음, 방향 무관).
7. **Given** 한 카드, **When** 그 카드의 연결점을 끌어 자기 자신에 놓는다, **Then** 아무 연결도 생기지 않는다.
8. **Given** 평상시 보드, **When** 어떤 카드에도 hover/선택하지 않았다, **Then** 연결점은 보이지 않는다(항상 떠 있지 않다).

---

### User Story 2 - 연결 끊기 (Priority: P2)

작가가 잘못 이었거나 더 이상 유효하지 않은 연결을 끊는다. 연결선에 마우스를 올리면 끊는 수단이 나타나고, 한 동작으로 연결이 사라진다.

**Why this priority**: 잇기를 쓰면 반드시 "잘못 이었다"가 생긴다. 끊기가 없으면 사용자는 실수를 되돌리지 못해 잇기 자체를 주저한다(심리적 안전장치). 단 잇기(P1) 없이는 끊을 대상이 없으므로 P2.

**Independent Test**: 이어진 두 카드 사이 선에 hover해 나타난 "연결 끊기"를 누르면 선이 사라지고 재진입 후에도 사라진 상태가 유지되면 검증 완료.

**Acceptance Scenarios**:

1. **Given** 이어진 카드 두 장, **When** 그 사이 연결선에 hover한다, **Then** "연결 끊기" 수단(✕)이 나타난다.
2. **Given** 연결선의 끊기 수단이 보이는 상태, **When** 그것을 누른다, **Then** 연결이 끊기고 저장된다.
3. **Given** 연결을 끊은 직후, **When** 보드를 다시 연다, **Then** 끊긴 상태가 유지된다.

---

### User Story 3 - 이어진 카드 한눈에 보기 (이웃 하이라이트) (Priority: P3)

연결이 많아진 보드에서, 작가가 한 카드를 선택하면 그 카드와 직접 이어진 카드·선만 또렷해지고 나머지는 흐려져 "이 카드가 무엇과 엮였나"를 즉시 본다.

**Why this priority**: 카드·연결이 늘면 캔버스가 빽빽해져 특정 카드의 관계가 묻힌다. 하이라이트는 그 관계를 끄집어낸다. 잇기/끊기가 먼저 있어야 하이라이트할 연결이 존재하므로 P3. 별도 목록 패널 없이 캔버스 위에서 처리해 "책상 위 카드" 단순함을 지킨다.

**Independent Test**: 한 카드가 둘 이상과 이어진 보드에서, 그 카드를 선택하면 이어진 것만 또렷하고 무관한 카드는 흐려지며, 빈 곳을 클릭하면 전부 복원되면 검증 완료.

**Acceptance Scenarios**:

1. **Given** 한 카드가 여러 카드와 이어진 보드, **When** 그 카드를 선택한다, **Then** 선택 카드·이어진 카드·그 사이 선이 또렷해지고 나머지 카드/선은 흐려진다.
2. **Given** 이웃 하이라이트가 켜진 상태, **When** 빈 곳을 클릭한다, **Then** 모든 카드/선이 원래 또렷함으로 복원된다.

---

### Edge Cases

- **빈 곳 drop 확인 취소**: 빈 곳에 놓고 "여기에 새 카드를 만들어 이을까요?"에서 취소하면 새 카드도 연결도 생기지 않는다.
- **연결 직후 출발 카드가 임시(미저장) 상태**: 새로 만든 카드의 저장이 끝나기 전에 연결을 시도하면, 카드 저장 확정 뒤에 연결이 맺어지거나 그 시도가 무시된다(미저장 카드로 향한 연결이 깨진 채 남지 않는다).
- **연결 저장 실패**: 네트워크/서버 오류로 연결 저장이 실패하면 방금 그린 선이 화면에서 사라지고(직전 상태로 복원) 사용자에게 알림이 뜬다.
- **끊기 저장 실패**: 끊기가 실패하면 끊었던 선이 화면에 되살아나고 알림이 뜬다.
- **하이라이트 중 편집/드래그**: 이웃 하이라이트가 켜진 동안에도 카드 본문 편집·드래그 배치는 정상 동작한다(하이라이트는 표시 계층일 뿐).

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 사용자는 한 카드의 연결점을 끌어 다른 카드 위에 놓아 두 카드를 이을 수 있어야 한다.
- **FR-002**: 연결점은 평상시 숨겨지고, 카드에 hover하거나 카드를 선택했을 때만 보여야 한다.
- **FR-003**: 연결 드래그 중 연결선이 커서를 따라오고, 놓을 수 있는 유효한 카드가 시각적으로 강조(초록)되어야 한다.
- **FR-004**: 사용자가 연결을 빈 곳에 놓으면, "여기에 새 카드를 만들어 이을까요?" 확인을 거쳐 그 자리에 새 카드를 만들고 출발 카드와 이어야 한다. 취소 시 새 카드·연결 모두 생기지 않아야 한다.
- **FR-005**: 사용자는 드래그 없이도 — 카드를 선택하고 "잇기"를 누른 뒤 대상 카드를 클릭하는 방식으로 — 두 카드를 이을 수 있어야 한다(클릭-클릭). 이 모드는 ESC 또는 빈 곳 클릭으로 취소되어야 한다.
- **FR-006**: 시스템은 이미 이어진 두 카드 사이에 중복 연결이 생기지 않게 해야 한다. 방향과 무관하게(A→B든 B→A든) 한 쌍은 한 번만 이어진다.
- **FR-007**: 시스템은 카드를 자기 자신에 잇는 시도를 무시해야 한다.
- **FR-008**: 연결선에는 방향 표시(화살표)가 없어야 한다 — 연결은 "두 카드가 이어졌다"는 무방향 관계만 표현한다.
- **FR-009**: 사용자는 연결을 한 동작으로 끊을 수 있어야 한다. 연결선에 hover하면 "연결 끊기" 수단(✕)이 나타나고, 그것으로 연결을 끊는다.
- **FR-010**: 사용자가 카드를 선택하면, 그 카드·직접 이어진 카드·그 사이 연결선이 또렷해지고 나머지 카드/연결선은 흐려져야 한다. 빈 곳을 클릭하면 원래대로 복원되어야 한다.
- **FR-011**: 연결과 끊기는 사용자 동작 즉시 화면에 반영(낙관적)되어야 하고, 저장 실패 시 직전 상태로 복원되며 사용자에게 알림을 보여야 한다.
- **FR-012**: 맺은 연결과 끊은 결과는 영속되어 보드를 다시 열어도 그대로 복원되어야 한다.
- **FR-013**: 연결 기능 추가가 기존 카드 만들기·본문 편집·드래그 배치·뷰포트 복원 동작을 회귀시키지 않아야 한다.
- **FR-014**: 모든 화면 문구는 작가 언어("잇기", "연결 끊기", "이을 카드를 선택하세요", "여기에 새 카드를 만들어 이을까요?")를 쓰고, `node`/`edge` 같은 내부·기술 용어를 노출하지 않아야 한다.

### Key Entities

- **연결(Link)**: 한 보드 안에서 두 카드를 잇는 관계. 화면에서는 **무방향**(방향/화살표 없음). 내부 저장은 출발·도착 카드를 기록하지만(보존된 백엔드 계약), 사용자에게는 방향이 드러나지 않는다. 한 보드에 전속이며, 같은 두 카드 사이에는 최대 1개.
- **카드(Card/노드)**: 보드 위의 한 장. 트랙 A에서 새로 추가되는 속성은 없고, 기존 카드가 연결의 양 끝점이 된다.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 첫 사용자가 별도 안내 없이 화면 단서만으로 두 카드를 잇는 데 성공한다(연결점 발견 → 드래그 → 연결).
- **SC-002**: 드래그 경로와 클릭-클릭 경로 **둘 다**로 연결을 완성할 수 있다.
- **SC-003**: 빈 곳 drop으로 "새 카드 만들기 + 연결"이 한 흐름(확인 1회)으로 완료된다.
- **SC-004**: 잘못 맺은 연결을 한 동작으로 끊을 수 있다.
- **SC-005**: 맺거나 끊은 연결이 보드 재진입 시 100% 복원된다(누락·유령 연결 0).
- **SC-006**: 이미 이어진 쌍 재연결·자기연결 시도에서 중복 연결이나 오류 노출이 0건이다.
- **SC-007**: 기존 카드 만들기·편집·배치·뷰포트 기능의 회귀가 0건이다(자동 게이트 + dogfooding).
- **SC-008**: 한 카드를 선택하면 그 카드와 이어진 것만 또렷해지고, 빈 곳 클릭으로 복원된다.

## Out of Scope (트랙 A 비범위)

- 빈 곳 **더블클릭** 단독 카드 생성(연결 없는 생성, UX worksheet TASK-1 잔여).
- 첫 진입 온보딩 코치마크(TASK-7).
- 카드 종류 변경 UI / "이건 뭔가요?" 칩(TASK-3) — 빈 곳 drop으로 만든 새 카드는 **기본 종류**로 생성.
- 유비쿼터스 언어 전면 rename(`node/edge/노드/board_nodes` → `Card/Link/cards`) — 별도 트랙 B. 트랙 A는 **화면 문구만** 작가 언어로 바꾸고 내부 식별자(`node` 등)는 유지.
- 보드 진입점·매핑·아이디어 보드 고도화 — 트랙 C.
- 연결의 관계 종류(relation_type, "어떤 사이인가요?") — 후속.
- 백엔드 신규 엔드포인트·마이그레이션·신규 에러코드 — **0**. 보존된 계약만 재사용.
- 실시간 공동 편집·모바일 터치 연결 최적화(단 모바일에서 기존 기능 회귀는 없어야).

## Assumptions

- **끊기 기본 수단 = 연결선 hover ✕**. 추가로 연결 선택 후 삭제 키도 동작할 수 있으나, 1차 발견 수단은 hover ✕로 둔다(별도 명세 없이 합리적 default).
- **빈 곳 drop = 확인 모달을 거친다**(즉시 생성 아님). PRD/UX worksheet의 "여기에 새 카드를 만들어 이을까요?" 문구를 따른다.
- 빈 곳 drop으로 만드는 새 카드의 **종류는 기본값**(무지정/기본 종류). 종류 지정은 별도 트랙.
- 보존된 자산을 재사용한다: 백엔드 엣지 계약(연결 생성/삭제, 보드 하이드레이션의 연결 목록)·FE 연결 API·연결 mutation 훅·연결 응답 타입·`BoardDetail.edges`. **신규 백엔드 0**.
- 백엔드 연결 저장은 출발/도착을 순서쌍으로 기록하고 정확한 같은 순서쌍만 중복으로 막으므로(자기연결은 거부), 무방향 의미를 위해 **FE가 "이미 이어진 쌍(어느 방향이든)·자기연결"을 백엔드 호출 전에 선제 차단**한다(불필요한 오류 응답 회피).
- 단일 사용자 가정(동시 편집 비범위).
- 데스크탑/넓은 화면 기준. 모바일 터치 연결 제스처 최적화는 비범위(기존 화면 회귀는 없어야).
- 외관은 앱 단일 라이트 테마에 고정하고(서드파티 캔버스의 OS 자동추종 금지) 기존 앱 강조색(terracotta) 컨벤션을 따른다.

## dogfooding 파생 변경 (2026-06-25, 구현·검증 중 확정)

> 아래는 spec 작성 시점엔 없던, dogfooding에서 사용자와 확정한 변경. 원래 "신규 백엔드 0" 전제가 바뀌었다.

1. **연결 테두리 앵커 영속 → BE V26 확장 (FR-001·SC-005 보강)**. dogfooding에서 "연결선이 항상 카드 위(Top)에만 붙는다"는 문제 발견. 무방향이라도 사용자가 **드래그로 고른 테두리(상/하/좌/우)** 에 선이 붙고 **재진입에도 유지**되어야 한다는 요구. floating(자동 마주보기) 접근을 폐기하고, `board_edges`에 `source_handle`·`target_handle`(VARCHAR(8) nullable, V26) 컬럼을 추가해 앵커를 영속. React Flow 핸들 4개(top/right/bottom/left, id 부여) + `ConnectionMode.Loose`로 **drop 위치에서 가장 가까운 테두리**가 잡힌다. 기존 엣지(앵커 null)는 기본 핸들로 렌더. → **트랙 A가 "신규 BE 0"에서 "BE V26 앵커 추가"로 범위 변경**(BE 선행→FE).
2. **클릭-클릭 진입 UI 변경 (FR-005·FR-014 보강)**. worksheet의 "잇기" 칩(카드 우상단 10px)이 "작고 뭔지 모르겠다"는 dogfooding 피드백 → **카드 바깥 하단에 분리된 은은한(반투명 점선) 인디케이터 "↗ 연결할 카드 고르기"** 로 교체(선택 시 노출). 안내 바 문구도 "이을 카드를 선택하세요" → **"연결할 카드를 클릭하세요"** 로 일관화. 목업 `docs/research/2026-06-25-board-link-button-mockup.html`로 사용자 승인.
