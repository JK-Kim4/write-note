**Feature Branch**: `027-onboarding-tour`

**Created**: 2026-06-20

**Status**: Draft

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 최초 사용자가 핵심 기능을 가이드로 안내받는다 (Priority: P1)

처음 로그인(가입)한 사용자가 홈에 도착하면, 무엇부터 해야 할지 막막하다.
이때 화면의 핵심 버튼들을 순서대로 강조하며 "여기서 이걸 할 수 있어요"라고 설명하는
가이드가 자동으로 뜬다. 사용자는 새 작품 시작 / 메모(곁쪽지) / 등장인물 / 집필
진입의 위치와 용도를 한 바퀴 둘러본 뒤, 무엇을 할지 감을 잡고 첫 행동으로 넘어간다.

**Why this priority**: 본 기능의 본질. 이것만 있어도 "신규 사용자가 핵심 기능을
인지하고 첫 행동으로 이어진다"는 가치를 단독으로 제공한다.

**Independent Test**: 가이드를 한 번도 본 적 없는 계정으로 홈에 진입했을 때
가이드가 자동으로 뜨고, 4개 핵심 요소를 순서대로 강조하며 설명한 뒤 정상 종료되는지로
검증할 수 있다.

**Acceptance Scenarios**:

1. **Given** 가이드를 본 적 없는 사용자가, **When** 홈에 진입하면, **Then** 가이드가 자동으로 시작되어 첫 단계(새 작품 만들기)를 강조·설명한다.
2. **Given** 가이드가 진행 중일 때, **When** 사용자가 "다음"을 누르면, **Then** 다음 핵심 요소(메모 → 인물 → 집필)로 순서대로 강조가 이동한다.
3. **Given** 마지막 단계일 때, **When** 사용자가 "시작하기"를 누르면, **Then** 가이드가 종료되고 사용자는 홈에서 자유롭게 행동할 수 있다.

---

### User Story 2 - 한 번 본 사용자에게는 다시 보이지 않는다 (Priority: P2)

가이드를 이미 끝까지 보거나 건너뛴 사용자가 다시 홈에 들어와도 가이드가 또 뜨면
방해가 된다. 한 번 본 사용자에게는 — 다른 기기나 브라우저로 접속하더라도 — 다시
노출되지 않아야 한다.

**Why this priority**: US1을 쓸 만하게 만드는 필수 보완. 영속이 없으면 매번 떠서
오히려 이탈을 부른다.

**Independent Test**: 가이드를 완료(또는 건너뛰기)한 계정으로 — 같은 기기든 다른
기기든 — 홈에 재진입했을 때 가이드가 뜨지 않는지로 검증할 수 있다.

**Acceptance Scenarios**:

1. **Given** 가이드를 완료한 사용자가, **When** 홈에 다시 진입하면, **Then** 가이드가 뜨지 않는다.
2. **Given** 한 기기에서 가이드를 완료한 사용자가, **When** 다른 기기/브라우저로 로그인해 홈에 진입하면, **Then** 가이드가 뜨지 않는다.

---

### User Story 3 - 언제든 가이드를 건너뛸 수 있다 (Priority: P3)

이미 사용법을 아는 사용자나 지금 당장 쓰기 시작하고 싶은 사용자는 가이드를 중간에
바로 닫을 수 있어야 한다. 건너뛰기도 "이미 봤음"으로 처리되어 다시 뜨지 않는다.

**Why this priority**: 통제감을 주는 보조 흐름. 없어도 US1은 동작하지만, 강제 투어는
사용자 반감을 사므로 포함한다.

**Independent Test**: 가이드 첫 단계에서 "건너뛰기"를 눌렀을 때 즉시 닫히고, 재진입 시
다시 뜨지 않는지로 검증할 수 있다.

**Acceptance Scenarios**:

1. **Given** 가이드가 진행 중인 어느 단계에서든, **When** 사용자가 "건너뛰기"를 누르면, **Then** 가이드가 즉시 종료된다.
2. **Given** 건너뛰기로 가이드를 닫은 사용자가, **When** 홈에 다시 진입하면, **Then** 가이드가 뜨지 않는다.

---

### Edge Cases

- **완료 상태 조회 실패**: 사용자의 "이미 봤음" 상태를 불러오지 못하면, 가이드를 띄우지 않는다(핵심 화면 흐름을 절대 차단하지 않음). 잘못 띄워 방해하기보다 안 띄우는 쪽을 택한다.
- **완료 상태 저장 실패**: 가이드를 닫을 때 "이미 봤음" 저장에 실패해도 사용자 동작을 막지 않는다(다음 진입에서 다시 시도될 수 있음).
- **강조 대상 요소 부재**: 가이드가 가리키려는 버튼이 화면에 없으면 해당 단계를 건너뛴다. 정상 경로(홈 + 고정 네비)에선 항상 존재하므로 발생하지 않는다.
- **작품 0개 상태**: 최초 사용자는 작품이 없다. "집필" 단계는 작품이 없어도 위치·용도를 설명하는 데 문제 없도록 한다.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 가이드를 본 적 없는 사용자가 홈에 진입하면 시스템은 가이드 투어를 자동으로 시작해야 한다.
- **FR-002**: 가이드는 핵심 요소 4가지를 정해진 순서로 강조해야 한다 — (1) 새 작품 만들기, (2) 메모(곁쪽지), (3) 등장인물, (4) 집필 진입.
- **FR-003**: 각 단계에서 사용자는 다음 단계로 진행하거나 가이드를 건너뛸 수 있어야 한다.
- **FR-004**: 가이드를 끝까지 완료하거나 건너뛰면 시스템은 해당 사용자의 "가이드 완료" 상태를 영속 저장해야 한다.
- **FR-005**: "가이드 완료" 상태가 저장된 사용자에게는 — 접속 기기·브라우저와 무관하게 — 가이드를 다시 노출하지 않아야 한다.
- **FR-006**: 가이드는 페이지 이동 없이 홈 한 화면에서 진행되어야 한다(여러 화면을 넘나들지 않는다).
- **FR-007**: "가이드 완료" 상태 조회에 실패하면 시스템은 가이드를 띄우지 않아야 한다(핵심 화면 흐름 비차단).
- **FR-008**: 가이드는 강조하는 대상 외 화면을 흐리게 처리해 집중을 유도하되, 종료 후에는 화면을 원래대로 되돌려야 한다.

### Key Entities *(include if feature involves data)*

- **가이드 완료 상태(onboarding completion)**: 사용자별로 "최초 가이드를 이미 보았는가"를 나타내는 단일 상태값. 기존 사용자별 설정 저장소(서버 영속)에 보관한다. 미저장 = 미완료로 간주.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 가이드를 본 적 없는 사용자가 홈에 처음 진입했을 때 100% 가이드가 자동 노출된다.
- **SC-002**: 가이드를 완료/건너뛴 사용자가 재방문(같은 기기·다른 기기 모두)했을 때 가이드가 다시 노출되는 비율 0%.
- **SC-003**: 사용자는 가이드 전체(4단계)를 30초 이내에 끝까지 보거나 즉시 건너뛸 수 있다.
- **SC-004**: 가이드는 어떤 경우에도 로그인·집필 등 핵심 화면 동작을 막지 않는다(가이드 관련 장애로 인한 핵심 흐름 차단 0건).

## Assumptions

- 기존 사용자별 설정 영속 저장소(서버)를 재사용해 "가이드 완료" 상태를 보관한다. 신규 데이터 저장 구조나 스키마 변경은 도입하지 않는다.
- 가이드는 홈 화면과 모든 화면 공통의 상단 네비게이션 요소를 대상으로 하며, 이들은 항상 화면에 존재한다고 가정한다.
- 가이드 시각 효과(특정 요소 강조 + 주변 흐림 + 설명 말풍선)는 검증된 경량 가이드 도구로 구현한다(구체 도구 선택은 구현 계획에서 확정).
- "설정 화면에서 가이드 다시 보기" 기능은 v1 범위 밖이다(추후 손쉽게 추가 가능).
- 화면별 맥락 도움말이나 여러 화면을 넘나드는 투어는 범위 밖이다(홈 단일 화면 미니 투어로 한정).
