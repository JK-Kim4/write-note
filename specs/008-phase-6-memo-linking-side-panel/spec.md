## User Scenarios & Testing *(mandatory)*

작가(앱의 단일 로컬 사용자)가 떠오른 생각을 메모로 캡처한 뒤, 그 메모를 관련 작품에 연결해 정리하고, 집필 중에는 현재 쓰는 작품에 연결된 메모만 곁에서 참고한다.

### User Story 1 - 메모를 작품에 연결하고 해제한다 (Inbox) (Priority: P1)

작가가 메모 화면(Inbox)에서 미연결 메모를 특정 작품에 연결하거나, 이미 연결된 메모의 연결을 해제한다. 메모마다 현재 어느 작품에 연결돼 있는지 한눈에 보인다.

**Why this priority**: 캡처된 메모가 작품 맥락으로 들어가는 핵심 동선이다. 이 슬라이스만으로도 Inbox 가 "흩어진 메모를 작품별로 정리하는 공간"이 되어 단독으로 가치를 준다.

**Independent Test**: 미연결 메모 1개를 만들고 작품 A 에 연결 → 메모 행에 작품 A 가 표시됨을 확인. 연결을 해제 → 다시 미연결로 표시됨을 확인. 다른 화면 없이 Inbox 안에서 완결된다.

**Acceptance Scenarios**:

1. **Given** 미연결 메모가 있고 작품이 하나 이상 존재, **When** 메모의 연결 동작으로 작품 A 를 선택, **Then** 그 메모가 작품 A 에 연결되고 메모 행에 작품 A 가 표시된다.
2. **Given** 작품 A 에 연결된 메모가 있음, **When** 그 연결을 해제, **Then** 메모는 미연결 상태가 되고 작품 A 표시가 사라진다.
3. **Given** "미연결" 필터가 켜져 있음, **When** 어떤 미연결 메모를 작품에 연결, **Then** 그 메모는 미연결 목록에서 빠진다.
4. **Given** 작품이 하나도 없음, **When** 메모 연결을 시도, **Then** "먼저 작품을 만들어 주세요" 류의 안내가 보이고 연결은 생성되지 않는다.

---

### User Story 2 - 집필 중 현재 작품에 연결된 메모를 곁에서 본다 (Priority: P1)

작가가 한 작품의 집필 화면을 열면, 우측의 조용한 사이드 패널에 **그 작품에 연결된 메모만** 표시된다. 집필 중 떠오른 맥락을 끊지 않고 참고할 수 있으며, 더는 관련 없는 메모는 패널에서 바로 연결 해제할 수 있다.

**Why this priority**: "컨텍스트가 안 죽는 작업공간"이라는 제품 본질의 집필측 표현이다. 직전 Phase 5 dogfooding 에서 이 패널이 더미라 혼란을 일으킨 바로 그 지점을 실데이터로 해소한다.

**Independent Test**: 작품 A 에 메모 1개를 연결해 둔 뒤 작품 A 집필 화면을 연다 → 패널에 그 메모만 보임을 확인. 패널에서 해제 → 패널에서 사라짐을 확인. 작품 B 집필 화면에는 그 메모가 보이지 않음을 확인.

**Acceptance Scenarios**:

1. **Given** 작품 A 에 메모 2개가 연결되고 작품 B 에는 0개, **When** 작품 A 집필 화면을 연다, **Then** 패널에 작품 A 의 메모 2개만 보인다(작품 B·미연결 메모는 안 보임).
2. **Given** 작품 A 집필 화면에서 패널에 메모가 보임, **When** 패널에서 그 메모를 해제, **Then** 패널에서 즉시 사라진다.
3. **Given** 작품 A 에 연결된 메모가 0개, **When** 작품 A 집필 화면을 연다, **Then** 패널이 비어 있어도 어색하지 않은 빈 상태 안내가 보인다.
4. **Given** 집필 화면이 열려 있음, **When** 패널을 본다, **Then** 패널은 글쓰기 영역보다 시각적으로 약하게(조용하게) 나타난다.

---

### User Story 3 - 한 메모를 여러 작품에 연결한다 (다중 연결) (Priority: P2)

하나의 메모가 여러 작품에 동시에 의미를 가질 수 있다. 작가는 한 메모를 둘 이상의 작품에 연결하고, 각 작품의 집필 화면 모두에서 그 메모를 만난다.

**Why this priority**: 다중 연결은 본 Phase 의 범위 확장 핵심이지만, US1·US2 의 단일 연결 흐름이 선행되어야 의미가 있다. 단일 연결이 동작하면 다중은 그 위의 확장 슬라이스다.

**Independent Test**: 메모 1개를 작품 A 와 작품 B 양쪽에 연결 → 메모 행에 두 작품이 모두 표시됨을 확인. 작품 A 집필 화면과 작품 B 집필 화면 양쪽 패널에서 그 메모가 보임을 확인.

**Acceptance Scenarios**:

1. **Given** 메모 1개와 작품 A·B 가 있음, **When** 그 메모를 작품 A 와 작품 B 에 연결, **Then** 메모 행에 두 작품이 모두 연결로 표시된다.
2. **Given** 메모가 작품 A·B 에 연결됨, **When** 작품 A 집필 화면과 작품 B 집필 화면을 각각 연다, **Then** 양쪽 패널 모두에 그 메모가 보인다.
3. **Given** 메모가 작품 A·B 에 연결됨, **When** 작품 A 와의 연결만 해제, **Then** 메모는 여전히 작품 B 에 연결돼 있고 작품 B 패널에는 그대로 보인다.
4. **Given** 메모가 이미 작품 A 에 연결됨, **When** 같은 작품 A 에 다시 연결을 시도, **Then** 중복 연결이 생기지 않는다(이미 연결됨 상태 유지).

---

### Edge Cases

- **작품 없음에서 연결 시도**: 작품이 0개일 때 연결하려 하면 안내만 보이고 연결은 만들어지지 않는다.
- **연결된 작품 삭제**: 작품을 삭제하면 그 작품의 메모 연결만 사라지고, 메모 자체는 보존된다(다른 작품과의 연결은 유지).
- **삭제된(soft-delete) 메모**: Inbox·집필 패널 어디에도 보이지 않는다. 복원하면 기존 연결이 그대로 복귀한다.
- **중복 연결**: 같은 메모를 같은 작품에 두 번 연결해도 연결은 하나로 유지된다(멱등).
- **연결 메모 없는 작품 패널**: 빈 상태 안내가 보이고 화면이 어색하지 않다.
- **긴 본문 메모**: 패널·Inbox 카드에서 본문이 과도하게 영역을 차지하지 않게 표시된다.
- **연결 변경 후 갱신**: 연결을 바꾼 뒤 해당 작품 집필 화면 패널이 최신 연결 상태를 반영한다(재진입 또는 패널 내 변경 시 즉시).

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 작가는 미연결 메모를 특정 작품에 연결할 수 있어야 한다.
- **FR-002**: 작가는 하나의 메모를 둘 이상의 작품에 동시에 연결할 수 있어야 한다(다중 연결).
- **FR-003**: 작가는 메모와 작품의 연결을 해제할 수 있어야 한다.
- **FR-004**: Inbox 의 각 메모는 현재 연결된 작품들을 표시해야 한다(미연결이면 미연결로 표시).
- **FR-005**: Inbox 에서 메모 연결은 메모 행의 전용 연결 동작으로 시작하며, 연결 표시(칩)에서도 해제할 수 있어야 한다.
- **FR-006**: 집필 화면은 현재 열려 있는 작품에 연결된 메모만 표시해야 한다(다른 작품 메모·미연결 메모·삭제된 메모는 노출하지 않는다).
- **FR-007**: 작가는 집필 화면의 사이드 패널에서 메모 연결을 해제할 수 있어야 한다(연결 추가는 Inbox 에서 한다).
- **FR-008**: 집필 화면 사이드 패널은 글쓰기 영역보다 시각적으로 약하게(조용한 참조 영역으로) 표현되어야 한다.
- **FR-009**: 연결을 변경한 뒤 해당 작품의 집필 패널은 최신 연결 상태를 반영해야 한다(재진입 시, 그리고 패널 내에서 해제 시 즉시).
- **FR-010**: 메모 캡처 시 현재 활성 작품이 있으면 그 작품에 자동 연결하고, 없으면 미연결로 저장해야 한다(기존 캡처 동작 보존).
- **FR-011**: 작품을 삭제하면 그 작품의 메모 연결만 제거되고 메모 자체는 보존되어야 한다.
- **FR-012**: 삭제된(soft-delete) 메모는 Inbox·집필 패널에 표시되지 않으며, 복원 시 기존 연결이 복귀해야 한다.
- **FR-013**: 작품이 하나도 없을 때 연결을 시도하면 안내를 제공하고 연결은 생성하지 않아야 한다.
- **FR-014**: 같은 메모와 같은 작품 사이의 중복 연결은 생성되지 않아야 한다(멱등).
- **FR-015**: 연결된 메모가 없는 작품의 집필 패널은 어색하지 않은 빈 상태 안내를 제공해야 한다.

### Key Entities *(include if feature involves data)*

- **메모 (Memo)**: 작가가 캡처한 짧은 글. 본문, 캡처 시각, 삭제(soft-delete) 상태를 가진다. 0개 이상의 작품에 연결될 수 있다.
- **작품 (Project)**: 글쓰기 프로젝트. 0개 이상의 메모와 연결될 수 있다.
- **메모-작품 연결 (Memo–Project Link)**: 메모와 작품 사이의 다대다 연관. 한 메모가 여러 작품에, 한 작품이 여러 메모에 연결될 수 있다. 같은 (메모, 작품) 쌍은 최대 하나만 존재한다.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 작가는 미연결 메모를 작품에 연결하는 동작을 메모 행의 한 진입점(연결 → 작품 선택)으로 완료할 수 있다.
- **SC-002**: 한 메모를 둘 이상의 작품에 연결할 수 있고, 연결된 모든 작품의 집필 화면에서 그 메모가 100% 빠짐없이 표시된다.
- **SC-003**: 집필 화면 패널에는 현재 작품에 연결된 메모만 정확히 표시된다(다른 작품 메모·미연결 메모·삭제된 메모 노출 0건).
- **SC-004**: 연결을 해제하면 해당 작품 집필 패널에서 그 메모가 사라진다(패널 내 해제는 즉시, 다른 화면에서의 변경은 재진입 시 반영).
- **SC-005**: 메모·작품이 적은 초기 상태에서도 Inbox·집필 패널이 무겁게 느껴지지 않는다(빈 상태 안내 제공, 과한 UI 없음).
- **SC-006**: 기존에 단일 연결돼 있던 메모는 본 Phase 적용 후에도 그 연결이 보존된다(데이터 손실 0건).

## Assumptions

- 앱은 로컬 단일 사용자 데스크탑 앱이다. 인증·다중 사용자·권한은 범위가 아니다.
- 작품·메모·문서의 로컬 영속 계층은 기존 것을 재사용한다.
- 연결 단위는 **작품(project)** 이다. 등장인물(character)·개별 문서 단위 연결은 본 Phase 범위가 아니다.
- 캡처 시 자동 연결 기준은 "현재 활성 작품"이며, 이 동작은 기존(Phase 5)에서 그대로 계승한다.
- 다중 연결 포함은 본 Phase 범위 확장이다(작업 명세 §2). 기존 단일 연결 데이터는 보존 이관한다.
- 연결/해제 변경은 외부 데이터 스토어가 아닌 로컬 저장소에만 일어난다.
