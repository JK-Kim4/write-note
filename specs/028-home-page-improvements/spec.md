# Feature Specification: 홈(메인) 페이지 개선

**Feature Branch**: `develop` (전용 브랜치 미생성 — 사용자 지시로 develop 직접 작업)

**Created**: 2026-06-20

**Status**: Draft

**Input**: User description: 홈 메인 페이지 3가지 개선 — (A) 인사 문구를 문학 인용구 무작위 회전으로, (B) 집필 리듬 즉시 반영 + 오늘 강조 (+B′ 세션 최소시간 완화), (C) 오늘 작업시간 원통형 게이지 + 일일 목표 시간 설정.

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 집필한 시간이 홈에 즉시 반영된다 (Priority: P1)

작가가 집필실에서 글을 쓴 뒤 홈으로 돌아오면, 방금 작업한 시간이 "집필 리듬" 카드에 곧바로 반영되어 보인다. 또한 오늘이 이번 주의 어느 요일인지 한눈에 알 수 있다.

**Why this priority**: 현재 작가가 분명히 작업했는데도 리듬 카드가 비어 보이는 결함이 보고되었다. "내가 쓴 게 기록된다"는 신뢰가 도구의 핵심 가치이며, 이게 깨지면 나머지 개선은 의미가 약하다. 가장 먼저 바로잡아야 할 결함.

**Independent Test**: 작품에 들어가 일정 시간(최소 임계 이상) 머무른 뒤 홈으로 나와, 리듬 카드에 그 시간이 막대로 나타나고 오늘 요일이 강조되는지로 단독 검증 가능.

**Acceptance Scenarios**:

1. **Given** 작가가 이번 주에 한 번도 집필 세션을 종료한 적이 없다, **When** 홈을 연다, **Then** 리듬 카드는 빈 상태임을 알리는 안내 문구를 보여준다(빈 막대만 덩그러니 X).
2. **Given** 작가가 집필실에서 최소 임계 시간 이상 작업했다, **When** 집필실을 떠나 홈으로 돌아온다, **Then** 추가 새로고침 없이 그 작업시간이 리듬 카드 막대에 반영되어 보인다.
3. **Given** 리듬 카드가 표시된다, **When** 작가가 카드를 본다, **Then** 이번 주 7일 중 "오늘"에 해당하는 막대가 날짜와 함께 시각적으로 명확히 구분된다.

---

### User Story 2 - 오늘 작업시간을 목표 대비로 한눈에 본다 (Priority: P2)

작가는 오늘 자신이 얼마나 작업했는지를, 스스로 정한 하루 목표 시간 대비 얼마나 찼는지 원통형 게이지로 즉시 파악한다. 목표 시간은 설정에서 바꿀 수 있다.

**Why this priority**: 동기 부여·습관 형성의 핵심 피드백. P1(즉시 반영)이 전제되어야 의미가 있으므로 그 다음 우선순위. 목표 설정은 다기기 동기화되어야 일관된 경험을 준다.

**Independent Test**: 설정에서 일일 목표를 정한 뒤 오늘 일정 시간 작업하고 홈에서 게이지 채움이 (오늘 작업시간 ÷ 목표)에 맞게 나타나는지로 단독 검증 가능.

**Acceptance Scenarios**:

1. **Given** 일일 목표가 기본값(60분)이고 오늘 작업시간이 30분이다, **When** 홈을 연다, **Then** 원통형 게이지가 약 절반 찬 상태로 "오늘 30분 / 목표 1시간"을 보여준다.
2. **Given** 오늘 작업시간이 목표를 초과했다, **When** 홈을 연다, **Then** 게이지는 가득 찬 상태로 목표 달성이 표시된다(넘침으로 깨지지 않음).
3. **Given** 작가가 설정 페이지에서 일일 목표를 90분으로 바꾼다, **When** 다른 기기/세션에서 홈을 연다, **Then** 게이지 기준이 90분으로 동기화되어 적용된다.
4. **Given** 오늘 작업시간이 0이다, **When** 홈을 연다, **Then** 게이지는 빈 상태이되 "오늘 0분 / 목표 …"로 정상 표시된다(NaN·깨짐 없음).

---

### User Story 3 - 인사 자리에 문학 인용구가 회전한다 (Priority: P3)

홈에 들어올 때마다 인사 부제 자리에 유명 문학 작품의 한 구절이 저자 표기와 함께 무작위로 바뀌어 나타난다. "안녕하세요." 인사와 오늘 날짜는 그대로 유지된다.

**Why this priority**: 정서적 가치(작가에게 영감)를 주는 마감 터치. 기능적 결함이 아니므로 가장 낮은 우선순위지만 독립적으로 구현·배포 가능.

**Independent Test**: 홈을 여러 번 새로 열어 부제의 인용구가 바뀌고, 날짜·"안녕하세요."는 유지되며, 각 인용구에 저자가 표기되는지로 단독 검증 가능.

**Acceptance Scenarios**:

1. **Given** 작가가 홈을 연다, **When** 화면을 본다, **Then** "안녕하세요."와 오늘 날짜는 유지되고, 그 뒤에 문학 인용구 한 구절과 저자가 표시된다.
2. **Given** 작가가 홈을 다시 연다(재방문/새로고침), **When** 화면을 본다, **Then** 직전과 다른 인용구가 나타날 수 있다(목록에서 무작위 선택).

---

### Edge Cases

- **빈 막대의 진짜 원인**: 리듬 카드가 비어 보이는 현상의 근본 원인(세션 미기록 / 세션 미종료 / 표시 캐시)은 구현 1단계에서 라이브 데이터를 직접 관찰해 확정한다. 임계값 완화·즉시 갱신만으로 해결되지 않으면(예: 세션 종료 신호 유실) 원인을 더 좁혀 별도 보고한다. **관찰 전 원인 단정 금지.**
- 작가가 집필실에 들어갔다가 최소 임계 시간 미만으로 곧장 나오면 그 세션은 기록되지 않는다(노이즈 제거 의도). 임계 완화로 더 짧은 작업도 잡히지만, 임계 미만은 여전히 미기록임을 빈 상태 안내가 호도하지 않아야 한다.
- 일일 목표 설정값이 서버에 아직 없는 신규/기존 사용자는 기본값(60분)으로 동작한다.
- 인용구 목록이 비어 있으면(이론상) 인사 부제는 기존 톤의 기본 문구로 안전하게 폴백한다.
- 자정을 넘겨 "오늘"이 바뀌면 게이지·오늘 강조는 새 날짜 기준으로 갱신된다.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 홈은 집필 세션이 종료된 직후 작가가 홈으로 돌아오면 추가 수동 새로고침 없이 갱신된 이번 주 작업시간을 보여줘야 한다.
- **FR-002**: 집필 리듬 카드는 이번 주 7일(월~일) 중 "오늘"을 날짜와 함께 시각적으로 명확히 구분해 표시해야 한다.
- **FR-003**: 집필 리듬 카드는 이번 주 기록이 전혀 없을 때 빈 막대 대신 빈 상태임을 알리는 안내 문구를 보여줘야 한다.
- **FR-004**: 시스템은 집필 세션의 최소 인정 시간(이 시간 미만은 폐기) 임계를 더 짧게(10초) 적용해 짧은 작업도 더 잘 기록되게 해야 한다.
- **FR-005**: 홈은 오늘의 누적 작업시간을 일일 목표 시간 대비 채움 비율로 보여주는 원통형 게이지를 제공해야 한다.
- **FR-006**: 오늘 작업시간이 일일 목표를 초과해도 게이지는 가득 참으로 안정적으로 표시되고 목표 달성을 알려야 한다.
- **FR-007**: 작가는 일일 작업 목표 시간을 정해진 이산 선택지(30/60/90/120/180/240/300분) 중에서 설정 페이지에서 선택·변경할 수 있어야 하며, 미설정 시 기본 60분이 적용되어야 한다.
- **FR-008**: 일일 작업 목표 설정값은 서버에 영속되어 기기·세션 간 동기화되어야 한다(기존 환경설정 동기화 방식과 동일).
- **FR-009**: 홈 인사 영역은 "안녕하세요."와 오늘 날짜를 유지하면서, 부제 뒷문구 자리에 문학 인용구 한 구절과 저자 표기를 보여줘야 한다.
- **FR-010**: 인용구는 홈을 열 때(로드)마다 내장된 큐레이션 목록에서 무작위로 선택되어야 한다.
- **FR-011**: 인용구 목록은 저작권상 안전한 출처(저자 사후 70년 경과 퍼블릭 도메인 원문, 또는 자체 번역/한국 작가 원문)만으로 구성해야 한다.
- **FR-012**: 일일 목표 설정의 백엔드 변경(허용 설정 키 추가)은 프론트가 해당 키를 전송하기 전에 먼저 배포되어야 한다(BE 선행 → FE 후행).

### Key Entities *(include if feature involves data)*

- **일일 작업 목표(dailyGoalMinutes)**: 사용자별 하루 목표 작업 시간(분). 허용 이산값 {30,60,90,120,180,240,300}, 기본 60. 기존 사용자 환경설정 저장소(key-value)에 신규 키로 추가, 다기기 동기화.
- **문학 인용구**: 본문(text)과 저자(author)로 구성된 정적 큐레이션 항목. repo 내장, 외부 네트워크 비의존.
- **집필 리듬(작업 세션 파생)**: 이번 주 요일별 작업시간 합계와 오늘 작업시간. 기존 작업 세션 집계를 재사용하며, 신규 데이터 소스나 신규 조회 경로를 추가하지 않는다.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 작가가 집필실에서 10초 이상 작업한 뒤 홈으로 돌아오면, 추가 조작 없이 그 작업시간이 리듬 카드에 반영되어 보인다.
- **SC-002**: 홈을 본 작가가 별도 설명 없이도 이번 주 7일 중 어느 막대가 "오늘"인지 즉시 식별할 수 있다.
- **SC-003**: 홈을 본 작가가 오늘 작업시간이 자신의 목표 대비 어느 정도인지 게이지 한 번 보고 파악할 수 있다.
- **SC-004**: 일일 목표를 한 기기에서 바꾸면 다른 기기/세션의 홈에 동일 기준이 적용된다.
- **SC-005**: 홈을 연속해서 열었을 때 인용구가 회전(변경 가능)하며, "안녕하세요."·날짜·저자 표기가 항상 함께 보인다.
- **SC-006**: 위 변경 어디에서도 빈 데이터(작업 0분, 기록 없는 주)로 인한 화면 깨짐(NaN·빈 막대 오해·레이아웃 붕괴)이 없다.

## Assumptions

- 작업 브랜치는 신규 feature 브랜치를 만들지 않고 `develop`에서 직접 작업한다(사용자 명시 지시 — speckit `before_specify` 브랜치 훅 건너뜀).
- "집필 리듬"과 "오늘 작업시간"의 데이터는 기존 작업 세션 집계(이번 주 요일별 합계)를 재사용하며, 신규 백엔드 조회 endpoint를 추가하지 않는다.
- 일일 목표는 기존 사용자 환경설정 동기화 인프라(서버 SoT + 로컬 캐시 미러)에 신규 키로 얹는다. 설정 페이지 UI도 기존 설정 항목과 동일한 패턴으로 추가한다.
- 세션 최소시간 임계는 애플리케이션 설정값이며, 운영 환경에 별도 환경변수 override가 존재할 수 있다(그 경우 운영 배포 시 사용자 영역에서 갱신 필요).
- 인용구 목록은 20~40개 규모로 시작하며, 항목 추가는 데이터 파일 편집만으로 가능하다.
- 빈 리듬 카드의 근본 원인은 추측하지 않고 구현 1단계 관찰로 확정한다. 본 스펙의 즉시 갱신·임계 완화는 가장 유력하고 안전한 개선이지만, 관찰 결과가 다르면 후속 보고 후 범위를 조정한다.
