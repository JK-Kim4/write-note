**Feature Branch**: `050-share-enhancements`

**Created**: 2026-07-01

**Status**: Draft

**Input**: 브레인스토밍 확정 요구사항 정의서 `docs/superpowers/specs/2026-07-01-share-enhancements-design.md` + 사용자 승인 목업 4종(`docs/research/2026-07-01-share-*-mockup.html`). 046 공유하기 + 047 공유 사용성 개선 위에 4개 갭 해소.

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 작가용 피드백 맥락 뷰 (Priority: P1)

작가는 홈의 "공유" 메뉴에서 자기가 만든 공유 링크의 글 전문을, 독자들이 남긴 피드백이 본문 어느 구간에 달렸는지와 함께 한 화면에서 본다. 지금은 공유 관리 화면에 링크를 여는 버튼조차 없어 URL을 복사해 브라우저에 붙여넣어야 하고, 받은 피드백은 조각 텍스트 모달로만 볼 수 있다.

**Why this priority**: 사용자가 가장 먼저 요청한 갬이며, "공유한 글을 읽으며 피드백을 맥락으로 확인"이 공유 기능의 핵심 가치다. 이것 하나만 있어도 작가의 피드백 확인 경험이 완결된다.

**Independent Test**: 공유 링크가 있고 독자 댓글이 달린 작품에서 "공유" 메뉴 → 그 링크의 "받은 피드백"을 눌러, 전문 위에 하이라이트된 피드백을 우측 패널로 보고 항목 클릭 시 해당 구간으로 이동하는지 확인하면 독립적으로 검증된다.

**Acceptance Scenarios**:

1. **Given** 공유 링크에 독자 댓글 2건이 달림, **When** 작가가 "공유" 메뉴에서 그 링크의 "받은 피드백"을 연다, **Then** 글 전문이 종이 형태로 표시되고 댓글 달린 구간이 하이라이트되며 우측 패널에 2건이 나열된다.
2. **Given** 맥락 뷰가 열려 있음, **When** 작가가 우측 패널의 한 항목을 클릭한다, **Then** 본문이 그 구간으로 스크롤되고 해당 구간이 강조된다.
3. **Given** 같은 작품을 두 번 공유해 링크가 2개, **When** 작가가 한 링크의 맥락 뷰를 연다, **Then** 그 링크(스냅샷)에 달린 피드백만 보이고 다른 링크의 피드백은 섞이지 않는다.
4. **Given** 맥락 뷰에 안 읽은 피드백이 있음, **When** 작가가 뷰를 연다, **Then** 그 링크의 받은 피드백이 읽음 처리되어 안 읽음 배지가 사라진다.
5. **Given** 다른 사용자의 공유 링크, **When** 그 링크의 맥락 뷰 접근을 시도한다, **Then** 접근이 거부된다.

### User Story 2 - 비로그인 방문자 로그인 (Priority: P2)

공유 링크로 들어온 비로그인 방문자가 의견을 남기고 싶을 때, 공개 페이지에서 바로 로그인하고 로그인 후 보던 그 공유 페이지로 돌아온다. 지금은 공개 페이지에 로그인 수단이 아예 없어(안내 문구만) 외부 방문자가 회원이 되어 피드백을 남길 길이 없다.

**Why this priority**: 피드백을 남길 사람(외부 독자)이 로그인조차 못 하면 공유 피드백 기능 자체가 닫힌다. 새 피드백 유입의 관문.

**Independent Test**: 로그아웃 상태로 공유 페이지에 들어가 로그인 진입점을 눌러 로그인한 뒤, 홈이 아니라 같은 공유 페이지로 복귀하는지 확인하면 독립 검증된다.

**Acceptance Scenarios**:

1. **Given** 비로그인 방문자가 공유 글을 보는 중, **When** 헤더 "로그인" 또는 본문 CTA를 누른다, **Then** 로그인 화면으로 이동하고, 로그인 성공 후 보던 공유 페이지로 자동 복귀한다.
2. **Given** 비로그인 방문자가 본문 구간을 드래그한다, **When** 의견을 남기려 한다, **Then** 로그인 유도가 뜨고 로그인 후 같은 페이지로 복귀한다.
3. **Given** 카카오로 로그인, **When** 카카오 화면을 거쳐 돌아온다, **Then** 홈이 아니라 원래 공유 페이지로 복귀한다.
4. **Given** 복귀 대상이 공유 페이지가 아닌 임의 주소로 조작됨, **When** 로그인 복귀가 일어난다, **Then** 임의 외부 주소로 이동하지 않고 안전한 기본 경로로 처리된다.

### User Story 3 - 댓글 고도화: 이모지 반응 + 전체 의견 (Priority: P2)

독자가 글을 쓰지 않고도 마음에 드는 구간에 이모지로 빠르게 반응하고(모든 열람자가 반응 개수를 봄), 특정 구간이 아니라 작품 전체에 대한 한마디("전체 의견")도 남긴다. 지금은 반드시 텍스트 구간을 선택해 평문 댓글만 남길 수 있다.

**Why this priority**: 피드백의 문턱을 낮추고(반응) 표현 범위를 넓힌다(전체 의견). 단, US1/US2가 없으면 가치가 반감되므로 P2.

**Independent Test**: 회원이 공유 글에서 구간에 이모지를 눌러 개수가 오르고 다시 눌러 취소되는지, 하단 "작품 전체에 한마디"로 앵커 없는 의견이 남는지 확인하면 독립 검증된다.

**Acceptance Scenarios**:

1. **Given** 로그인 회원이 공유 글을 봄, **When** 본문 구간을 선택해 이모지(❤️)를 누른다, **Then** 그 구간에 반응이 기록되고 개수가 1 오른다.
2. **Given** 이미 ❤️를 누른 구간, **When** 같은 회원이 ❤️를 다시 누른다, **Then** 반응이 취소되고 개수가 1 내린다.
3. **Given** 어떤 구간에 반응이 5개, **When** 다른 열람자(비로그인 포함)가 그 글을 본다, **Then** "❤️ 5"처럼 개수가 보이되 누가 눌렀는지 개인은 보이지 않는다.
4. **Given** 로그인 회원이 공유 글을 봄, **When** 하단 "작품 전체에 한마디"에 의견을 남긴다, **Then** 구간 앵커 없이 작품 전체 대상 의견으로 저장되고 작가에게만 보인다.
5. **Given** 작가가 US1 맥락 뷰를 봄, **When** 반응과 전체 의견이 존재한다, **Then** 구간별 반응 집계와 전체 의견을 함께 확인한다.

### User Story 4 - "종이 위의 글" 레이아웃 (Priority: P3)

공유 열람 화면이 집필 화면처럼 종이(흰 페이지) 위에 글이 놓인 형태로 보인다. 지금은 빈 배경에 글만 흘러 밋밋하고, 다크 모드에서 본문 색이 고정되어 있다.

**Why this priority**: 읽기 경험·브랜드 완성도를 높이는 마감. 기능 의존이 없어 독립 배포 가능하나 핵심 기능은 아니라 P3.

**Independent Test**: 공유 열람 화면과 작가 맥락 뷰에서 흰 종이 페이지·어두운 바깥 배경이 집필 화면과 같은 느낌으로 보이고, 다크 모드에서 본문이 가독한지 확인하면 독립 검증된다.

**Acceptance Scenarios**:

1. **Given** 공개 공유 글 열람, **When** 화면을 본다, **Then** 흰 종이 페이지 위에 글이 놓이고 바깥은 어두운 톤(집필 화면과 동일)으로 보인다.
2. **Given** 작가 맥락 뷰(US1), **When** 화면을 본다, **Then** 같은 종이 스타일이 적용된다.
3. **Given** 다크 모드, **When** 공유 글을 본다, **Then** 종이·본문이 다크 톤으로 전환되어 읽기 좋다.

---

### Edge Cases

- 한 작품을 여러 번 공유(링크/스냅샷 여럿)했을 때 피드백·반응이 링크별로 분리되어 섞이지 않아야 한다.
- 공유 대상(작품/시리즈) 삭제로 링크가 비활성화된 뒤에도 보존된 피드백·반응 열람(맥락 뷰)이 깨지지 않아야 한다(046 보존 정책 정합).
- 로그인 복귀 도중 세션 만료·실패 시 임의 이동 없이 안전하게 처리되어야 한다.
- 비로그인 방문자가 반응·댓글을 시도하면 로그인으로 유도되어야 한다.
- 반응 앵커 구간이 글 댓글 하이라이트와 겹칠 때 시각적으로 구분/집계 표기가 무너지지 않아야 한다.
- 이미 반응/댓글을 남긴 회원이 같은 스냅샷을 다시 열 때 자신의 상태(내가 누른 반응)가 반영되어야 한다.

## Requirements *(mandatory)*

### Functional Requirements

**작가용 피드백 맥락 뷰 (US1)**
- **FR-001**: 작가는 공유 관리 화면에서 각 공유 링크의 "받은 피드백"을 통해, 그 링크로 공유한 글 전문과 받은 피드백을 한 화면에서 볼 수 있어야 한다(URL 복사·붙여넣기 없이).
- **FR-002**: 맥락 뷰는 글 전문을 읽기 전용으로 표시하고, 댓글이 달린 구간을 항상 하이라이트로 표시해야 한다.
- **FR-003**: 우측 피드백 패널 항목(독자 이름·인용·내용·시각·안읽음 표시)을 선택하면 본문이 해당 구간으로 이동하고 그 구간이 강조되어야 한다.
- **FR-004**: 맥락 뷰는 한 공유 링크(스냅샷) 범위의 피드백만 모아 보여야 한다(같은 글을 여러 번 공유했으면 링크별로 분리).
- **FR-005**: 맥락 뷰 진입 시 그 링크의 받은 피드백은 읽음 처리되어야 한다.
- **FR-006**: 맥락 뷰는 해당 공유의 소유 작가만 접근할 수 있어야 하며, 타인 접근은 거부되어야 한다.

**비로그인 방문자 로그인 (US2)**
- **FR-007**: 비로그인 방문자는 공개 공유 페이지에서 로그인으로 진입할 수단을 가져야 한다(헤더 진입점 + 본문 안내 + 구간 선택 시 유도).
- **FR-008**: 로그인 진입 시 현재 보던 공유 페이지 위치가 기억되어, 로그인 성공 후 그 페이지로 자동 복귀해야 한다(이메일·카카오 모두).
- **FR-009**: 복귀 위치는 공유 페이지 범위로 제한되어, 조작된 임의 외부 주소로 이동하지 않아야 한다.
- **FR-010**: 로그인한 회원은 복귀 후 곧바로 반응·댓글을 남길 수 있어야 한다.

**댓글 고도화 (US3)**
- **FR-011**: 로그인 회원은 본문 구간을 선택해 이모지 반응(❤️ 👍 😮 😢 🔥 5종)을 남길 수 있어야 한다.
- **FR-012**: 같은 회원이 같은 구간에 같은 이모지를 다시 누르면 반응이 취소(토글)되어야 한다(회원·구간·이모지당 최대 1개).
- **FR-013**: 모든 열람자(비로그인 포함)는 구간별 반응 개수를 볼 수 있어야 하며, 개별 반응자 신원은 다른 열람자에게 공개되지 않아야 한다.
- **FR-014**: 로그인 회원은 구간을 선택하지 않고 작품 전체에 대한 "전체 의견"을 남길 수 있어야 한다.
- **FR-015**: 전체 의견과 글 댓글은 작가에게만 보여야 한다(다른 열람자 비공개 — 046 정책 유지). 반응만 공개 집계다.
- **FR-016**: 작가는 맥락 뷰(FR-001~)에서 구간별 반응 집계와 전체 의견을 함께 확인할 수 있어야 한다.

**종이 레이아웃 (US4)**
- **FR-017**: 공개 공유 열람 화면은 집필 화면과 같은 "종이 위의 글"(흰 종이 페이지 + 어두운 바깥 배경) 형태로 표시되어야 한다.
- **FR-018**: 동일 종이 스타일이 작가 피드백 맥락 뷰(US1)에도 적용되어야 한다.
- **FR-019**: 공유 열람 본문은 다크 모드에서도 읽기 좋게 표시되어야 한다(고정 밝은 색 → 테마 대응).

**회귀 보호 (전 스토리 공통)**
- **FR-020**: 046/047 기존 동작(공유 링크 생성·on/off·삭제·작품당 개수 제한, 공개 열람, 작가 전용 텍스트 댓글, 받은 피드백 읽음)이 회귀 없이 유지되어야 한다.

### Key Entities *(include if feature involves data)*

- **공유 스냅샷(Share Snapshot)**: 공유 시점에 동결된 글의 불변 사본. 피드백·반응 앵커의 기준. (046 기존)
- **피드백/댓글(Comment)**: 스냅샷의 특정 구간 또는 작품 전체(구간 미지정)에 대한 독자 의견. 작가 전용 비공개. (046 기존 + 앵커 선택적 확장)
- **반응(Reaction, 신규)**: 스냅샷의 특정 구간에 대한 회원의 이모지 표시. 회원·구간·이모지당 최대 1개. 개수는 공개 집계.
- **공유 링크(Share Link)**: 작품/시리즈당 여러 개 가능(1:N). 각 링크는 하나의 스냅샷을 가리킴. (046/047 기존)

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 작가가 URL 복사·붙여넣기 없이 공유 관리 화면에서 클릭 한 번으로 자기 공유 글 전문과 받은 피드백을 맥락과 함께 열람할 수 있다.
- **SC-002**: 비로그인 방문자가 공유 페이지에서 로그인한 뒤 추가 탐색 없이 같은 공유 페이지로 복귀한다(이메일·카카오 모두 복귀 성공).
- **SC-003**: 열람자가 글 없이 이모지 한 번으로 감상을 남길 수 있고, 반응 개수가 모든 열람자에게 보인다.
- **SC-004**: 공유 열람·작가 맥락 뷰가 집필 화면과 동일한 종이 질감으로 보이며 라이트·다크 모두 가독하다.
- **SC-005**: 기존 046/047 동작이 회귀 없이 유지된다(공유 생성·공개 열람·작가 전용 댓글·링크 삭제/개수 제한·피드백 읽음).

## Assumptions

- 반응자 개인 신원은 다른 열람자에게 비공개(개수만 공개)이며, 작가에게도 v1은 구간별 집계 개수 중심이다(개별 반응자 목록은 범위 밖).
- 작품에 링크(스냅샷)가 여럿일 때 맥락 뷰의 정본 진입점은 각 링크의 "받은 피드백" 버튼이다. 관리 화면 상단 요약은 개요로 유지한다.
- 로그인 복귀는 클라이언트에 복귀 위치를 임시 저장하는 방식을 전제한다(같은 브라우저 내 로그인 왕복에서 생존, 기존 로그인 화면 재사용).
- 046/047 자산(스냅샷·작가 전용 댓글·작가 인박스·헤더 공유 메뉴)을 재사용하며, 글 댓글의 작가 전용 비공개 정책은 불변이다.
- 이모지 종류는 5종(❤️ 👍 😮 😢 🔥)으로 고정한다(v1).

## Dependencies

- **046 공유하기**(공유 링크·불변 스냅샷·공개 열람·작가 전용 위치 지정 댓글)와 **047 공유 사용성 개선**(헤더 공유 메뉴·관리 허브·받은 피드백 읽음·링크 삭제/개수 제한) 위에 얹힌다.
