# Feature Specification: 마이페이지 계정 셸 재구성

**Feature Branch**: `037-mypage-account-shell`

**Created**: 2026-06-24

**Status**: Draft

**Input**: User description: "마이페이지를 좌측 사이드 메뉴를 가진 계정 셸로 재구성한다(036 마이페이지 확장). 중첩 라우트 /mypage/{섹션}. 사이드 메뉴 5개: 프로필(닉네임+계정정보), 환경설정(테마·기본 용지·일일 목표 — 기존 /settings 흡수), 계정 연결(카카오/이메일 연결 상태·관리), 문의·도움말, 회원 탈퇴(맨 아래 분리). /settings는 마이페이지로 리다이렉트, 헤더 nav '설정' 정리. 기존 기능 보존하며 위치만 이동. 계정 연결 액션 UI는 신규."

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 마이페이지가 좌측 메뉴를 가진 계정 셸이 된다 (Priority: P1)

사용자가 마이페이지에 들어가면 좌측에 사이드 메뉴(프로필 · 환경설정 · 계정 연결 · 문의·도움말 · 회원 탈퇴)가 있고, 메뉴를 고르면 우측에 해당 내용이 표시된다. 각 섹션은 고유한 주소를 가져 새로고침·뒤로가기·북마크가 자연스럽게 동작한다. "프로필"에는 기존 마이페이지 내용(닉네임 변경 + 계정정보)이, "문의·도움말"에는 문의 진입이, "회원 탈퇴"에는 기존 탈퇴 기능(확인 모달)이 들어간다.

**Why this priority**: 흩어져 있던 계정·신원 관련 기능을 한 곳(계정 셸)으로 모으는 것이 본 작업의 핵심 가치다. 셸과 네비게이션이 성립해야 나머지 섹션(환경설정 흡수·계정 연결)이 얹힐 자리가 생긴다.

**Independent Test**: 마이페이지에 진입해 사이드 메뉴 항목을 클릭하면 해당 섹션 주소로 바뀌고 내용이 전환되는지, 각 섹션 주소로 직접 들어가거나 새로고침해도 그 섹션이 열리는지 확인하면 단독 검증된다(환경설정 흡수·계정 연결 없이도).

**Acceptance Scenarios**:

1. **Given** 로그인 사용자가 마이페이지에 진입, **When** 좌측 사이드 메뉴를 보면, **Then** 프로필·환경설정·계정 연결·문의·도움말·회원 탈퇴 항목이 보이고 회원 탈퇴는 맨 아래에 시각적으로 분리(위험 톤)되어 있다.
2. **Given** 마이페이지에서, **When** "프로필"을 선택하면, **Then** 닉네임 변경과 계정정보(이메일·가입방식·가입일)가 표시된다.
3. **Given** 특정 섹션(예: 프로필)에 있을 때, **When** 브라우저를 새로고침하거나 그 섹션 주소를 직접 열면, **Then** 같은 섹션이 그대로 열린다.
4. **Given** 마이페이지에서, **When** "회원 탈퇴"를 선택하고 탈퇴를 진행하면, **Then** 기존과 동일한 확인 문구 모달을 거쳐 탈퇴된다(기능 보존).
5. **Given** 로그인하지 않은 사용자가, **When** 마이페이지의 임의 섹션 주소로 접근하면, **Then** 로그인 화면으로 보내진다.

---

### User Story 2 - 환경설정이 마이페이지로 흡수된다 (Priority: P2)

기존에 별도 화면(`/settings`)이던 앱 환경설정(테마 · 새 작품 기본 용지 · 일일 작업 목표)이 마이페이지의 "환경설정" 섹션으로 들어온다. 기존 설정 화면 주소로 들어오면 마이페이지의 환경설정으로 안내되고, 헤더에서 설정로 바로 가던 진입점은 마이페이지로 통합된다. 설정 항목의 동작은 그대로 보존된다.

**Why this priority**: 설정이 마이페이지와 따로 노는 상태를 없애 계정 셸을 완성한다. 다만 셸(P1)이 먼저 있어야 흡수할 자리가 생기므로 그다음이다.

**Independent Test**: 마이페이지 "환경설정"에서 테마·기본 용지·일일 목표를 바꿀 수 있고 그 값이 기존과 동일하게 반영되는지, 기존 `/settings` 주소로 들어오면 마이페이지 환경설정으로 안내되는지 확인하면 단독 검증된다.

**Acceptance Scenarios**:

1. **Given** 마이페이지에서, **When** "환경설정"을 선택하면, **Then** 테마·새 작품 기본 용지·일일 작업 목표를 기존과 동일하게 보고 바꿀 수 있다.
2. **Given** 사용자가 기존 설정 화면 주소(`/settings`)로 접근하면, **When** 페이지가 열릴 때, **Then** 마이페이지의 환경설정으로 안내된다(끊긴 링크가 되지 않는다).
3. **Given** 앱 헤더에서, **When** 계정·설정에 접근하려 하면, **Then** 마이페이지를 통한 단일 진입점으로 통합되어 있다(설정 중복 진입점 없음).

---

### User Story 3 - 계정 연결을 관리한다 (Priority: P3)

사용자가 "계정 연결" 섹션에서 자신의 로그인 수단(이메일/비밀번호, 카카오) 연결 상태를 확인하고, 아직 연결되지 않은 수단을 추가로 연결할 수 있다. 이메일로 가입한 사용자는 카카오를 추가 연결할 수 있고, 카카오로 가입한 사용자는 이메일·비밀번호를 추가 등록할 수 있다.

**Why this priority**: 신규 액션 UI라 구현 비중이 가장 크고, 셸(P1)·흡수(P2) 없이도 독립적으로 나중에 붙일 수 있어 마지막 우선순위다.

**Independent Test**: 계정 연결 섹션에서 현재 연결 상태가 정확히 표시되고, 미연결 수단에 대해 연결 동작을 시작할 수 있는지 확인하면 단독 검증된다.

**Acceptance Scenarios**:

1. **Given** 이메일로 가입(카카오 미연결)한 사용자가 계정 연결 섹션에 있을 때, **When** 화면을 보면, **Then** 이메일은 연결됨으로, 카카오는 미연결로 표시되고 카카오를 연결하는 동작을 시작할 수 있다.
2. **Given** 카카오로 가입(비밀번호 미설정)한 사용자가 계정 연결 섹션에 있을 때, **When** 비밀번호 추가 등록을 하면, **Then** 이후 이메일·비밀번호로도 로그인할 수 있는 상태가 된다.
3. **Given** 이미 두 수단이 모두 연결된 사용자, **When** 계정 연결 섹션을 보면, **Then** 둘 다 연결됨으로 표시되고 추가 연결 액션은 제공되지 않는다.

---

### Edge Cases

- **알 수 없는 섹션 주소**: 마이페이지의 존재하지 않는 섹션 주소로 접근하면 기본 섹션(프로필)으로 안내하거나 안전하게 처리한다.
- **계정 연결 해제**: 연결 해제(unlink)는 본 범위 밖이다(해당 기능이 시스템에 없음). 연결된 수단을 제거하는 액션은 제공하지 않는다.
- **카카오 추가 연결 도중 이탈/충돌**: 카카오 연결이 외부 인증 흐름을 거치므로, 이미 다른 계정에 연결된 카카오 등 충돌 시 사용자가 이해할 수 있는 안내로 끝난다(기존 연결 충돌 처리 보존).
- **회원 탈퇴 오작동 방지**: 회원 탈퇴는 확인 문구 입력 모달을 그대로 유지해 우발적 탈퇴를 막는다.
- **기존 설정 북마크**: 기존 `/settings`를 북마크한 사용자가 끊긴 화면을 보지 않도록 마이페이지 환경설정으로 이어준다.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 마이페이지는 좌측 사이드 메뉴(프로필 · 환경설정 · 계정 연결 · 문의·도움말 · 회원 탈퇴)와 우측 콘텐츠 영역으로 구성되어야 한다.
- **FR-002**: 각 사이드 메뉴 섹션은 고유한 주소를 가져 직접 접근·새로고침·뒤로가기·북마크가 동작해야 한다.
- **FR-003**: 현재 보고 있는 섹션이 사이드 메뉴에서 선택 상태로 강조되어야 한다.
- **FR-004**: "프로필" 섹션은 닉네임 변경(036)과 계정정보(이메일·가입방식·가입일)를 제공해야 한다.
- **FR-005**: "환경설정" 섹션은 기존 설정 화면의 테마·새 작품 기본 용지·일일 작업 목표를 동일하게 제공하고 그 동작을 보존해야 한다.
- **FR-006**: 기존 설정 화면 주소(`/settings`)로 접근하면 마이페이지의 환경설정으로 안내되어야 한다.
- **FR-007**: 앱 헤더의 설정 직접 진입점은 마이페이지를 통한 단일 진입점으로 통합되어야 한다(설정 중복 진입점 제거).
- **FR-008**: "계정 연결" 섹션은 이메일·카카오 각 로그인 수단의 연결 상태를 표시해야 한다.
- **FR-009**: "계정 연결" 섹션은 미연결 수단에 대해서만 연결 동작(이메일 가입자 → 카카오 추가 연결, 카카오 가입자 → 비밀번호 추가 등록)을 제공해야 한다. 이미 연결된 수단에는 추가 연결 액션을 노출하지 않는다.
- **FR-010**: 계정 연결 해제(unlink)는 제공하지 않는다(시스템 미지원).
- **FR-011**: "문의·도움말" 섹션은 문의 채널(`/contact`)로의 진입을 제공해야 한다.
- **FR-012**: "회원 탈퇴" 섹션은 기존 탈퇴 기능(확인 문구 입력 모달 포함)을 동일하게 제공하며, 사이드 메뉴 맨 아래에 위험 액션으로 시각적으로 분리되어야 한다.
- **FR-013**: 마이페이지의 모든 섹션은 로그인한 사용자만 접근할 수 있어야 하며, 미인증 접근은 로그인 화면으로 리다이렉트되어야 한다.
- **FR-014**: 카카오 연결 성공 후 사용자는 끊긴 화면이 아니라 적절한 화면(마이페이지 등)으로 안내되어야 한다.
- **FR-015**: 마이페이지의 존재하지 않는 섹션 주소로 접근하면 기본 섹션으로 안전하게 안내되어야 한다.

### Key Entities *(include if feature involves data)*

- **사용자 계정(User)**: 본 기능은 새로운 데이터를 만들지 않고 기존 사용자 정보를 표시·관리한다. 표시·판단에 쓰이는 속성 = 닉네임, 이메일, 가입방식, 가입일, **로그인 수단 연결 상태**(이메일/비밀번호 설정 여부, 카카오 연결 여부).
- **환경설정(Settings)**: 기존 사용자 환경설정(테마·기본 용지·일일 목표) — 위치만 마이페이지로 이동, 데이터·동작 불변.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 사용자는 마이페이지 한 곳에서 프로필·환경설정·계정 연결·문의·회원 탈퇴 5개 영역에 모두 도달할 수 있다(계정 관련 기능의 단일 집결지).
- **SC-002**: 마이페이지 각 섹션은 직접 주소 접근·새로고침 시 100% 해당 섹션이 열린다(딥링크 동작).
- **SC-003**: 기존 `/settings` 주소로 들어온 사용자는 끊긴 화면 없이 환경설정에 도달한다(깨진 링크 0건).
- **SC-004**: 환경설정 이동 후에도 테마·기본 용지·일일 목표 변경 동작이 기존과 동일하게 유지된다(기능 회귀 0건).
- **SC-005**: 계정 연결 섹션은 사용자의 실제 연결 상태를 정확히 반영하고, 미연결 수단에만 연결 액션을 노출한다.
- **SC-006**: 회원 탈퇴는 재배치 후에도 확인 모달을 거쳐야만 실행된다(우발 탈퇴 방지 유지).

## Assumptions

- **계정 연결 해제는 범위 밖**이다 — 시스템에 해제(unlink) 기능이 없으므로 연결 추가만 제공한다.
- **계정 연결 액션의 백엔드는 기존 흐름을 재사용**한다(카카오 추가 연결 / 비밀번호 추가 등록). 단, 화면이 "비밀번호 설정 여부·카카오 연결 여부"를 정확히 표시하려면 사용자 정보에 그 상태가 드러나야 하며, 이는 설계 단계에서 다룬다.
- **환경설정·회원 탈퇴·문의·닉네임 등 기존 기능은 동작 변경 없이 위치만 이동**한다. 본 작업은 정보 구조(IA) 재배치가 본질이며 새 비즈니스 로직은 계정 연결 액션 UI에 한정된다.
- **온보딩 투어(기존 기능)는 영향받지 않는다** — 투어는 백엔드 설정 저장 기능만 사용하고 프론트 설정 화면 주소에 의존하지 않는다.
- **마이페이지 섹션 구조는 고유 주소 기반**으로 하여 딥링크·새로고침을 지원한다(사용자 결정: 중첩 라우트 방식).
