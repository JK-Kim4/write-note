# Export 기능 구현 계획 (Round 3) — PDF · HWPX · DOCX

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 작가가 집필 화면에서 본 시각 형식 그대로(PDF는 픽셀 동일, 워드 문서는 형식+구조 보존) 선택한 챕터들을 PDF·HWPX·DOCX로 내보낸다.

**Architecture:** 공통 기반(export 설정창 + 챕터 본문 수집)을 먼저 세운 뒤, 포맷별로 추가한다. PDF는 집필실의 페이지 분할 CSS(`column-wrap`)와 인쇄(@page)의 정합이 검증된 적 없어 **PoC로 먼저 방법을 확정**한다. HWPX는 `hwpxlib` 실현가능성이 미확정이라 **스파이크로 먼저 검증**한다. PDF·DOCX는 프론트(브라우저)만으로, HWPX는 백엔드(JVM)에서 생성한다.

**Tech Stack:** Next.js 16 App Router + TypeScript + React 19 + TipTap(StarterKit) + vitest/RTL (프론트), Kotlin 2.2 + Spring Boot 4 + `hwpxlib` (백엔드), JS `docx` 라이브러리.

```ts
// collectChapters.ts
import type { DocumentResponse } from "@/types/api";

/** export 합본 한 챕터 — 본문 변환의 입력. */
export type CollectedChapter = { id: number; title: string; bodyJson: string };

/**
 * 선택한 챕터 id들을 [orderedIds] 순서 그대로 본문과 함께 모은다.
 * @param fetchDoc 시스템 경계(HTTP) 주입 — 기본은 호출부에서 getDocument 전달. 테스트는 mock.
 */
export async function collectChapters(
    orderedIds: number[],
    fetchDoc: (id: number) => Promise<DocumentResponse>,
): Promise<CollectedChapter[]> {
    const docs = await Promise.all(orderedIds.map((id) => fetchDoc(id)));
    return docs.map((d) => ({ id: d.id, title: d.title, bodyJson: d.bodyJson }));
}
```

```tsx
// ExportDialog.tsx
"use client";

import { useState } from "react";
import type { ChapterMetaResponse } from "@/types/api";
import type { PaperSize } from "@/components/editor/pageLayout";

export type PdfExportRequest = { orderedIds: number[]; lined: boolean };

type ExportDialogProps = {
    open: boolean;
    chapters: ChapterMetaResponse[];
    paperSize: PaperSize;
    onExportPdf: (req: PdfExportRequest) => void;
    onClose: () => void;
};

type Format = "pdf" | "hwpx" | "docx";

export function ExportDialog({ open, chapters, paperSize, onExportPdf, onClose }: ExportDialogProps) {
    // 선택 + 순서를 하나의 ordered 배열로 관리(초기 = sortOrder 순 전체 선택).
    const [order, setOrder] = useState<number[]>(() => chapters.map((c) => c.id));
    const [selected, setSelected] = useState<Set<number>>(() => new Set(chapters.map((c) => c.id)));
    const [format, setFormat] = useState<Format>("pdf");
    const [lined, setLined] = useState(false);

    if (!open) return null;

    const toggle = (id: number) =>
        setSelected((prev) => {
            const next = new Set(prev);
            next.has(id) ? next.delete(id) : next.add(id);
            return next;
        });

    const move = (id: number, dir: -1 | 1) =>
        setOrder((prev) => {
            const i = prev.indexOf(id);
            const j = i + dir;
            if (j < 0 || j >= prev.length) return prev;
            const next = [...prev];
            [next[i], next[j]] = [next[j], next[i]];
            return next;
        });

    const orderedSelected = order.filter((id) => selected.has(id));
    const canExport = orderedSelected.length > 0;
    const byId = new Map(chapters.map((c) => [c.id, c]));

    const handleExport = () => {
        if (!canExport) return;
        if (format === "pdf") onExportPdf({ orderedIds: orderedSelected, lined });
        // hwpx/docx 는 Phase 4·5 에서 활성
    };

    return (
        <div role="dialog" aria-label="내보내기" className="export-dialog">
            <div className="export-dialog__formats">
                {(["pdf", "hwpx", "docx"] as const).map((f) => (
                    <button
                        key={f}
                        type="button"
                        aria-pressed={format === f}
                        disabled={f !== "pdf"} // Phase 0: PDF 만 활성
                        onClick={() => setFormat(f)}
                    >
                        {f.toUpperCase()}
                    </button>
                ))}
            </div>

            <ul className="export-dialog__chapters">
                {order.map((id) => {
                    const c = byId.get(id);
                    if (!c) return null;
                    return (
                        <li key={id}>
                            <label>
                                <input
                                    type="checkbox"
                                    checked={selected.has(id)}
                                    onChange={() => toggle(id)}
                                    aria-label={`${c.title} 포함`}
                                />
                                {c.title}
                            </label>
                            <span>{c.wordCount.toLocaleString()}자</span>
                            <button type="button" aria-label={`${c.title} 위로`} onClick={() => move(id, -1)}>⌃</button>
                            <button type="button" aria-label={`${c.title} 아래로`} onClick={() => move(id, 1)}>⌄</button>
                        </li>
                    );
                })}
            </ul>

            <label className="export-dialog__option">
                <input type="checkbox" checked={lined} onChange={(e) => setLined(e.target.checked)} />
                줄노트 줄 포함
            </label>
            <p className="export-dialog__paper">용지 {paperSize} (작품 설정)</p>

            <button type="button" onClick={handleExport} disabled={!canExport}>내보내기</button>
            <button type="button" onClick={onClose}>닫기</button>
        </div>
    );
}
```

## Phase 2 — PDF 인쇄 정합 PoC (#51)

**목표:** 집필실의 페이지 분할(`column-width`/`column-height`/`column-wrap` — Chrome 145+ 신형 CSS, `paper-editor.css:134-139`)을 **인쇄/PDF로 충실히 출력하는 방법을 확정**한다. 화면 분할 메커니즘과 인쇄(@page) 페이지네이션은 별개라, 그대로 인쇄했을 때 분할 위치가 보존되는지 검증이 필요하다.

- [ ] **Step 1: 현황 인쇄 실험** — 임시 PoC 라우트(`app/poc/export-print/page.tsx` 등)에 `.paper-editor` 마크업으로 2~3장 분량 본문을 렌더하고 `window.print()` → "PDF로 저장". 화면의 장 구분이 PDF 페이지로 보존되는지 관찰.
- [ ] **Step 2: 방법 후보 평가** — 다음 후보를 실측 비교:
  - (a) **화면 CSS 그대로 + `@media print`** — column-wrap 레이아웃이 인쇄에서 분할되는가

- [ ] **Step 3: 용지·줄노트 반영 확인** — 선택한 방법에서 ① 용지 크기(A4/B4/A3/A2 — `paperGeometry(size)`) ② 줄노트(`.sheet--lined`) ON/OFF ③ 챕터 제목 표시 가 인쇄에 반영되는지 확인.
- [ ] **Step 4: 한국어 폰트 확인** — 명조체(`--pe-serif`)가 인쇄/PDF에 정상 임베딩·표시되는지(한국어 영역 검증 cadence, typescript code-quality HARD-GATE).
