# 프로젝트 대시보드 디자인 개선

**날짜:** 2026-05-31
**상태:** 설계 확정
**범위:** 프론트엔드 프로젝트 화면 1차 디자인 개선

## 배경

`write-note`는 “컨텍스트가 안 죽는 작가용 작업공간”이다. 현재 프로젝트 상세 화면은 `MetaCard`와 관리 버튼 중심으로 구성되어 있어, 사용자가 프로젝트에 다시 들어왔을 때 “무엇을 이어 써야 하는지”보다 CRUD 관리 기능이 먼저 보인다.

이번 1차 개선은 백엔드 기능을 추가하지 않고, 이미 있는 프론트/API 데이터만 사용해 프로젝트 화면의 정보 위계를 재정렬한다. 세션 노트 기능은 Week 5 범위로 남겨두고, 화면에는 추후 연결 가능한 슬롯만 둔다.

## 목표

프로젝트 화면의 첫 인상을 “관리 페이지”에서 “이 작품에 다시 들어가는 입구”로 바꾼다.

핵심 우선순위:

1. **작업 재진입:** 이어 쓰기, 진행률, 마지막 저장 시각이 먼저 보인다.
2. **작품 맥락:** 시놉시스, 톤, 세계관, 등장인물이 대시보드 카드로 정리된다.
3. **관리 액션:** 편집, 등장인물 관리, 보관, 삭제는 하단 보조 영역으로 낮춘다.

## 사용자 결정

- 프로젝트 화면 1차 개선만 진행한다.
- 방향은 `A. 재진입 허브`를 기본으로 한다.
- 최상단 재진입 카드는 `지난 세션 메모 + 이어쓰기 상태` 조합이 맞다.
- 실제 구현은 프론트 디자인 개선에 한정한다.
- `SessionNote` 백엔드/API는 이번 범위에 포함하지 않는다.
- 재진입 카드 아래 구성은 `대시보드형`이 가장 적합하다.

## 화면 구조

### 1. 상단 헤더

프로젝트명과 기본 메타를 노출한다.

- 제목
- 장르
- 마지막 저장 시각
- 프로젝트에 연결된 최근 메모 수 또는 최근 메모 요약
- `이어 쓰기` CTA

`이어 쓰기`는 `/write?projectId={id}`로 이동한다.

### 2. 재진입 히어로

화면에서 가장 강한 영역이다.

이번 범위에서 실제 데이터로 표시할 항목:

- 현재 자수: `DocumentResponse.wordCount`
- 목표 분량: `ProjectResponse.targetLength`
- 진행률: `wordCount / targetLength`
- 마지막 저장 시각: `DocumentResponse.updatedAt`

이번 범위에서 슬롯만 둘 항목:

- 지난 세션 메모

세션 노트 데이터가 없을 때는 “아직 지난 세션 메모가 없습니다”처럼 빈 상태를 보여준다. Week 5에서 `SessionNote`가 들어오면 이 슬롯만 실제 데이터로 교체한다.

### 3. 보조 알림

재진입 히어로 오른쪽 또는 바로 아래에 작은 알림 카드를 둔다.

- 미분류 메모 개수
- 프로젝트에 이미 연결된 최근 메모 요약
- 오늘 잡아둘 것 빈 슬롯

이번 범위에서는 `listMemos({ unclassified: true, size: 1 })`로 전체 inbox의 미분류 총량만 표시할 수 있다. “이 작품 관련 후보”처럼 추천 로직이 필요한 표현은 만들지 않는다.

### 4. 대시보드 카드

하단에는 2열 카드 그리드를 둔다.

- 시놉시스
- 톤·문체
- 등장인물
- 최근 메모
- 세계관 메모
- 관리

빈 값은 “비어 있음”을 노출하되, 화면이 실패처럼 보이지 않게 낮은 대비의 empty state로 처리한다.

### 5. 관리 액션

현재 상세 페이지에 있던 관리 액션은 하단 `관리` 카드로 이동한다.

- 메타 편집
- 등장인물 관리
- 보관 / 보관 해제
- 삭제

삭제 확인 dialog와 mutation 흐름은 기존 동작을 유지한다.

## 데이터 소스

신규 백엔드/API는 만들지 않는다.

- `getProject(id)`
- `getProjectDocument(id)`
- `listCharacters(id, { page: 0, size: 5 })`
- `listMemos({ projectId: id, page: 0, size: 2, sort: "capturedAt,desc" })`
- `listMemos({ unclassified: true, page: 0, size: 1 })`

문서 조회가 실패해도 프로젝트 상세 전체가 깨지지 않아야 한다. 진행률 영역만 빈 상태로 처리한다.

## 컴포넌트 설계

새 컴포넌트:

- `ProjectDashboard`
- `ContinueProjectHero`
- `ProjectInfoCard`
- `ProjectManagementCard`

기존 컴포넌트:

- `MetaCard`는 삭제하지 않는다.
- 작성 화면 사이드패널 등 좁은 영역에서 계속 사용할 수 있다.
- 필요하면 `ProjectInfoCard` 내부 구현에 일부 표시 로직을 옮긴다.

`/projects/{id}/page.tsx`는 데이터 조회와 mutation orchestration만 맡고, 실제 표시 구조는 `ProjectDashboard`로 넘긴다.

## 스타일 원칙

- 기존 디자인 토큰을 유지한다.
- 그림자는 쓰지 않는다.
- elevation은 `surface` 차이와 `hairline border`로만 표현한다.
- 카드 안에 카드를 중첩하지 않는다.
- CTA는 `이어 쓰기` 하나만 가장 강하게 보인다.
- 보관/삭제 같은 관리 액션은 시각적으로 낮춘다.

## 테스트와 검증

기존 테스트가 있으면 `/projects/[id]/page.test.tsx`를 대시보드 구조에 맞게 갱신한다.

검증 항목:

- 프로젝트 제목이 표시된다.
- `이어 쓰기` CTA가 `/write?projectId={id}`로 연결된다.
- 목표 분량이 있을 때 진행률과 자수가 표시된다.
- 목표 분량이 없을 때도 화면이 깨지지 않는다.
- 문서 조회 실패 시 프로젝트 정보 카드는 계속 보인다.
- 등장인물/최근 메모가 없으면 빈 상태가 표시된다.
- 보관/보관 해제/삭제 흐름은 기존 동작을 유지한다.

검증 명령:

```bash
cd frontend
pnpm test
pnpm lint
pnpm typecheck
```
