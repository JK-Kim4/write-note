# B형 대시보드(작가 홈) 설계

- **작성일:** 2026-06-13
- **상태:** 설계 승인됨 (브레인스토밍 → 본 문서 → 구현 계획)
- **브랜치(예정):** `021-b-dashboard` (또는 후속 명명)
- **선행/맥락:** 018 대시보드 허브(A형 `/`)는 구현 완료. 020에서 B형(`app/b/...`)이 기본 디자인으로 확정됐으나 **B형에는 대시보드/홈이 없음**(`/b` = 작품 벽뿐). 본 작업은 B형에 작가 홈(대시보드)을 추가한다.
- **다음 진입 보류:** Round 2.5 챕터(본 작업과 별개 트랙, 본 작업 후 재개).

## 1. 배경 · 목적

작가가 B형 홈을 열었을 때 **마지막 작업으로 한 번에 복귀하고 맥락을 회복**하는 것이 1순위. 제품 본질("컨텍스트가 안 죽는 작업공간")과 정합한다.

근거(deep research, 2026-06-13, sonnet 전용·축소판):
- **재진입 마찰 최소화** — 최근 작업·현재 위치를 전면에.
- **게이미피케이션 절제** — streak(연속일)은 "수단-목적 전도"(글쓰기보다 streak 유지가 목적이 됨)를 일으킴(Journal of Consumer Research 2023). 과도한 게이미피케이션은 내재적 동기를 약화. → 목표·배지·streak은 배제하거나 opt-in으로 절제.
- 단, 리서치 축소판이라 대시보드 정보 *우선순위*의 강한 외부 증거는 없음. 정보 위계는 위 **원칙 + 제품 본질 + 검증된 내부 자산(018)** 으로 결정.

## 2. 정보 위계

1. **이어서 쓰기**(최상단, 최우선) — 재진입.
2. 작품 미니카드(전환) · 집필 리듬(습관, 표시만).
3. 곁쪽지(컨텍스트 영속, 우측 상시) · 빠른 메모.
4. 진척·목표 게이지 — **opt-in, 기본 꺼짐**(목표 설정한 작품만).

## 3. 레이아웃 — C형(2컬럼 + 곁쪽지 사이드)

```
┌──────────────────────────────────────────────┐
│  이어서 쓰기 (풀폭)                            │
│  제목·N장 / "마지막 문장" / 다음 장면 ·         │
│  저장시각 · 글자수 / [이어 쓰기 →]              │
├───────────────────────────┬──────────────────┤
│ 좌 컬럼 (넓게)            │ 우 컬럼           │
│  · 작품 미니카드 그리드   │  곁쪽지 패널 상시  │
│  · 집필 리듬 카드         │  최근 3장 + 빠른메모│
└───────────────────────────┴──────────────────┘
```

- **반응형:** 880px 미만에서 우 컬럼(곁쪽지)을 020에서 만든 drawer 토글 패턴으로 접음(백드롭·ESC). 기존 자산 재사용.
- **스킨:** B형(흰 시트 · gray-200 보더 · 인디고 액센트). A형 웜페이퍼 미적용.

## 4. 진입 구조 · 라우팅

A형(`/`=대시보드, `/library`=작품 벽)과 **대칭**.

| 항목 | 변경 |
|---|---|
| `/b` | 작품 벽 → **대시보드(작가 홈)** |
| 작품 벽 | `/b` → **`/b/library`** 로 이동 |
| B형 네비(`app/b/layout.tsx`) | 최상단에 **"홈"(`/b`)** 신설, 기존 "작품"은 `/b/library` 로 |
| 로그인 후 B형 기본 진입 | `/b`(대시보드) — 현행 유지(이미 `/b`로 push) |

**갱신 대상 링크(plan에서 세밀 grep):** `app/b/layout.tsx`(네비 22, 로고 81, `router.push("/b")` 123·186·238), `app/b/page.tsx`(replaceState 69), `app/b/works/[id]/page.tsx`(117·190·377 작품 벽 복귀), `LoginForm.test.tsx`(64, 로그인 후 `/b`는 대시보드로 의미 변경 — 테스트 의도 점검). 작품 벽 복귀 의도의 `/b`는 `/b/library`로 교체.

## 5. 데이터 · 백엔드 — **변경 0**

모든 블록이 기존 A/B 공유 훅·엔드포인트로 충족.

| 블록 | 데이터 출처 | 상태 |
|---|---|---|
| 이어서 쓰기 | `GET /api/projects/cards` (최신 `documentUpdatedAt`) | 기존 |
| 작품 미니카드 | `GET /api/projects/cards` | 기존 |
| 집필 리듬 | `GET /api/work-sessions/total?from=&to=` (요일별 7회) | 기존 |
| 곁쪽지 | `GET /api/memos` | 기존 |
| **opt-in 게이지** | `ProjectCardResponse.targetLength` + `wordCount` (이미 응답에 포함) | **기존** |

- **마이그레이션 신규 없음.** opt-in 목표는 기존 `projects.target_length`(V5, "목표 분량", nullable, CHECK 1~1억) 재사용. `targetLength`가 null이면 게이지 미표시 = opt-in의 자연 구현.
- 목표 분량 입력 UI: `CreateProjectRequest`·`UpdateProjectRequest`에 `targetLength` 이미 존재. **B형 새작품/작품편집 화면에 입력이 노출되는지만 plan에서 확인**(A형 `/projects/[id]/edit`에는 있음). 누락 시 B형 폼에 필드 추가(프론트 한정).

## 6. opt-in 게이지 배치

- 목표(`targetLength`) **설정한 작품만** 가는 달성률 바(`wordCount / targetLength`) — 이어쓰기 타일 + 해당 작품 미니카드.
- 미설정 작품은 현재 글자수만 표시, 게이지 없음.

## 7. 게이미피케이션 절제 가드 (설계 불변 원칙)

- streak(연속일) · 배지 · 등급 · 달성 축하 애니메이션 **도입 안 함**.
- 리듬 카드 = 요일 막대 · 작품별 누적 **표시만**. 목표선 · 달성 강조 없음.
- 게이지 = opt-in · 기본 꺼짐.
- → 018/PRODUCT.md 게이미피케이션 배제 원칙 및 리서치 경고와 정합.

## 8. 컴포넌트 전략

A형 `ResumeCard` · `RhythmCard` · `WorkMiniCard`(`components/dashboard/`)의 **데이터 훅(`useProjectCards`·`useWeeklyByDay`·`useInboxMemos`)·파생 로직(`lib/dashboardView.ts`)은 공유**, **표현은 B 스킨**. 공유 컴포넌트에 variant prop을 줄지 B 전용 컴포넌트를 신설할지는 plan에서 코드 보고 확정(A형 표현이 desktop-app.css·Rail 셸에 묶여 있으면 B 전용 신설이 안전).

## 9. 범위 밖 (Non-goals)

- A형 대시보드(`/`) 변경 — 동결.
- 챕터 기능(Round 2.5) — 별도 트랙.
- 백엔드 신규 엔드포인트 · 마이그레이션 — 없음.
- streak/배지/목표 게이미피케이션 — 영구 배제.

## 10. 검증 기준 (성공 조건)

1. 로그인한 B형 사용자가 `/b`에서 대시보드를 보고, [이어 쓰기]로 최근작 집필실에 진입.
2. 작품 벽이 `/b/library`에서 정상 동작, 네비 "홈"/"작품" 전환 정상.
3. 곁쪽지 패널이 우측에 보이고 빠른 메모로 캡처 가능, 880px 미만에서 drawer 토글.
4. `targetLength` 설정 작품만 게이지 표시, 미설정은 글자수만.
5. 게이트 GREEN: 프론트 `pnpm`(typecheck·lint·vitest·build). 백엔드 변경 없으므로 백엔드 게이트는 회귀 확인만.
6. dogfooding: 한글 IME·시각·redirect(라우트 이동 후 깨진 링크 0).
