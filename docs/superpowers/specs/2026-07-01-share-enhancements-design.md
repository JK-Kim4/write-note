| # | 사용자 요구 | 현재 코드 상태(근거) | 성격 |
|---|---|---|---|
| 1 | 홈에서 내 공유 페이지 **전문 + 받은 피드백을 맥락으로** 확인 | 공유 관리 화면(`ShareLinkManager`)에 공유 링크를 **여는 버튼 없음**(URL 복사만, `ShareLinkManager.tsx:116-130`). 받은 피드백은 **모달 조각**(`AuthorCommentInbox`)뿐, 전문 위 맥락 뷰 없음(클릭 시 편집화면 `/works/{id}`로 감) | 신규 뷰 + 소규모 BE |

## Topic 1 · 작가용 피드백 맥락 뷰

### 결정
공유 관리 화면에서 **한 공유 링크(스냅샷)의 전문 + 그 위에 얹힌 받은 피드백**을 한 화면에서 보는 작가 전용 뷰. 레이아웃 = **안 B(우측 피드백 패널)**. (호버 말풍선·여백 말풍선 기각 — 사용성 이유.)

### 하드 제약 — 스냅샷 단위
댓글·반응은 특정 불변 스냅샷(`share_snapshot`)에 앵커된다. 1:N(작품/시리즈당 링크 여러 개)이면 스냅샷 텍스트가 서로 달라 **여러 스냅샷 피드백을 한 본문 위에 합칠 수 없다.** 따라서 이 뷰는 **링크(스냅샷) 하나 단위**이며, 진입점은 각 링크의 "받은 피드백 N" 버튼.

### 기능 요구사항
- **FR-1.1** 공유 관리 화면 각 링크(작품 링크·시리즈 링크의 개별 공개작품)에 "전문 보기/받은 피드백 N" → 그 스냅샷의 맥락 뷰 진입.
- **FR-1.2** 맥락 뷰 = 스냅샷 전문(읽기 전용, Topic 4 종이 스타일) + 우측 피드백 패널. 댓글 달린 구간은 **하이라이트 항상 표시**.
- **FR-1.3** 패널 항목(독자 닉네임·인용·내용·시각·안읽음 점) 클릭 → 본문이 해당 구간으로 스크롤 + 하이라이트 반짝.
- **FR-1.4** 패널은 안 읽은 것 우선, 뷰 진입 시 그 스냅샷 피드백 **읽음 처리**(현재 "열면 읽음" 계승).
- **FR-1.5** 반응(Topic 3)이 있으면 패널/본문에 **반응 집계**도 함께 표기(예: 구간에 ❤️3 👍2).
- **FR-1.6** 이 뷰는 **작가 본인만**(스냅샷 소유 = `share_link.owner_id == 요청자`) 접근. 타인 403.

### 백엔드 함의
현재 조회는 (a) `listMineForSharedWork`=요청자 본인 댓글만, (b) 인박스 `listForAuthor`=projectId **스냅샷 횡단** 집계 — 둘 다 "이 스냅샷의 전체 독자 댓글(작가 권한)"이 아니다. → **스냅샷 스코프 작가용 조회 추가** 필요(신규 endpoint `GET /api/share-snapshots/{id}/comments` 또는 인박스 응답에 `shareSnapshotId` 포함 후 FE 필터). 소유 검증은 `share_link.owner_id` 기준.

### 열린 세부(spec에서 확정)
- 관리 화면 상단 projectId 단위 "받은 피드백" 요약과 링크 단위 맥락 뷰의 진입 매핑(작품에 링크 여럿일 때 어느 스냅샷으로 갈지). 기본안: 링크 단위 버튼이 정본 진입, 상단 요약은 개요로 유지.

## Topic 2 · 비로그인 방문자 로그인

### 결정
공개 공유 페이지에 로그인 진입점 신설 + **로그인 방식 = 안 1(로그인 페이지로 이동, `/auth/login` 재사용)**. 로그인 후 **보던 공유 페이지로 복귀**. (모달 기각 — 카카오가 어차피 전체 리다이렉트라 이점 반감 + 로그인 UI 이중관리.)

### 기능 요구사항
- **FR-2.1** 진입점 3개: 공개 페이지 헤더 우상단 "로그인" 버튼 / 본문 하단 CTA("로그인하고 남기기", 현재 비클릭 안내문 대체) / 비로그인 상태 드래그 선택 시 "로그인" 유도.
- **FR-2.2** 진입점 클릭 → 현재 공유 페이지 경로를 **복귀지로 저장** 후 `/auth/login`으로 이동.
- **FR-2.3** 로그인 성공(이메일·카카오 공통) 후 저장된 복귀지로 **자동 이동**. 복귀지 없으면 기존 홈 흐름 유지.
- **FR-2.4** 복귀지는 **`/shared/` prefix로 검증**해 open-redirect 방지.

### 백엔드 함의
BE 변경 **없음**(기존 `/auth/login`·OAuth 흐름 재사용). 복귀는 **FE-only** — 로그인 직전 `localStorage`에 복귀지 저장(같은 origin이라 카카오 왕복에도 생존), 로그인 후 도착지(`/entering` 또는 홈)에서 소비해 `router.replace`. (현재 이메일→`/entering`→홈, 카카오→홈 하드코딩이라 `returnTo` 파라미터 미지원 → localStorage 방식이 최소 변경.)

## Topic 3 · 댓글 고도화 (이모지 반응 + 전체 의견)

### 결정
- **이모지 반응 5종 [❤️ 👍 😮 😢 🔥] — 공개 집계**: 모든 독자가 개수를 봄.
- **전체 의견(구간 미지정) — 작가 전용 비공개**: 앵커 없이 작품 전체 감상.
- 채택 안 함(범위 밖): 작가 답글/양방향 대화, 서식 있는 입력, 독자 본인 댓글 수정.

### 모델 변화 주의
반응은 **공개**(이 서비스 최초로 독자끼리 서로의 흔적=반응 개수를 봄)지만, **글 댓글·전체 의견은 여전히 작가 전용 비공개**. 혼합 모델.

### 기능 요구사항 — 이모지 반응
- **FR-3.1** 로그인 회원이 본문 구간 선택 → 툴바에서 이모지 클릭 = 즉시 반응(그 구간 앵커에 기록). 로그인 필요(비로그인은 Topic 2 유도).
- **FR-3.2** 반응은 **켜기/끄기 토글**(같은 회원·같은 구간·같은 이모지 1개). 다른 이모지는 별개.
- **FR-3.3** **모든 열람자(비로그인 포함)가 구간별 반응 개수**를 봄(예: ❤️ 5). 개수만 표시, **누가 눌렀는지는 독자에게 비공개**(기본값).
- **FR-3.4** 작가 맥락 뷰(Topic 1)에는 구간별 반응 집계 표기.

### 기능 요구사항 — 전체 의견
- **FR-3.5** 공개 페이지 하단 "작품 전체에 한마디" 입력 → 구간 앵커 없이 작품(스냅샷) 전체에 대한 댓글 저장. **작가 전용 비공개**(기존 댓글 가시성과 동일).
- **FR-3.6** 전체 의견도 max 2000자·`@NotBlank`(기존 계승). 작가 인박스/맥락 뷰에 "전체 의견" 구획으로 노출.

### 백엔드 함의
- **반응:** 신규 도메인 — `share_reaction(id, share_snapshot_id FK, anchor_block_index, anchor_start, anchor_length, emoji, reactor_id FK users, created_at)` + `UNIQUE(share_snapshot_id, anchor_block_index, anchor_start, anchor_length, emoji, reactor_id)`. 마이그레이션 신규 V(diff). endpoint: `POST/DELETE` 반응(회원, `/api/shared/**` optional auth + 미회원 거부) / 집계는 공개 read(공유 작품 응답에 구간별 `{emoji: count, myReactions:[...]}` 포함). 앵커 검증 = 기존 `AnchorValidator` 재사용(스냅샷 평탄화 정합, 룰 §32).
- **전체 의견:** `share_comment` 앵커 컬럼(`anchor_block_index/start/length`)을 **nullable**로(= 전체 의견) 또는 스코프 플래그 추가. `CreateCommentRequest` 앵커 선택적. 가시성 로직 불변.
- 신규 에러코드 최소화(미회원 반응 = 기존 `COMMENT_UNAUTHENTICATED` 계열 재사용 검토).

### 결정
공유 열람 화면을 집필 화면처럼 **종이(페이지) 위에 글** — **안 1(흰 종이)**. 종이 = `--w-ms-page`(#ffffff), 바깥 = `--w-ms-outer`(#eceae4), 은은한 그림자·둥근 모서리·넉넉한 여백.
