# 타임워치(Timewatch) — 집필 시간 사용자 제어 측정 설계

- 작성일: 2026-06-24
- 상태: 설계 확정 대기 → 구현 계획 전 단계
- 목업: `docs/research/2026-06-23-timewatch-mockup.html` (스타일 A 확정)

## 1. 배경 / 목적

지금 집필실은 **머무는 동안 시간이 자동 측정**된다(진입=세션 시작, 이탈=세션 종료). 문제:

- 사용자가 시간이 측정되고 있는지 **인지하기 어렵다**.
- 자리를 비워도(딴짓·자리비움) 측정이 계속돼 **실제 집필 시간과 어긋난다**.

해결: **사용자가 직접 제어하는 스톱워치("타임워치")**를 도입해 측정 책임을 사용자에게 넘긴다. 눈에 보이는 카운트업 + 시작/일시정지/종료로 **사용성**과 **재미 포인트**를 만든다.

핵심 전환:

| 구분 | 변경 전 | 변경 후 |
|---|---|---|
| 작업 시간 측정 | 집필실 머무름 = 자동 | **타임워치로 잰 시간만** |
| 측정 책임 | 시스템 | **사용자** |
| 안 누르면 | (항상 측정) | **0초 기록** |

## 2. UI (스타일 A)

### 2-1. 위치 / 형태
- 집필실 **우패널 최상단의 독립 카드**. 아래 메모/인물 패널과 **시각적으로 분리**(별도 흰 카드 + 간격).
- 큰 시간 표시(`HH:MM:SS`, tabular-nums) + 상태 점/라벨 + 상태별 버튼.
- 좁은 폭(<880px) 우측 drawer 안에서도 동일하게 최상단에 노출.

### 2-2. 상태 3종
| 상태 | 표시 | 버튼 |
|---|---|---|
| 대기(idle) | `00:00:00`(회색) · 점 회색 "집필 시간" | **[▶ 시작]** |
| 집필 중(running) | 카운트업 · **초록 점** "집필 중" | **[⏸ 일시정지] [■ 집필 종료]** |
| 일시정지(paused) | 시간 고정(누적 유지) · 점 "일시정지" | **[▶ 다시 시작] [■ 집필 종료]** |

- 카운트업 표시값 = 이번 방문에서 측정한 **누적**(일시정지·재개를 거쳐도 이어짐). 클라이언트 로컬 상태이며 서버에 영속하지 않는다. **재진입 시 0부터** 시작(그전 기록은 서버에 이미 저장됨).

### 2-3. "집필 종료" 모달
- 제목 "집필을 마칠까요?" + "측정한 시간 N분 N초가 기록됩니다 …(선택 메모)".
- 선택 메모 textarea.
- 버튼 2개를 모달 폭 꽉 차게 가운데: **[메모 없이 종료] [기록하고 종료]**.
- **우상단 ✕** (+ 바깥 클릭 · ESC) → 모달만 닫고 **집필 화면 복귀, 타이머 계속**(집필 중/일시정지 상태 유지). 즉 모달이 종료 **확인 단계** 역할.

## 3. 동작 ↔ 백엔드 매핑

기존 `work-sessions` API를 그대로 재사용한다(엔티티 = `startedAt`/`endedAt` 타임스탬프, 종료 세션의 `endedAt-startedAt` 합이 총 작업시간). **일시정지=세션 종료 / 재개=새 세션**이 자연스러운 매핑이라 스키마 변경이 없다.

| 사용자 액션 | 호출 | 결과 |
|---|---|---|
| 시작 | `POST …/work-sessions/start` | 세션 1개 열림 |
| 일시정지 | `POST …/work-sessions/end` | 구간 종료(경과 기록). 화면 누적은 클라가 유지 |
| 다시 시작 | `start` | 새 구간. 화면 누적 이어감 |
| 집필 종료(메모 없이) | `end` → `/library` 이동 | 시간만 기록(로그 없음) |
| 집필 종료(기록하고) | `end-with-log(body)` → `/library` | 시간 기록 + 집필 기록 로그 생성 |
| 멈추지 않고 이탈/탭닫기/새로고침(집필 중) | unmount=`end`, pagehide=`endBeacon` | 그때까지 시간 **자동 기록**(메모 없음) |

- **일시정지 동안 시간은 차감**된다(세션이 닫혀 있으므로). 측정 책임 전가의 핵심.
- 작업 시간 집계(홈 "오늘 작업시간" 게이지·주간 리듬·작품별 누적, `/기록` 통계)는 **종료 세션 합** = 타임워치로 잰 시간만 반영.

### 3-1. 30초 미만 폐기 규칙 제거 (백엔드 소폭 변경)
- 현재 `end`는 **30초 미만 세션을 폐기**한다(자동측정 잡음 방지용). 자동측정이 사라지면 이 규칙은 불필요하고 오히려 사용자가 잰 짧은 구간을 버린다.
- **타임워치 도입 후: 30초 제한 없이 측정된 시간은 전부 저장.** `WorkSessionService.end`의 30초 폐기 분기를 제거(또는 무력화)한다.
- 이미 `end-with-log`는 짧아도 보존이라 변경 없음.

### 3-2. 메모 nullable
- "메모 없이 종료" = `end`(로그 미생성, 시간만 기록). "기록하고 종료" = `end-with-log(body)`.
- 따라서 메모는 사실상 선택이며 **백엔드 계약 변경 불필요**(엔드포인트 선택으로 처리). `/기록`의 시간 통계는 메모 유무와 무관하게 반영, 로그 목록엔 메모 남긴 것만.

## 4. 컴포넌트 / 책임

- **`Timewatch` (신규, client)** — 우패널 최상단 카드. 상태(idle/running/paused) + 누적 경과 표시 + 버튼. `useTimewatch` 훅 사용.
- **`useTimewatch(projectId)` (신규 훅)** — 클라 상태(상태값·세그먼트 시작시각·누적 ms) + `sessions.start/end/endWithLog` 호출 + 1초 틱 표시. 기존 `useWorkSession`을 대체/재구성.
  - `start()` / `pause()` / `resume()` / `stop(memo?)` 노출. `stop`은 종료 후 네비게이션은 호출 측(셸)이 담당.
  - 이탈(unmount)·pagehide 시 **열린 세션이 있으면** `end`/`endBeacon`(없으면 no-op — `end`는 열린 세션 없으면 null).
- **`BStudioShell`** — 우패널 상단에 `Timewatch` 결선. 좌패널 하단 **"작업 종료" 버튼 + 모달 제거**. "집필 종료" 모달은 타임워치/셸 쪽으로 이동(네비 = 기존 `handleNavigateBack`의 /library 경로 재사용, flush-후-네비 보장 유지).
- **백엔드 `WorkSessionService.end`** — 30초 폐기 분기 제거(§3-1).

### 제거 대상
- 좌패널 하단 "작업 종료" 버튼 및 관련 상태(`endWorkOpen` 등) → "집필 종료" 모달로 흡수.
- `useWorkSession`의 **자동 start-on-mount** (타임워치 시작이 대체).

## 5. 엣지 케이스

- **타임워치 미시작 후 이탈**: 열린 세션 없음 → 기록 0. (의도)
- **일시정지 상태로 이탈**: 이미 `end`된 상태라 추가 기록 없음. 화면 누적(클라)만 사라짐(서버엔 이미 반영).
- **집필 중 새로고침/탭닫기**: `endBeacon`/`end`로 자동 기록(§3).
- **시작↔일시정지 빠른 반복**: 매번 start/end. 30초 미만도 보존(§3-1)이라 다 합산. 동시 start 경합은 기존 `uq_work_session_open` + 컨트롤러 멱등 처리로 흡수.
- **모달에서 ✕/취소**: 세션 상태 불변(타이머 계속). `end` 미호출.

## 6. 비포함(YAGNI)
- 타이머 누적의 서버 영속/재진입 복원(재진입 0부터로 충분).
- 카운트다운/목표시간 알람, 포모도로.
- 일시정지 자동(유휴 감지).
- 데스크톱(`desktop/`) 동기 — 본 설계는 web 집필실 한정. desktop `electronAPI.sessions`는 별도(영향 없음, 후속).

- 배포 순서: BE(30초 폐기 제거 — 느슨해지는 변경, 구 FE 무영향) 먼저 또는 무관. FE는 타임워치 도입.
