# 페이지 분할(0-4 안1) — dogfooding 검증 가이드

집필 에디터 실시간 페이지 분할 Phase 1 구현의 실앱(Electron) 검증 체크리스트.
자동 게이트(vitest 179 / tsc / build)는 GREEN. 아래는 **실키보드·마우스로만 확인되는 항목**.

- 실행: `cd desktop && export PATH="$HOME/.nvm/versions/node/v24.14.0/bin:$PATH" && pnpm dev`
- 각 항목: **할 일 → 정상(GREEN) → 문제면 보고(RED)**. 하나라도 RED(특히 A 한글 IME)면 멈추고 보고.
- 준비: 작품 하나 열어 집필 화면 진입. 상단 **보기 메뉴**에서 줄노트·줌 토글.

---

## A. 한글 IME (HARD-GATE — 깨지면 즉시 RED)

> **줄노트 ON / OFF 두 상태 모두** 확인. 특히 **페이지 경계(한 장 끝~다음 장 첫 줄) 근처**에서.

| # | 할 일 | 정상(GREEN) |
|---|---|---|
| A1 | 한글을 **빠르게 연타**(조합 중 다음 자모 계속 입력) | 자모 분해·중복 없이 또박또박 조합 |
| A2 | 글자 조합 **도중에 ⌘B**(굵게) 토글 후 계속 입력 | 조합 안 깨지고 굵게 적용 |
| A3 | 한글 입력 중 **한자 변환**(자판 한자키) | 정상 변환 |
| A4 | 받침 있는 글자에서 **Backspace** | 자모 단위로 정상 삭제(분해 깨짐 없음) |
| A5 | **페이지 경계 줄**(장 끝)에서 A1~A4 재확인 | 위와 동일하게 정상 |

→ RED 예: 자모가 분리되어 입력됨 / 마지막 글자가 튐 / 조합이 끊김.

## B. 실시간 페이지 분할

| # | 할 일 | 정상(GREEN) |
|---|---|---|
| B1 | 본문을 **한 장 넘게** 길게 입력 | 한 장 채우면 **아래 다음 장**으로 실시간으로 넘어감 |
| B2 | **엔터 없이 한 문단**을 길게 입력 | 문단 한가운데 줄에서 다음 장으로 흐름 |
| B3 | 장이 여러 개일 때 **각 장이 독립 종이**(그림자·둥근 모서리)로 보이고 **사이에 책상(나무결)** | 낱장이 떠 있는 모습 |
| B4 | 중간 글 **삭제**로 분량이 줄면 | 장 수가 실시간으로 줄어듦 |

## C. 노트식 마우스 클릭

| # | 할 일 | 정상(GREEN) |
|---|---|---|
| C1 | 글자 **아래 빈 줄**(예: 15번째 줄)을 클릭 | 그 줄까지 채워지고 **커서가 그 줄로** 가서 바로 쓸 수 있음 |
| C2 | **글자 위**를 클릭 | 그 글자 위치로 커서 이동(채우기 안 일어남) |
| C3 | 글자 위에서 **드래그 선택** | 정상 선택 |
| C4 | 줌을 바꾼 뒤 C1 | 클릭한 줄로 정확히 이동 |

→ RED 예: 빈 줄 클릭해도 커서가 안 감 / 클릭한 줄보다 한 줄 위·아래로 감.

## D. 외관 (줄노트 · 쪽번호 · 제목)

| # | 할 일 | 정상(GREEN) |
|---|---|---|
| D1 | 줄노트 ON | 줄선이 글자 줄 바로 아래 정렬, **2장·3장 내려가도 안 어긋남** |
| D2 | 줄노트 ON | **간격(책상) 구간엔 줄선 없음**, 종이 안에만 |

| D6 | 제목 | 종이 밖(책상 위)에 분리되어 표시 |
| D7 | 본문 클릭/포커스 | **하늘색 외곽선 없음**(장마다 박스 안 생김) |
