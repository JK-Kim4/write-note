# 02 · 와이어프레임 (화면 구조)

> 4개 핵심 화면의 구조 정의. 시각 token은 [`01-design-system.md`](./01-design-system.md), 상태 변형은 [`03-screen-states.md`](./03-screen-states.md), 실제 렌더는 [`wireframes.html`](./wireframes.html).
> 각 화면은 공통으로 **상단 titlebar + 좌측 64px rail + 본문 영역** 구조를 가진다.

## 공통 chrome

```
┌───────────────────────────────────────────────┐
│ ● ● ●        [화면명]            [상태/단축키]  │  ← titlebar 38px
├──────┬────────────────────────────────────────┤
│ rail │             본문 영역                    │
│ 64px │     (화면마다 컬럼 구성 다름)             │
└──────┴────────────────────────────────────────┘
```

- **rail (64px):** 화면 전환 버튼 4개. **작품 · 집필 · 메모 · 기록.** 현재 화면 버튼이 active(blue).
  - 작품 = 작품 화면 / 집필 = 집필실 / 메모 = 메모함 / 기록 = 빠른 메모(modal 열기).
- **titlebar:** 좌측 신호등 dots, 중앙 화면명, 우측 보조 라벨(저장 상태·글자수·단축키 등).

> rail 라벨은 좁은 폭(64px)에 맞춘 2자. 아래 다이어그램에서 `▶`는 현재 화면을 뜻한다.

---

## 1. 작품

**목적:** 작성할 작품을 만들거나 고른다.

**컬럼:** `64px(rail) / 1fr(새 작품) / 250px(작품 목록)`

```
┌──────┬───────────────────────────┬──────────────────┐
│ rail │        새 작품 작성         │    작품 목록      │
│ ▶작품│  ┌─────────────────────┐  │   최근 작업순     │
│  집필│  │ 새 작품              │  │  ┌────────────┐  │
│  메모│  │ 제목만으로 시작...    │  │  │ ▓▓▓ (선택) │  │
│  기록│  │ [작품 제목         ] │  │  └────────────┘  │
│      │  │ [한 줄 메모 optional]│  │  ┌────────────┐  │
│      │  │ [목표 분량 optional] │  │  │ ▓▓         │  │
│      │  │      [초기화][만들기]│  │  └────────────┘  │
│      │  └─────────────────────┘  │                  │
└──────┴───────────────────────────┴──────────────────┘
```

**main body — 새 작품 작성 블럭 (max-width 560px, 중앙):**
- 제목 "새 작품" + 설명 "제목만으로 시작하고, 나머지는 나중에 보강한다."
- 입력 3개: **작품 제목**(필수), **한 줄 메모**(optional), **목표 분량**(optional).
  - MVP UI 노출 = `title` + `summary`(한 줄 메모). `목표 분량`은 데이터 모델의 `targetLength` 자리표시이나 첫 버전에서 비워둬도 됨 (설계 후속결정 #3).
- 액션: `초기화`(secondary) + `작품 만들기`(primary), 우측 정렬.

**우측 — 작품 목록 (250px):**
- 헤더 "작품 목록" + "최근 작업순".
- 작품 카드 리스트. 현재 선택 카드는 active(blue 테두리+옅은 배경).
- 카드 클릭 → 집필실 진입.

## 2. 집필실

**목적:** 현재 작품 문서를 쓴다. **에디터가 화면의 주인공.**

**컬럼:** `64px(rail) / 1fr(에디터) / 260px(연결된 메모)` — large 프레임(가장 큰 화면).

```
┌──────┬───────────────────────────┬──────────────────┐
│ rail │      ┌──────────────┐     │  연결된 메모   + │
│  작품│      │  단막극 초안   │     │  현재 작품에 연결│
│ ▶집필│      │  ───────────  │     │  ┌────────────┐ │
│  메모│      │  ──────────── │     │  │ ▓▓ 메모    │ │
│  기록│      │  ───────      │     │  └────────────┘ │
│      │      │  (paper page) │     │  ┌────────────┐ │
│      │      └──────────────┘     │  │ ▓ 메모     │ │
│      │      max-width 660px       │  └────────────┘ │
│      │      중앙 정렬              │  ┌─ dashed ───┐ │
│      │                            │  │ (empty)    │ │
└──────┴───────────────────────────┴──────────────────┘
   titlebar 우측: "저장됨 · 1,248자"
```

**중앙 — 에디터 (paper):**
- `max-width 660px`, 중앙 정렬, 아주 약한 종이색(`--paper`) + 약한 그림자.
- 문서 제목(h2 22px) + 본문.
- titlebar 우측에 **저장 상태 + 글자수** 표시 (`저장됨 · 1,248자`).

**우측 — 연결된 메모 panel (260px, 얇게):**
- 헤더 "연결된 메모" + "현재 작품에 연결됨" + 추가(`+`) 버튼.
- 현재 작품에 연결된 메모 카드 리스트.
- 연결 메모 없으면 dashed empty 카드: "작품에 연결된 메모가 없으면 이 패널은 조용히 비어 있다."

> **HARD 규칙:** 이 panel이 에디터보다 시각적으로 강해지면 안 된다. 배경은 `--panel-2` 계열, 좌측 얇은 border로만 분리.

## 3. 메모함

**목적:** 캡처한 메모를 모으고 작품에 붙인다.

**컬럼:** `64px(rail) / 1fr(메모 리스트)`

```
┌──────┬───────────────────────────────────────────┐
│ rail │  메모함                    [빠른 메모]      │
│  작품│  캡처한 생각을 작품에 붙인다.               │
│ ▶메모│  ┌─────────────────────────────────────┐  │
│  기록│  │ ▓▓ 메모        [단막극 초안] (연결) │  │
│      │  │ ──────────────────                  │  │
│      │  └─────────────────────────────────────┘  │
│      │  ┌─────────────────────────────────────┐  │
│      │  │ ▓▓ 메모        [미연결]             │  │
│      │  │ ──────────────                      │  │
│      │  └─────────────────────────────────────┘  │
└──────┴───────────────────────────────────────────┘
   titlebar 우측: "메모 12개"(개수) · 헤더 제목 아래 [전체|미연결] 세그먼트 토글(필터)
```

- 헤더 "메모함" + "캡처한 생각을 작품에 붙인다." + `빠른 메모`(primary) 버튼.
- 메모 카드 리스트 (최신순):
  - 각 카드 우측에 **연결 상태 칩**(secondary 버튼 모양): 연결된 작품명 또는 "미연결".
  - 칩 클릭 → 작품 연결/해제.
- **필터 = 본문 헤더 세그먼트 토글** `[전체 | 미연결]` (제목 행 아래, 좌측 정렬). titlebar 우측은 "메모 N개" 개수 표시로 활용. (필터 위치 B안 확정 — 2026-06-01)

## 4. 빠른 메모

**목적:** 떠오른 생각을 최소 마찰로 캡처. **modal.**

```
┌──────────────────────────────────────────────────┐
│ ● ● ●          빠른 메모                Esc        │  titlebar
│        (배경 어둡게 dim — rgba 15,23,42 .22)        │
│        ┌────────────────────────────────┐          │
│        │ 빠른 메모              [닫기]    │          │
│        │ 본문만 적고 바로 저장한다.       │          │
│        │ ┌────────────────────────────┐ │          │
│        │ │ 떠오른 장면, 대사, 관찰...   │ │  textarea│
│        │ │                            │ │  130px   │
│        │ └────────────────────────────┘ │          │
│        │ 현재 작품: 단막극 초안  [저장]  │          │
│        └────────────────────────────────┘          │
└──────────────────────────────────────────────────┘
```

- backdrop dim 위에 modal(width 430px) 중앙.
- 헤더 "빠른 메모" + "본문만 적고 바로 저장한다." + `닫기`.
- **textarea** (본문만). placeholder "떠오른 장면, 대사, 관찰...".
- 하단: 좌측 "현재 작품: {이름}" (활성 작품 없으면 미연결 저장), 우측 `저장`(primary).
- 단축키: 열기는 후속 phase의 전역 단축키(설계 후속결정 #2), 닫기 `Esc`.

> rail 라벨 4개(작품·집필·메모·기록)는 모든 화면에 공통 노출, 현재 화면만 active.
