## 3. 미해결 핵심 이슈

### (A) 글이 괘선 영역 밖 여백에 써짐 — 최우선, 미해결
- **진짜 A4 모델**(`column-wrap` + sheet 297mm): sheet(297mm=32.48줄) > 본문영역(27줄)이라 **하단에 괘선 없는 여백이 구조적으로 생기고**, 거기로 텍스트·커서가 침범.
- **26줄 모델로 롤백**했고, **헤드리스(puppeteer) 측정상 26줄 모델은 여백 침범 0 + 26줄 차면 다음 장**으로 정상이었음.
- 그러나 **사용자 브라우저 화면에서는 롤백 후에도 여백 침범이 계속 보임**(이미지17). `.next` 캐시 삭제 + dev 재시작 + 강한 새로고침 안내했으나 **사용자 확인 시점에 미해결 / 원인 미확정**.
- ⚠️ **헤드리스 측정(작품 919, 깨끗한 "가나다" 텍스트)과 사용자 실제 화면(자모 분해 + 빈 줄 누적 + 특정 작품)이 불일치.** 측정 환경이 사용자 환경을 재현하지 못한 것이 이 세션 실패의 핵심.

### (B) 한글 IME 자모 분해 — 미진단
- 사용자 스크린샷(이미지15·16·17)에서 빠른 타이핑 시 자모가 분해되어 입력됨("ㅁㄴㅇㄹ..."). IME 가드(`view.composing`)는 코드상 존재하나 실제 깨짐. 별도 진단 필요.

### (C) click-fill로 인한 빈 줄 누적 오염
- "빈 종이 클릭 시 빈 줄 자동 채우기"(click-fill, `handlePaperMouseDown`/`handlePagedMouseDown`)가 빈 문단을 대량 삽입 → 작품 오염. 사용자 작품 **918("장편")에 빈 줄 24개 누적**. 진짜 A4 모델에서 이 빈 줄이 분할을 1줄씩 깎고 여백 침범을 악화시킨 정황.

## 5. 실측으로 확인된 사실(참고)

- 26줄 모델 + 빈 줄 없는 일반 텍스트: page0 26줄 + 다음 장, **여백 침범 0** (헤드리스).
- 진짜 A4 모델: 줄높이 34.56px·A4 297mm 모두 비정수 → 줄/페이지 서브픽셀 누적 + `column-height=N*LINE` 경계에서 마지막 줄이 밀림.
- `column-wrap`/`column-height`는 **CSS Multicol L2 표준**(비표준 아님), Chrome 145+ 구현. 단 진짜 A4(비정수)와 정렬이 불안정.
