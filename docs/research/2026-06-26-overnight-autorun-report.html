D-1 · C-2를 두 spec으로 분할 (042 내부 탭 / 043 집필 참조)

무엇

코드베이스

PRD
§5.4 ② 내부 탭(관리)과 ③ 집필 참조(읽기)는 의도가 다름.

대안
한 spec 일괄 → 게이트·배포 단위가 섞이고 회귀 추적이 어려움(기각).

D-2 · 내부 탭 호스트 UI = 작품(집필 화면 탭) / 시리즈(라이브러리 섹션)

무엇
"작품/시리즈 상세에 보드 탭"을 어디에 다는가.
코드베이스
실측: 별도 작품/시리즈 상세 페이지가 없음(라우트 = /library·/works/[id]·/boards뿐). 작품 상세 = 집필 화면(works/[id]), 시리즈 상세 = /library 드릴인(?folder=).
PRD
§5.4 ② "그 주체에 매달린 보드만, 생성 시 owner 자동".
채택
작품 = BWorkSidePanel에 "보드" 탭(메모·인물·보드) · 시리즈 = /library 드릴인에 "시리즈 보드" 섹션. 둘 다 owner 자동 생성(picker 없이 이름만).
대안
새 작품/시리즈 상세 라우트 신설 → 범위 폭증·기존 흐름 충돌(기각).
되돌리기
탭/섹션 추가뿐 — 제거 시 원복. 에디터 레이아웃 무변경.

D-3 · reference-boards 경로 = /api/boards/reference?projectId=

무엇
핸드오프가 적은 GET /works/:id/reference-boards의 실제 경로.
코드베이스
API 베이스는 /api/projects("works"는 FE 라우트명일 뿐). 보드 도메인 로직이라 /api/boards 하위가 응집적. /{boardId}보다 리터럴 /reference를 앞에 둬 라우팅 충돌 회피.
PRD
§9·§10 — 그 작품 보드 + 상위 시리즈 보드.
채택
GET /api/boards/reference?projectId= · 상위 시리즈 = project.categoryId(실측) · 본인 작품 아니면 404.
대안
/api/projects/{id}/reference-boards — 컨트롤러 교차 결합(보드 로직이 ProjectController로)이라 기각.
되돌리기
엔드포인트 추가뿐, 기존 계약 무변경.

D-4 · 마지막 본 보드 저장소 = localStorage

무엇
작품별 "마지막 본 보드"를 서버에 둘지 localStorage에 둘지.
코드베이스
실측: SettingsService.ALLOWED는 key별 "값 화이트리스트"(theme∈{light,dark,system} 등). 임의 boardId를 받는 키를 추가하면 그 검증 불변식이 깨짐.
PRD
§9 "마지막으로 본 보드를 기억했다가 기본으로 연다".
채택
localStorage writenote.board.lastViewed.v1(projectId→boardId 맵, 손상 내성). 비파괴 기본값.
대안
서버 키 신설 — ALLOWED 화이트리스트 모델을 깨거나 새 테이블 필요(기각, 마이그레이션 회피).

D-5 · 집필 참조 UI = 우측 슬라이드오버(overlay), 분할 레이아웃 비변경

무엇
참조 보드를 어떻게 띄우는가(레이아웃 분할 vs 오버레이).
코드베이스
집필실은 자체 EditContext 에디터 + 3패널 flex로 회귀 민감(ISSUE-048 포커스 등). 에디터 flex를 재구성하면 위험.
PRD
§5.4 ③ "곁눈질 — 읽기 중심", §9 "원고와 나란히".
채택
우측 슬라이드오버 overlay(BoardReferencePanel) — 집필 3패널 flex 무변경. 보드 캔버스는 dynamic(ssr:false)로 번들 격리(PRD §9). 토글 켜면 원고 위에 곁들여 표시, 끄면 전폭 복귀.
대안
에디터 옆 4번째 컬럼 = 880px에서 과밀 + flex 재구성 회귀 위험(기각).
되돌리기
토글 + 패널 컴포넌트만 — 제거 시 집필실 원복.

D-6 · E(메모·인물 통합) 방향 = ③ 참조 통합 권고 (단, 사용자 결정 대기)

무엇
메모·인물을 보드로 어떻게 통합하나(파괴적 결정 영역).
코드베이스
실측 충돌 확정 — 메모=M:N(memo_projects)·iOS 캡처(source/captured_at)·soft-delete, 인물=구조화 필드(age/gender/traits), 카드=1보드 전속·평문. 차원이 안 맞아 "그냥 이관" 불가.
PRD
§0 "메모·인물 메뉴 폐기 → Card 흡수"(이상). §3 N:N 비목표.
채택
③ 참조 통합("가져오기") 권고 — 원본 보존, 보드에 "가져오기"로 카드 복제만 추가(비파괴, 마이그레이션 0). 단 자동 구현 금지 — 사용자가 D1~D6 결정(가드 3).
대안
① 완전 통합 / ② 부분 통합 = 데이터·캡처 경로 손실(파괴적) → 자동 진행 금지. ④ 보류 = 안전.
되돌리기
현재 변경 0(문서만). ③ 구현 시에도 "가져온 카드 삭제"로 원복.

042 내부 탭

  • 작품 보드 탭 — 집필실(/works/{id}) 우측 보조 패널에 메모·인물·보드 3탭. "보드" 탭에 이 작품에 붙은 보드만 보임.
  • 작품 보드 생성 — "보드" 탭의 "+ 새 보드" → 이름만 입력 → 생성되며 그 보드 캔버스로 이동(/boards/{id}). 만든 보드의 소속이 이 작품인지(상단 보드 메뉴 라벨 "작품 · …").
  • 시리즈 보드 섹션 — 작품 보관함(/library)에서 시리즈를 열면(드릴인) 아래 "시리즈 보드" 섹션에 그 시리즈 보드만 보임 + "+ 새 보드"(이 시리즈 소속 자동).
  • 빈 상태 — 보드가 없을 때 빈 캔버스 대신 안내 + "+ 새 보드"만 보임.
  • 열기 — 목록의 보드 클릭 시 그 보드 캔버스로 이동.

043 집필 참조 (슬라이드오버)

  • 토글 — 집필실 좌측 패널(목차 위)의 "보드 참조" 버튼 → 우측에서 슬라이드오버가 열림. ✕ 또는 ESC로 닫힘.
  • 후보 — 그 작품 보드 + 상위 시리즈(작품이 시리즈 소속일 때) 보드가 모두 후보. 여러 개면 상단 드롭다운으로 전환, 하나면 바로 표시.
  • 캔버스 — 슬라이드오버 안에 보드 카드/연결이 보이고 읽을 수 있음(곁눈질). 원고는 왼쪽에 그대로(레이아웃 안 깨짐).
  • 마지막 본 보드 — 보드를 전환한 뒤 닫았다 다시 열면, 마지막에 보던 보드가 기본으로 열림(작품별 기억).
  • 빈 상태 — 작품에 붙은 보드가 하나도 없으면 "보드로 가기" 안내.
  • 집필 무회귀 — 보드 참조를 열고 닫은 뒤 본문 입력(한글 IME)·자동저장·목차가 정상인지(에디터 레이아웃 회귀 점검).

무회귀 (트랙 A~D 보존)

  • 보드 캔버스(전역 /boards/{id})에서 카드 만들기·잇기·종류 칩·한눈에 보기가 그대로 동작.
  • 전역 보드 허브(/boards) 목록·검색·소속 라벨·생성 picker 정상.

4. 미해결 / 막힌 지점

게이트 RED 없음

BE/FE 모든 단위가 GREEN으로 커밋. 회귀를 누적하지 않았습니다.

결정 보류 E 데이터 모델

메모·인물 통합 방향(D1~D6)은 파괴적 결정이라 자동 확정하지 않았습니다. 비파괴 초안만 작성 — ③ 참조 통합 권고 / ④ 보류도 안전. 사용자 결정 후 spec 승격.

미검증 authed 동작

로그인 불가로 042·043의 인증 뒤 화면 동작은 못 봤습니다(자동 게이트는 GREEN이나, 게이트가 화면 상호작용을 증명하진 않음). 위 체크리스트로 확인 필요.

참고 작품 탭 ↔ 참조 분리

집필실에 "보드 탭"(관리, 042)과 "보드 참조"(읽기, 043)가 둘 다 있습니다. PRD의 관리/읽기 구분을 따른 의도적 분리지만, 써 보고 중복감이 들면 통합을 후속으로 잡을 수 있습니다.
