기술 스택

공개된 GitHub·채용공고·스택 분석 기준입니다. 1차 출처로 확정된 것은 뮤블의 데스크톱 빌드(Tauri)뿐이며, 노벨라의 상세 스택과 두 서비스의 에디터 엔진은 공개 자료가 없어 추정하지 않습니다.

⚠️ 추측 금지 원칙

‘노션식·블록 기반’이라는 마케팅 문구만으로 ProseMirror/TipTap/자체 엔진 여부를 단정하지 않습니다. 미확인 항목은 공개 자료 없음으로 표기했습니다.

확정된 사실 — 뮤블 = Tauri

뮤블 개발사 공식 GitHub(KimuSoft/muvel-public) 릴리스 노트에 직접 명시:

“자동 빌드된 Tauri 앱입니다. production 브랜치 푸시 시 자동 릴리스.”

레포 토픽에 Rust·TypeScript가 명시되어 있습니다. 마케팅 문구가 아닌 CI/CD 릴리스 노트라 신뢰도가 높습니다. 교차검증 3-0 · primary

github.com/KimuSoft/muvel-public/releases

Tauri 선택의 함의 (분석)

  • Electron(우리) 대비 번들이 가볍고 메모리 사용이 적음 — OS 내장 웹뷰(macOS=WebKit) 사용.
  • 그런데 macOS/iOS에서 Tauri 웹뷰 = WebKit → 우리가 겪은 것과 동일한 iOS WebKit 한글 IME 토양 위에 있을 가능성. (상세 §4)
  • 즉 뮤블도 ‘크로스플랫폼 한 코드’지만 iOS만 베타로 둔 것은 의미심장합니다.

미확인 — 노벨라 스택 & 두 서비스의 에디터 엔진

노벨라의 프론트 프레임워크·백엔드·호스팅을 밝힌 공개 채용공고나 기술블로그는 발견되지 않았습니다. ‘구글 드라이브 실시간 백업’ 연동만 1차 확인됩니다. 두 서비스 모두 ‘블록 기반’으로 묘사되나 내부 에디터 엔진은 추정 불가입니다.

소설비 관점 시사점

두 경쟁사 모두 에디터 엔진을 공개하지 않았고, 둘 다 ‘블록형’입니다. 소설비의 자체 EditContext 엔진 + 실시간 페이지 분할(줄 단위 측정·배치)은 블록형으로는 만들기 어려운 차별화 자산입니다. 다만 자체 엔진은 회귀 위험이 커 dogfooding 게이트가 필수라는 비용도 동시에 있습니다.
