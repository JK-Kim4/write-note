무엇을 만들었나 — 사용자 스토리 3

P1 · MVP

작품별 진척을 한눈에

기록 화면 = 작품 카드 목록(읽기 전용). 신규 데이터 없이 기존 작품·본문에서 파생.

  • 목표 글자수 대비 진척 % (미설정은 "목표 미설정")
  • 가장 최근 수정 날짜
  • 본문 마지막 문장 (lastSentence 재사용)
  • 작품 0개 빈 상태
P2

기록 메모 + 아코디언

집필 "작업 종료"에서 그날 작업을 한 줄로 남김. 메모(아이디어 캡처)와 별개의 작업 일지로 누적.

  • 카드에 최신 기록 1줄 표시
  • 펼치면 누적 기록 전체 (최신순, lazy 조회)
  • 저장/취소 모달 · 취소 시 미추가
P3

작업 시간 자동 추적

집필 진입=시작, 이탈/전환/앱닫힘=자동 종료. 작품별 합산해 총 작업 시간 표시.

  • 자동 시작 + 자동 종료 + "작업 종료" 버튼
  • 30초 미만 세션 폐기 (잠깐 진입 제외)
  • 비정상 종료 dangling 정리 (과대 합산 방지)
  • 명시 종료 = 세션종료+기록 한 트랜잭션

어떻게 동작하나 — 데이터 흐름

집필 진입→ App.tsx effect→ sessions.start→ IPC→ WorkSessionRepository→ SQLite
기록 화면 진입→ logs.list→ Store.listLogCards→ 진척%·마지막문장·최신기록·총시간 집계→ LogCard ×N

레이어: repository → Store → IPC(contract/handler/preload) → renderer(App·LogScreen·LogCard) — 기존 Electron 구조 그대로 재사용. 시각(시간)은 main 프로세스 단일 시계.

신규 데이터 (스키마 v5 → v6, 기존 데이터 보존)

project_logs

id · project_id(FK CASCADE) · body · created_at — 작품별 누적 기록 메모

work_sessions

id · project_id(FK CASCADE) · started_at · ended_at(null=진행중) — 작업 세션

리뷰 2회 — 발견과 조치

각 회차 2개 리뷰어(정확성/통합 · 엣지케이스/완전성)를 병렬로 띄워 적대적 검토. advisor가 발견을 직접 검증(진짜 결함 vs 개선 선별) 후 수정.

회차심각도발견조치
1엣지formatDuration 음수 ms → "-1시간" 깨짐ms<=0 가드 + 테스트
1UX무로그 작품도 아코디언 버튼 노출 / 조회 실패 무처리로그 0건 시 토글 숨김 + try/catch
1잠재명시 종료 후 cleanup의 중복 sessions.endskipNextSessionEnd ref 가드
2치명LogCard CSS 전 클래스 미정의 → 화면 무스타일디자인 토큰 정합 스타일 추가 + dead CSS 제거
2견고성endWithLog reject 시 ref 잔존 → 이후 세션 미종료.catch로 플래그 복원 + 회귀 테스트
2정리dead ref / FR-006 빈 상태 테스트 부재autoFocus 전환 + LogScreen.test 신규
검증 후 "수정 안 함"으로 판정한 발견 — N+1 조회(기존 listProjectCards와 동일 패턴, 로컬 수십 개 규모, YAGNI) · 아코디언 stale(워크플로우상 LogScreen에서 로그 추가 불가 → 재진입 remount로 해소) · StrictMode ref(dev 전용, 테스트 미사용). 근거를 확인해 비결함으로 분류.
