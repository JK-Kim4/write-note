## 요약

- **무엇**: "컨텍스트가 안 죽는 작가용 작업실." 한 작품의 모든 맥락(메모·등장인물·톤·목표 분량·진척·지난 세션의 마지막 한 줄)이 글쓰기 화면과 *같은 시스템*에 살아 세션이 끊겨도 영속하게 한다.
- **왜**: 작가의 두 통증 — ① 죽은 메모(왜 적었는지 잃음) ② 재진입 비용(매 세션마다 맥락 재구성). 둘을 **AI 없이, 마찰 설계로** 1차 해결한다.
- **스타일 한 줄**: **따뜻함 · 몰입 · 완결성.** 쓰는 면은 iA Writer식 순도, 감싸는 작업실은 따뜻한 정서. 현재 Desktop 결정본 비주얼 방향은 **"평평한 양피지(디지털 안식처)" + 잉크블루 단일 액센트**.
- **디자인 대상**: **Web(Next.js) 클라이언트 전반 비주얼 리디자인.** Desktop은 이미 브랜드·비주얼이 결정본으로 확정된 검증 MVP이고, Web은 그 화면을 1:1 포팅한 트랙이라 디자인 정리가 필요한 상태다.
- **즉시 짚을 디자인 이슈**: Web 브랜드명 불일치(welcome 화면 "소설비" vs `<title>` 메타데이터 "write-note").

## 1. 디자인 작업 대상과 범위

이 프로젝트는 **두 클라이언트**가 있다. 디자인 리디자인의 대상은 **Web**이다.

| 클라이언트 | 상태 | 비주얼 확정도 |
|---|---|---|
| **Desktop (Electron)** | 검증 완료 MVP (v0.1.0) | 브랜드·비주얼 **결정본 확정**("소설비") — 디자인의 *기준점* |
| **Web (Next.js)** | Desktop 화면을 웹으로 1:1 포팅 중(트랙 015/016) | 포팅 위주라 **정리·재디자인 필요** ← 이번 작업 대상 |

> Desktop의 비주얼 결정본은 디자인의 **레퍼런스**로 삼되, Web은 자체 사용 맥락(브라우저·로그인·다중 사용자·반응형)이 있으므로 Desktop을 무조건 복제하기보다 Web에 맞게 재해석한다.

## 2. 제품 목표 (무엇을 위한 앱인가)

**본질: "컨텍스트가 안 죽는 작가용 작업실."**

한 작품에 필요한 모든 맥락이 글쓰기 surface와 같은 시스템에 살아, 세션이 끊겨도 영속한다.

**정조준하는 두 통증**
1. **죽은 메모** — 일상에서 적어둔 영감이 며칠 뒤 "왜 적었는지"를 잃는다.
2. **재진입 비용** — 한 작품이 수십 개의 끊긴 세션에 걸쳐 진행되며, 매 진입마다 장르·분량·톤·인물·진척을 재구성하는 세금이 붙는다.

**타깃 사용자**
- 취미로 소설·단막극·장르 단편을 쓰는 작가. 본업이 따로 있고 **주말·저녁 30~90분 세션**으로 쓴다.
- 세션 간격이 며칠~몇 주라 "지난번 어디까지 했지" 재구성 비용이 크다.
- **한국어가 1차 언어.** 1차 사용자는 제작자 본인 + 같은 통증의 한국어 작가 커뮤니티.

**핵심 입장 (디자인에 직결)**
- **MVP는 AI를 쓰지 않는다.** "대신 써주는" 도구가 아니라 "맥락을 잃지 않게 하는" 도구. → 화면에 AI/자동생성 어포던스를 넣지 않는다.
- **1차 가치 = 컨텍스트 영속성** (글쓰기 효율 아님). 모든 화면은 "재진입 비용 0"에 기여하는지로 판단.
- **성공 기준** — 제작자 본인이 글쓰기 세션을 시작할 때 매주 한 번 이상 앱을 여는 것.

## 3. 추구하는 스타일

### 3-1. 브랜드 정체성

**3단어: 따뜻함 · 몰입 · 완결성 (warm · immersive · complete)**

정체성은 **두 레이어의 대비**다.
- **쓰는 surface는 순도 높게** — iA Writer처럼 도구가 물러나고 본문만 남는다. 글 쓰는 동안 어떤 chrome도 시선을 빼앗지 않는다.
- **감싸는 작업실은 따뜻하게** — 종이 질감, 세리프 활자, 잉크의 아날로그 무게.
- **한 작업실에 모든 맥락** — Scrivener의 *완결성*("필요한 모든 것이 한 곳에")을 따르되, Scrivener의 복잡함이 아니라 iA Writer의 절제로 실행.

**감정 목표** — 글 쓸 땐 **몰입**(도구가 사라짐), 다시 열 땐 **안도**(맥락이 이미 살아 있음), 매일 열어도 **따뜻하고 피로 없음**.

### 3-2. 디자인 원칙 5

1. **surface는 순도, 작업실은 따뜻함** — 두 레이어 대비를 모든 화면에서 유지.
2. **한 작업실에 모든 맥락 — 절제로 실행한 완결성** — 보조 정보는 필요할 때 조용히, 평소엔 물러남. 완결성은 기능 수가 아니라 *맥락이 새지 않음*으로 증명.
3. **마찰 설계 > 자동화** — 사용자가 의도적으로 적은 정보를 적시에 다시 만나게 한다.
4. **컨텍스트 영속성이 1차 가치** — 글쓰기 효율이 아니다.
5. **따뜻함은 재료로** (종이·세리프·잉크·여백 리듬), **장식으로가 아니라** — gradient·glow·cream-wash 같은 손쉬운 장식 금지.

### 3-3. Anti-references (피해야 할 것)

- **Notion형 만능 워크스페이스** — 블록·DB·슬래시 커맨드. 이건 글쓰기 작업실이지 범용 도구가 아니다.
- **Google Docs형 사무 편집기** — 툴바 과잉, 사무 문서의 밋밋함.
- **cream/beige "감성" slop** — AI의 2026 기본값인 cream 톤 wash로 따뜻함을 연출하지 않는다.
- **카드 그득한 SaaS 대시보드 / 큰 hero 랜딩** — 첫 화면부터 지표·카드·배너가 아니라 조용한 작업실.

### 3-4. Desktop 확정 비주얼 방향 (Web 리디자인의 레퍼런스)

2026-06-08 브랜딩 작업에서 확정된 결정본. **기존 "나무 책상" 방향에서 전환됨**에 유의.

- **"평평한 양피지(디지털 안식처)"** — 나무결 텍스처·vignette **제거**, 종이는 한 단계 밝게 띄워 톤 대비.
- **잉크블루 단일 액센트** (골드/오렌지 미도입).
- 본문 서식 = 영구 툴바 아닌 **BubbleMenu**(선택 시에만 노출). 글꼴·크기 고정 = **고운바탕 18px**(iA Writer식 순도, 변경 도구 없음).
- 로고 = **펜촉+날개 마크**(글씨 없음), 라이트=잉크블루 / 다크=밝은블루 자동 전환.
- 한글 개행 = `word-break: keep-all` 전역(음절 분리 방지). 단, 에디터 본문은 페이지 분할 보호 위해 `normal` 명시.
- 4화면 통일 골격: `[좌측 Rail][메인 1fr][우측 패널 320px(기본 숨김)]`.

### 3-5. 현재 Web 디자인 토큰 (실측 — `frontend/src/styles/tokens.css`)

리디자인 시 기존 토큰을 갱신·확장하게 됨. 현재 값:

| 역할 | 라이트 | 다크 |
|---|---|---|
| canvas | `#ffffff` | `#1c1c1e` |
| parchment(기본 surface) | `#f5f5f7` | `#242426` |
| manuscript-cream(원고지) | `#fbf8ee` | `#28231d` |
| ink(본문) | `#1d1d1f` | `#f4f4f6` |
| accent(액션 블루) | `#0066cc` | `#2997ff` |
| error | `#d70015` | `#ff453a` |
| hairline | `#e0e0e0` | `#2c2c2e` |

- **타이포그래피**: UI = SF Pro 계열 17px / 본문 prose = **Noto Serif KR** 18px·line-height 1.85 / 원고지 = Nanum Myeongjo. (라이트 본문 weight 400, 다크 300.)
- **반경**: 메모 카드 14px · 모드 카드 16px · 작품 카드 18px · 유틸 버튼 8px · pill 9999px.
- Web은 Desktop의 `desktop-app.css`를 전역으로 가져와 쓰는 구조(포팅 산물).

> ⚠️ 현재 토큰 주석은 (구) 루트 `DESIGN.md`를 SoT로 가리키나, 그 문서는 web-legacy다. 현행 비주얼 SoT는 `docs/DESIGN.md` + `design/DESIGN.md`(Desktop 결정본). 리디자인 시 토큰의 SoT 정합도 정리 대상.

### 3-6. 접근성 (디자인 제약)

- **WCAG 2.1 AA** 목표 — 본문 대비 ≥4.5:1, 큰 텍스트 ≥3:1, **placeholder도 동일 기준**. 종이색 위 세리프 본문이 대비 미달로 흐려지지 않게.
- **모션 줄이기 의무** — 모든 애니메이션은 `prefers-reduced-motion: reduce`에서 크로스페이드/즉시 전환 대체.
- **한국어 1차** — UI·콘텐츠·검색 모두 한국어. 본문 입력은 **한국어 IME 조합 안정성**, **한국어 세리프 본문 가독성**이 핵심 검증 영역.

## 4. 현재 제공 기능 (Web 기준)

> 실측: 아래 라우트 모두 `frontend/src/app/`에 `page.tsx` 존재. (`docs/web-app-features.md`(2026-06-08)는 `/logs`·`/contact`를 "미구현"으로 적었으나 그 후 구현됨 — 문서가 stale.)

| 영역 | 사용자가 할 수 있는 일 | 비고 |
|---|---|---|
| **회원/인증** | 이메일 가입·인증, 카카오 OAuth/연결, 비번 재설정, 로그인 잠금, 세션 유지 | **Web 전용** (Desktop엔 인증 없음). 화면 13종(`/auth/*`) |
| **작품 관리** | 작품 벽(홈 `/`), 생성(`/projects/new`), 상세·편집, 보관/해제, 삭제, 메타(장르·목표 분량·톤·시놉시스·세계관), **"다음 장면" 한 줄** | 카드에 마지막 문장·목표 분량·다음 장면 표시 |

| **곁쪽지(메모)** | 빠른 캡처(잉크 버튼), 메모 책상(`/memos`, 전역), 메모 서랍(작품별), 다중 연결·고정 | 삭제·복구는 백엔드 미지원으로 **일부 보류**(ISSUE-026) |
| **집필 기록(로그)** | 작품별 진척%·총 작업시간·기록 메모, 작업 세션 자동 추적 | `/logs`. 015 트랙에서 구현 |
| **문의** | 인앱 문의 폼(Formsubmit) + 카카오 채널 | `/contact`. 브라우저 CORS 미검증(ISSUE-027) |
| **설정** | 작성 모드(에디터/원고지)·원고지 크기·테마(라이트/다크), 계정 정보, **모바일 캡처 API 토큰 관리** | `/settings` |

**화면 목록 (실측 라우트)**
- 메인: `/`(작품 벽) · `/memos` · `/logs` · `/contact` · `/settings`
- 작품: `/projects/new` · `/projects/[id]` · `/projects/[id]/edit` · `/projects/[id]/write` · `/projects/[id]/characters`
- 인증: `/auth/login` 외 12종(가입·인증·재설정·카카오 연결)

**핵심 차별 흐름 (디자인 시 가장 중요)**
- **재진입 한 장** — 집필 재개 시 마지막 문장 + 다음 장면 + 곁쪽지 1장(고정→최근연결→최근캡처)을 한 장으로 보여줘 "어디서 멈췄는지"를 즉시 되짚게 함. → 제품의 1차 가치(컨텍스트 영속성)가 가장 또렷이 드러나는 화면.
- **빠른 캡처(잉크 한 방울)** — 화면 어디서든 최소 마찰로 메모 캡처, 활성 작품 자동 연결.

## 5. 제공 예정 기능

**Web 포팅 트랙 (미착수)**
- 등장인물 UI 보강 · 모바일 캡처(iOS Shortcut) · Desktop `.db` 가져오기(데이터 이관)
- **Vercel(프론트) + Render(백엔드) 런칭**
- 곁쪽지 백엔드 **soft-delete/restore**(현재 hard delete뿐 — ISSUE-026)

**제품 로드맵 (확정 방향)**
- **AI 도입** — 실사용 데이터가 쌓인 뒤 결정. MVP는 의도적 배제.
- **비동기 공동 집필** — 로드맵에 있음. *버전 충돌 감지는 이 토대이므로 제거 금지*(016 버전 토큰이 이 토대).

**Desktop 후속 (참고)**
- 페이지 분할 Phase 2(쪽 네비게이션) · Phase 3(메모↔본문 위치 앵커)
- 코드서명+공증(macOS/Windows) — 현재 무서명 마찰(ISSUE-025)

## 6. 디자인 진행 시 유의사항 (체크리스트)

- [ ] **브랜드명 통일** — Web `<title>` 메타데이터가 "write-note" → "소설비"로 정리 필요(welcome 화면은 이미 "소설비"). 로고(펜촉+날개) Web 적용 여부 확정.
- [ ] **두 레이어 대비 유지** — 집필 surface는 순도(chrome 최소), 그 외 작업실은 따뜻함. 한 화면에서 둘을 섞지 않기.
- [ ] **anti-ref 회피** — cream-wash·SaaS 대시보드 카드 그리드·툴바 과잉·만능 워크스페이스 느낌 배제.
- [ ] **한국어 본문** — Noto Serif KR 세리프 가독성 + IME 조합 안정성. 라이트/다크 양쪽 + 모바일 검증.
