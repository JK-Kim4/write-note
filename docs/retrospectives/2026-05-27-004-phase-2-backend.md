## §4 어긋난 점 (What went wrong)

1. **agent-workflow-discipline.md §6 누락 (Phase 6 R2)** — CharacterController 의 `@AuthenticationPrincipal` 타입과 `Result.success` factory 메서드 패턴이 003 ProjectController 와 다르게 박혔다가 정정. implement 진입 직전 ProjectController grep 의무 누락. **사후 정정 비용 = 1 Edit + Write 재작성**.

2. **Edit fail 후 검증 누락 (Phase 7 R2)** — CharacterService 의 `reorderCharacters` 메서드 Edit 호출 시 첫 번째가 경로 오타 (`jongwan-au-air`) 로 fail → 두 번째 Edit 박지 못한 채 진행 → ktlintFormat 가 미사용 import 제거 → compileKotlin fail 시점에 발견. **Edit fail 후 즉시 정독 + 재시도 의무 누락**.

3. **tasks.md 추측 정합 vs 003 실제 (Phase 8 R3)** — T055/T056 명시 = "@Tag + @Operation + @ApiResponse 보강" 박았는데 003 의 AuthController 실제 패턴 = `@ApiResponse` 미박힘. tasks.md 가 작성 시점 추측 박은 영역. **R3 진입 전 surfacing 으로 정합 확인 후 진행**.

4. **vault §2 통째 삭제 (Phase 8 R4)** — vault `02-PROGRESS.md` 의 §2 "진행 중" 섹션 갱신 Edit 시 §1 의 004 항목 추가 박은 영역이 §2 영역 통째로 삭제 박음. Read 후 재정독 박은 후 §2 신설 박음.

5. **Postgres JSONB normalize 회귀 (Phase 5 R2)** — entity body default String 비교 (`Document.EMPTY_DOC_JSON`) 가 DB roundtrip 후 normalize 박은 형식 (공백 추가) 과 불일치 → assertion fail. `JsonMapper` parse 후 비교로 정정. **JSONB 영역 assertion 패턴 사전 인지 누락**.

6. **N+1 케이스 Edit fail 사후 발견 (Phase 8 R2 → R6)** — ProjectRepositoryIT 에 N+1 케이스 Edit 박을 때 "File has not been read yet" fail 박혔는데 R2 진입 시점에 인지 못함. R2 BUILD SUCCESSFUL 박혔으나 ProjectRepositoryIT 의 신규 케이스 미박힌 채 기존 케이스만 통과. R6 commit 직전 git status grep 으로 발견 + 정정. **위 §4-2 와 동일 패턴 — Edit fail 직후 검증 의무 룰 갱신 강조 영역**.

## §5 교훈 (Lessons)

### §5-1. 일반 교훈

1. **Edit 실패 직후 정독 의무** — Edit 가 fail 박힌 turn 에서 다음 Bash / Read 호출 박기 전에 fail 메시지 확인 + 정정 + 재시도. 누락 시 다음 검증 단계에서 cascade fail.
2. **tasks.md 추측 영역 사전 surfacing** — tasks.md 명시 영역 (파일명 / endpoint 수 / 메서드 시그니처) 이 spec/plan 산출 시점 추측일 수 있음. implement 진입 직전 grep 1회 + surfacing 의무.
3. **JSONB 영역 String 비교 금지** — Postgres JSONB 는 normalize 박혀서 String 동등 비교 fragile. JsonMapper parse 후 비교 default 패턴.
