# 009 US3~US6 + 한국어 IME dogfooding 가이드 (사용자 실환경)

- 작성일: 2026-06-06
- 대상: `desktop` Electron 개발 앱
- 실행: `cd desktop && pnpm dev`
- 런타임: Node `v24.14.0` 필수(node:sqlite). 셸 기본이 v20이면 `export PATH="$HOME/.nvm/versions/node/v24.14.0/bin:$PATH"` 선행
- 작성 근거: `specs/009-workshop-redesign/spec.md`(FR/SC/Acceptance) + 실제 구현 코드
- 분업: MVP(US1·US2)는 `2026-06-06-009-mvp-dogfooding.md`에서 통과. 본 문서는 **US3~US6 + IME** — 시각·키보드 판단이 필요해 사용자 실환경 몫. (동작 보호는 `pnpm test` 145건이 담당. 빈 작품 빈 문구·대비 측정은 Claude가 자동화/정적 계산으로 별도 처리)

## 보는 법

- 각 표의 **결과**란에 직접 `통과 / 부분 / 실패 / 보류`를 적는다.
- 기대와 다른 게 보이면 맨 아래 **발견 보고** 표에 한 줄 남긴다.
- 화면 이동은 좌측 rail(세로 막대)의 아이콘으로 — `작품`(작업실 입구) / `쪽지`(쪽지 책상) / `기록`(placeholder, 본 범위 아님).

## 사전 준비 (fixture)

실제 판단을 하려면 데이터가 조금 필요하다. 앱을 처음 띄웠다면 아래를 먼저 만든다.

1. **작품 2개 이상** — 하나는 본문을 몇 문장 써둔 작품, 하나는 본문이 **빈 작품**(새로 만들고 글은 안 씀). 빈 작품은 US1·재진입 빈 문구 확인용.
2. **쪽지(메모) 4장 이상** — rail 하단 잉크 버튼이나 쪽지 책상 상단 `쪽지 한 줄 적기…`로 추가.
3. **한 작품에 쪽지 2장 이상 연결** — US6 고정 우선순위(고정 → 최근 연결 → 최근 캡처)를 보려면 같은 작품에 2장 이상 붙어 있어야 한다.

## US3 — 쪽지 책상 (쪽지를 작품에 다시 붙이는 책상)

진입: rail의 `쪽지`. (FR-011~015 / SC-003·SC-004 / Acceptance US3 1~4)

| # | 항목 | 조작 | 기대 | 결과 |
|---|---|---|---|---|
| 3-1 | 관리 언어 제거 | 쪽지 화면을 본다 | 전체/미연결 **개수 통계 패널 없음**, 전체/미연결 **세그먼트 필터 줄 없음**(SC-003). 상단은 `책상 위 쪽지` 제목 + (연결된 작품이 있을 때만) 작품 이름 추림 칩 | |
| 3-2 | 쪽지 본문 중심 | 흩어진 쪽지들을 본다 | 각 쪽지에서 **본문**이 가장 큰 요소. 날짜는 작게 아래에 | |
| 3-3 | 안 붙은 쪽지 붙이기 진입점 | 작품에 안 붙은 쪽지를 본다 | `＋ 작품에 붙이기` 버튼이 있다 | |
| 3-4 | 붙이기 즉시 반영 | `작품에 붙이기` → 작품 선택 | 1초 내 그 쪽지 위에 **작품 이름표**(탭)가 뜬다(SC-004, 체감 지연 없음) | |
| 3-5 | 붙은 쪽지 표시 | 이미 붙은 쪽지를 본다 | 어느 작품에 붙었는지 **이름표**로 드러난다. 이름표의 `✕`로 해제 가능 | |
| 3-6 | 여러 작품에 붙이기 | 붙은 쪽지의 `＋ 다른 작품에도 붙이기` | 두 번째 작품 이름표가 추가된다(복수 연결) | |
| 3-7 | 작품으로 부드럽게 추리기 | 상단 추림 칩에서 작품 이름 클릭 | "미연결" 같은 관리 필터가 아니라 **작품 이름**으로 그 작품 쪽지만 추려진다. `전부`로 복귀 | |
| 3-8 | 빈 추림 방지 | 추림 칩을 본다 | 실제 쪽지가 붙은 작품만 칩으로 나온다(붙은 쪽지 0인 작품 칩 없음) | |
| 3-9 | 쪽지 버리기 + 되돌리기 | 쪽지의 휴지통 아이콘 클릭 | 즉시 사라지고 하단에 `쪽지를 버렸어요` + `되돌리기` 토스트. 되돌리기로 복원 | |
| 3-10 | 빈 상태 안내 | 쪽지를 다 비우거나, 쪽지 0인 작품으로 추림 | `아직 쪽지가 없어요…` 또는 `이 작품에 붙인 쪽지가 아직 없어요.` | |

## US4 — 잉크 한 방울 빠른 메모

진입: 어느 화면에서든 rail **하단 잉크 버튼**(aria-label "빠른 메모"). (FR-016~019 / SC-005 / Acceptance US4 1~4)

| # | 항목 | 조작 | 기대 | 결과 |
|---|---|---|---|---|
| 4-1 | 캡처로 인지되는 진입점 | rail 하단 버튼을 본다 | "새 작품/새 항목"이 아니라 **메모 남기기(잉크 한 방울)**로 읽히는 형태·라벨 | |
| 4-2 | 모달 외관 | 버튼 클릭 | `잉크 한 방울` 제목 + 본문 textarea + `취소`/`저장`. textarea에 자동 포커스 | |
| 4-3 | 연결 안내 문구 | **집필 화면**에서 열기 / **그 외**(작품 벽·쪽지)에서 열기 | 집필 중이면 `현재 작품에 연결됩니다`, 아니면 `미연결로 저장됩니다` | |
| 4-4 | 초안 보존(핵심) | 본문 입력 중 **Esc** 또는 **바깥(backdrop) 클릭** | 내용이 있으면 모달이 **닫히지 않는다**(초안 유실 방지, FR-017/SC-005). 내용이 비어 있을 때만 가벼운 닫기로 닫힘 | |
| 4-5 | 명시적 닫기 | 내용이 있는 채 `취소` 버튼 | 취소 버튼은 초안이 있어도 닫힌다(명시적 의사) | |
| 4-6 | 포커스 복귀 | 집필 본문에 커서를 둔 채 캡처 열기 → 저장/닫기 | 모달이 닫히면 **직전 작업 위치(본문)**로 포커스가 돌아온다(SC-005) | |
| 4-7 | 집필 중 캡처 연결 | 집필 화면에서 캡처 저장 → 쪽지 책상 확인 | 그 쪽지가 **현재 작품에 연결**되어 이름표가 붙어 있다(FR-019) | |
| 4-8 | Tab focus trap | 모달 안에서 Tab/Shift+Tab 반복 | 포커스가 **모달 안에서만 순환**(밖으로 안 샘), 각 요소 포커스 위치가 시각적으로 보임(FR-018) | |

## US5 — 접근성 (또렷한 글자·또렷한 포커스)

대비(WCAG) 정밀 측정은 Claude가 OKLCH 토큰값으로 정적 계산해 별도 보고한다. **여기서는 사용자가 눈/키보드로 확인할 부분만.** (FR-020·021 / SC-006 / Acceptance US5 1~2)

| # | 항목 | 조작 | 기대 | 결과 |
|---|---|---|---|---|
| 5-1 | 보조 텍스트 가독성(라이트) | 라이트 테마에서 placeholder·날짜·`다음 장면` 라벨 등 보조 글자를 읽는다 | 흐릿하지 않고 또렷하게 읽힌다 | |
| 5-2 | 보조 텍스트 가독성(다크) | 집필 보기 메뉴에서 촛불(다크) 테마로 전환 후 같은 보조 글자 | 다크에서도 또렷 | |
| 5-3 | 키보드 포커스 표시 | Tab만으로 버튼·입력·모달·토스트 액션을 순회 | 지금 포커스가 어디인지 **시각적 ring**이 항상 보인다(FR-021) | |
| 5-4 | 잔여 placeholder | `여기서부터 이어 써요`, `쪽지 한 줄 적기…`, `떠오른 생각을 적어두세요…` | 모두 읽히는 대비 | |

## US6 — 곁에 둘 쪽지 고정

전제: 한 작품에 쪽지 **2장 이상** 연결돼 있어야 우선순위를 확인할 수 있다. 고정 토글은 **집필 화면 우측 `연결된 메모` 패널**(곁쪽지 서랍)에서 한다. (FR-023~026 / SC-008 / Acceptance US6 1~4)

| # | 항목 | 조작 | 기대 | 결과 |
|---|---|---|---|---|
| 6-1 | 서랍에서 고정 토글 | 집필 진입 → 우측 패널 열기 → 쪽지의 `☆` 클릭 | `★`로 바뀌며 그 쪽지가 곁쪽지로 고정(작품당 1개) | |
| 6-2 | 재진입 한 장 = 고정 쪽지 | 다른 화면 갔다가 그 작품에 **재진입** | 재진입 한 장(`여기서 멈췄어요`)의 `곁에 둘 쪽지`가 **고정한 그 쪽지** | |
| 6-3 | 작품당 1개 | 같은 작품의 다른 쪽지를 `☆`로 새로 고정 | **이전 고정이 풀리고**(★→☆) 새 쪽지가 ★ (FR-026) | |
| 6-4 | fallback(고정 없음) | 모든 고정 해제 후 재진입 | 곁쪽지가 **가장 최근 연결**된 쪽지(없으면 가장 최근 캡처)로 뜬다(FR-023) | |
| 6-5 | 작품별 독립 | 같은 쪽지를 다른 작품에도 붙이고, 한쪽만 고정 | 고정 여부가 **작품마다 독립** | |
| 6-6 | 연결 해제 시 고정 소멸 | 고정한 쪽지를 그 작품에서 연결 해제(패널 `✕`) | 그 작품에서의 **고정도 함께 사라진다**(FR-025) | |

## 한국어 IME 4케이스 (집필 본문 TipTap)

진입: 작품 집필 화면의 종이 본문. SoT = `docs/poc/0-1-tiptap-korean.md`. Computer Use 자동입력은 조합 단계를 우회해 미검증 — **실제 키보드(두벌식 한글 IME)**로 확인. (typescript code-quality.md 회귀 검증 cadence)

| # | 케이스 | 방법 | 기대 | 결과 |
|---|---|---|---|---|
| I-1 | 빠른 타자 | 한 글자 조합 중 다음 자모를 빠르게 연타(예: "안녕하세요" 빠르게) | 자모가 분리되지 않고 정상 조합. 글자 깨짐/중복 없음 | |
| I-2 | 조합 중 mark 적용 | 글자 조합 도중 BubbleMenu로 **굵게** 토글 | 조합이 깨지지 않고 mark가 적용됨 | |
| I-3 | 한자 변환 | 조합 완료 직전 한자 변환(예: "한자" → 漢字) | 변환이 정상, 본문 깨짐 없음 | |
| I-4 | Backspace 자모 삭제 | 조합 중인 글자에서 Backspace 1회 | 자모 한 개 단위로 정상 분해(글자 통째 삭제 X) | |

> 회귀 배경: Phase 4에서 `onUpdate` 매 입력 부모 re-render → 조합 중 자모 분리 회귀가 있었고 `editor.view.composing` guard로 해소. 본 4케이스로 회귀 재발 여부를 본다.
