# 009 MVP dogfooding 결과

- 일시: 2026-06-06
- 대상: `desktop` Electron 개발 앱
- 실행: `cd desktop && pnpm dev`
- 런타임: Node `v24.14.0`, pnpm `8.15.5`
- 도구: Computer Use로 Electron 창 조작, 입력 보조가 필요한 지점은 macOS 키 이벤트 병행

## 요약

작품 벽 진입, 마지막 문장 중심 카드, 다음 장면 저장/재시작 영속, 집필 화면 진입, 재진입 카드, 보기 팝오버, 곁쪽지 서랍, 다크 테마의 기본 가독성은 확인됐다.

핵심 회귀는 보이지 않았다. 카드에 날짜, 글자수, 메모 개수 지표는 없었다. 다음 장면은 앱 재시작 후에도 유지됐다.

주의할 점은 두 가지다. 첫째, Computer Use의 직접 `type_text`가 Electron 입력 필드에 안정적으로 들어가지 않아 macOS 키 이벤트를 보조로 사용했다. 둘째, 촛불 테마 전환 뒤 보기 팝오버와 곁쪽지 서랍을 함께 조작하면 접근성 상태와 시각 상태가 어긋나 보이는 순간이 있었다.

## US1 작품 벽

| 항목 | 결과 | 메모 |
|---|---:|---|
| Electron 실행 후 작품 목록으로 진입 | 통과 | 첫 화면이 `작품`, `이어 쓸 작품`인 작품 벽 |
| 카드의 가장 두드러진 글자가 마지막 문장 | 통과 | 마지막 문장이 카드 얼굴로 노출 |
| 본문이 빈 작품의 빈 문구 | 미검증 | 현재 fixture에 빈 본문 작품 없음 |
| 마지막 문장 아래 제목 | 통과 | 제목이 작은 글자로 배치됨 |
| 날짜, 글자수, 메모 개수 지표 없음 | 통과 | 카드에는 해당 지표 없음 |
| 다음 장면 입력 placeholder | 통과 | `여기서부터 이어 써요` 확인 |
| 다음 장면 입력 유지 | 통과 | 입력 후 화면에 유지 |
| 앱 재시작 후 다음 장면 영속 | 통과 | `dogfood-20260606 다음 장면next-scene-dogfood-20260606` 재시작 후 복원 |
| 카드 클릭 후 집필 화면 진입 | 통과 | `가가거걱 - 집필` 진입 |
| 새 작품 진입점 | 통과 | inline 생성 화면으로 전환, 취소로 복귀 |
| 삭제 확인 모달 | 통과 | 확인 모달과 취소 동작 확인. 실제 삭제는 수행하지 않음 |

## US2 서랍형 집필실

| 항목 | 결과 | 메모 |
|---|---:|---|
| 종이 본문이 화면의 주인공 | 통과 | 넓은 종이와 여백 중심 |
| 상단 상시 정보 | 통과 | 저장 상태와 글자수 노출 |
| 확대축소, 줄노트, 테마, 자동저장이 접힘 | 통과 | 보기 버튼으로 팝오버 노출 |
| 보기 팝오버 항목 구성 | 통과 | 크기, 줄노트, 테마, 자동저장 확인 |
| Esc로 보기 팝오버 닫힘 | 통과 | 종이 테마 상태에서 닫힘 확인 |
| 바깥 클릭으로 보기 팝오버 닫힘 | 통과 | backdrop 닫기 확인 |
| 곁쪽지 서랍 기본 닫힘 | 통과 | 진입 시 닫힘 |
| 곁쪽지 서랍 열림 | 통과 | 버튼으로 오른쪽 패널 열림, 본문을 덮지 않음 |
| 재진입 한 장 | 통과 | `여기서 멈췄어요`, 마지막 문장, 다음 장면 표시 |
| 재진입 한 장 닫기 | 통과 | X 버튼으로 사라짐 |
| 촛불 테마 가독성 | 부분 통과 | 본문은 읽힘. 팝오버/서랍 동시 상태는 아래 P2 참고 |
