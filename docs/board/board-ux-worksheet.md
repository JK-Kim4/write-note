# 작업지시서 — 보드 UI/UX (Board UX)

> **목적**: AI 코딩 에이전트가 보드의 핵심 흐름(카드 만들기·잇기·종류 부여 + 집필 중 보드 참조)을
> **초보 사용자도 직관적으로 이해**할 수 있게 구현하도록 지시한다.
> 각 작업은 단독 테스트 가능한 최소 단위로 쪼개져 있고 완료 기준(DoD)을 포함한다.

| 항목 | 내용 |
|------|------|
| 문서 상태 | v0.2 (유비쿼터스 언어 / 보드–시리즈·작품 매핑 반영) |
| 선행 문서 | `board-prd.md` (데이터 모델·API·영속화·매핑) |
| 범위 | FE 인터랙션 / UX |
| 전제 | Card↔카드 1:1 · Board↔(시리즈·작품) 1:N · 캔버스 float 좌표 · 드래그 종료 시 저장 |

---

## 0. 에이전트 작업 규칙 (먼저 읽을 것)

1. 작업은 `TASK-*` 단위. 각 작업의 "완료 기준"을 모두 만족해야 다음으로.
2. §7 권장 순서를 따른다. 특히 연결의 드래그 피드백을 우선 검증.
3. **§5 유비쿼터스 언어 + 화면 문구 규칙은 강제다.** 화면 텍스트에 `node/edge/메모` 같은 폐기·내부 용어가 새면 **결함**.
4. **두 갈래 원칙**: 모든 흐름은 *초보용 뻔한 길* + *숙련자용 빠른 길* 둘 다 제공.
5. 위치/연결/종류 변경은 모두 **낙관적 업데이트 후 실패 시 롤백**.
6. **React Flow의 `node/edge`는 어댑터 파일 안에서만.** 그 바깥 도메인 코드는 `Card/Link`로만(§PRD 8).

## 1. 컨텍스트

기존 `메모`·`인물` 메뉴가 **보드로 통합**되어 사라진다. 흩어진 메모·인물은 보드 위 **Card**가 된다("인물"은 `CardType=인물`인 Card일 뿐).

문제: 노드 캔버스 도구는 첫 사용자에게 복잡하게 **느껴진다**. 원인은 기능 수가 아니라 **결정·정밀함을 너무 일찍 요구**하기 때문. 첫 사용자는 매뉴얼을 안 읽고 화면 단서(recognition)로 움직인다.

원칙: **"책상에 카드를 툭 던져놓는 느낌"이어야지 "DB를 설정하는 느낌"이면 안 된다.**

## 2. 기술 제약 (PRD 준수)

| 관심사 | 도구 | 비고 |
|---|---|---|
| 캔버스 | React Flow (`@xyflow/react`) | `"use client"` + `dynamic(ssr:false)` |
| 드래그 중 transient | Zustand | **BE 미호출** |
| 서버 상태/영속화 | React Query | optimistic + 롤백 |
| 저장 타이밍 | `onNodeDragStop`(어댑터 이벤트) / 연결 완료 / 종류 탭 | 드래그 중 저장 금지 |

참고 상태 형태:
```ts
interface BoardUIState {
  editingCardId: string | null;     // 제목 인라인 편집
  hoveredCardId: string | null;     // hover 단서
  selectedCardId: string | null;    // 종류 칩 노출
  connecting: { fromId: string } | null;  // 드래그 연결 중
  connectMode: boolean;             // 클릭-클릭 연결
  activeBoardId: string;            // 현재 보드(집필 참조 시 전환 대상)
}
```

## 3. 작업 단위 (TASK)

### TASK-1 · 카드 만들기
**목표**: 무한 캔버스의 막막함 제거. 만들기와 이름 짓기를 한 동작으로.

**빌드 순서**
1. 툴바 `+ 새 카드` → 화면 중앙에 카드 생성 + **제목칸 자동 포커스**
2. 빈 보드 진입 시 중앙 단일 진입점(빈 캔버스 절대 노출 금지)
3. 빈 곳 더블클릭 → 그 자리에 카드 생성(빠른 길)

**인터랙션 → 상태**
| 사용자 동작 | 상태 변화 |
|---|---|
| `+ 새 카드` / 빈 곳 더블클릭 | 임시 카드 추가, `editingCardId = 새 id` |
| 제목 입력 후 Enter / 바깥 클릭 | mutation 영속화, `editingCardId = null` |
| 제목 비운 채 벗어남 | 임시 카드 폐기 |

**완료 기준**
- [ ] 버튼·더블클릭 두 경로가 동일 결과를 낸다.
- [ ] 생성 직후 바로 제목 타이핑 가능.
- [ ] 빈 보드에서 빈 캔버스가 노출되지 않는다.
- [ ] 제목 없이 벗어나면 카드가 남지 않는다.

---

### TASK-2 · 카드 잇기 ⭐ 최우선·최난도
**목표**: 발견성 낮은 연결 제스처를 단서·실시간 피드백·대체 경로로 누구나 성공하게.

**빌드 순서**
1. 카드 hover → "이을 수 있다" 시각 단서(테두리 글로우 + 가장자리 연결점). 클릭 영역은 보이는 것보다 크게.
2. 연결점 드래그 → **선이 커서를 따라오고** 이을 수 있는 카드가 **초록 강조** ← *먼저 검증*
3. 유효 카드에 놓음 → 두 카드가 연결됨
4. 빈 곳에 놓음 → **"여기에 새 카드를 만들어 이을까요?"** (연결+생성 동시 = 기능 학습 최강)
5. 대체 경로(클릭-클릭): 카드 선택 → `잇기` → 대상 카드 클릭

**인터랙션 → 상태**
| 사용자 동작 | 상태 변화 |
|---|---|
| 카드 hover | `hoveredCardId` → 연결점/글로우 |
| 연결점 드래그 시작 | `connecting={fromId}`, 임시 선 |
| 드래그 중 다른 카드 위 | 그 카드 `isValidTarget` → 초록 강조 |
| 유효 카드에 drop | Link 생성(optimistic) → 확정 / 실패 롤백 |
| 빈 곳에 drop | 새 카드 생성 + 그 카드로 Link |
| `잇기` 모드 후 대상 클릭 | Link 생성, `connectMode=false` |

> React Flow 힌트: 어댑터의 `onConnectStart/onConnect/onConnectEnd`. 빈 곳 drop은 `onConnectEnd`에서 타깃 부재로 분기.

**완료 기준**
- [ ] hover 시 "이을 수 있음"이 보인다(연결점이 항상 떠 있지 않다).
- [ ] 드래그 중 선이 따라오고 유효 대상이 강조된다.
- [ ] 빈 곳 drop이 "새 카드 만들어 잇기"를 제안한다.
- [ ] 드래그 없이 클릭-클릭으로도 연결 완성.
- [ ] 실패 시 화면이 롤백된다.

### TASK-3 · 카드가 무엇인지 정하기 (종류 부여)
**목표**: 추상적 "종류"를 강요 없이 *선택 후에만* 등장하는 가벼운 질문으로.

**빌드 순서**
1. 카드 선택 시 **"이건 뭔가요?"** + 칩 4개(인물·장소·사건·테마), 원탭 지정
2. 지정 시 카드에 **색+아이콘** 반영
3. 같은 칩 재탭 → 해제(무지정)

**인터랙션 → 상태**
| 사용자 동작 | 상태 변화 |
|---|---|
| 카드 선택 | `selectedCardId` → "이건 뭔가요?" 칩 |
| 칩 탭 | `card_type` 지정(optimistic) → 색/아이콘 즉시 |
| 같은 칩 재탭 | `card_type=null` |

**완료 기준**
- [ ] 기본 카드는 종류 없이도 완전히 작동.
- [ ] 만들 때 종류를 묻지 않는다(선택 후에만 등장).
- [ ] 지정 시 색/아이콘 즉시 반영.
- [ ] 재탭으로 무지정 복귀.

### TASK-4 · 보드 만들기 & 시리즈/작품/아이디어 매핑 ★신규
**목표**: 보드를 시리즈/작품에 매다는(또는 미소속으로 두는) 과정을 작가의 언어로.

**빌드 순서**
1. **작품/시리즈 내부 생성**: `+ 새 보드` → owner 자동 결정 → **이름만 입력** (권장 기본, 마찰 적음)
2. **전역 메뉴 생성**: `+ 새 보드` → **"이 보드는 어디에 쓸 건가요?"** → `이 작품 / 시리즈 전체 / 아이디어` 선택 → 이름 입력
3. 한 주체에 보드 여러 개(1:N). 보드 1개면 전환 UI 생략.
4. **아이디어 보드 → 나중에 붙이기**: 미소속 보드에 "작품/시리즈에 연결" 액션(`PATCH /boards/:id`로 owner 지정)

**인터랙션 → 상태**
| 사용자 동작 | 상태 변화 |
|---|---|
| 내부에서 `+ 새 보드` + 이름 | `POST /boards`(owner 자동) |
| 전역에서 `+ 새 보드` + 대상 선택 + 이름 | `POST /boards`(owner 선택, "아이디어" 선택 시 owner=null) |
| 아이디어 보드 "연결" + 대상 선택 | `PATCH /boards/:id`(owner 지정) |
| 보드 목록에서 선택 | `activeBoardId` 변경, 보드 하이드레이션 |

**완료 기준**
- [ ] 작품/시리즈 내부 생성은 owner를 자동 결정해 이름만 묻는다.
- [ ] 전역 생성은 "이 작품 / 시리즈 전체 / 아이디어"를 사용자어로 고른다("owner_type" 금지).
- [ ] 아이디어 보드(미소속)를 만들 수 있고, 나중에 작품/시리즈에 붙일 수 있다.
- [ ] 한 주체에 보드를 여러 개 만들 수 있고, 1개면 전환 UI가 생략된다.

### TASK-4B · 전역 보드 허브 (상단 `보드` 메뉴) ★신규
**목표**: 내 모든 보드를 한곳에서 찾고 관리.

**빌드 순서**
1. 내 모든 보드 목록(`GET /boards/mine`), 기본 **최근 연 순**
2. 각 보드 카드에 **소속 라벨**(시리즈명·작품명·"아이디어")
3. **키워드 검색**: 작품명/시리즈명/보드명 OR 매칭 (아이디어 보드는 보드명으로만 — null이어도 누락 금지)
4. (선택) 소속 종류 필터

**완료 기준**
- [ ] 모든 보드가 소속 라벨과 함께 최근순으로 보인다.
- [ ] 검색이 작품·시리즈·보드명을 가로질러 동작하고, 아이디어 보드도 보드명으로 검색된다.
- [ ] 전역/내부/집필참조에서 연 보드가 동일한 보드 뷰다.

### TASK-5 · 집필 중 보드 참조 ★신규
**목표**: 원고를 쓰면서 보드를 곁에서 참조.

**빌드 순서**
1. 집필 화면에서 참조 가능한 보드 = **그 작품 보드 + 상위 시리즈 보드** (`GET /works/:id/reference-boards`)
2. 여러 개면 탭/드롭다운 전환, 1개면 바로 표시
3. **마지막으로 본 보드 기억** → 다음에 기본으로 열기
4. 참조 패널/분할 뷰로 원고와 나란히

**완료 기준**
- [ ] 작품 집필 화면에서 그 작품 + 상위 시리즈 보드가 모두 후보로 보인다.
- [ ] 마지막 본 보드가 기억되어 기본으로 열린다.
- [ ] 보드는 dynamic import로 격리되어 집필 번들에 영향 없다.

## 5. 유비쿼터스 언어 + 화면 문구 (강제)

### 도메인 ↔ 화면
| 유비쿼터스 언어 (코드·문서) | 화면 문구 (UI) |
|---|---|
| Board | 보드 |
| Card | 카드 (종류 붙이면 "인물 카드") |
| Link | (동사) 잇기·이어요 |
| CardType | "이건 뭔가요?" |
| relation_type | "어떤 사이인가요?" |
| owner_type (series/work/null) | "이 보드는 어디에 쓸 건가요? — 이 작품 / 시리즈 전체 / 아이디어" |
| owner = null (미소속 보드) | "아이디어" (소속 라벨로도 사용) |
| fit view | 한눈에 보기 |
| viewport | (노출 안 함) |
| delete card | 삭제 |
| delete link | 연결 끊기 |
| node / edge | (어댑터 내부 전용, UI·도메인 노출 금지) |
| 메모 / 인물(구 메뉴) | (폐기 — 카드로 흡수) |

### 바로 쓰는 문구 상수 (드롭인)
```ts
export const COPY = {
  card: {
    newButton: '+ 새 카드',
    emptyBoard: '여기에 첫 카드를 적어보세요',
    emptyBoardButton: '+ 카드 만들기',
    titlePlaceholder: '카드 제목',
  },
  link: {
    hoverHint: '끌어서 잇기',
    dropEmptyConfirm: '여기에 새 카드를 만들어 이을까요?',
    confirmCreate: '만들기',
    cancel: '취소',
    connectAction: '잇기',
    connectModeBar: '이을 카드를 선택하세요',
  },
  cardType: {
    prompt: '이건 뭔가요?',
    types: { character: '인물', place: '장소', event: '사건', theme: '테마' },
    relationPrompt: '어떤 사이인가요?',
  },
  board: {
    newButton: '+ 새 보드',
    ownerPrompt: '이 보드는 어디에 쓸 건가요?',
    ownerWork: '이 작품',
    ownerSeries: '시리즈 전체',
    ownerNone: '아이디어',            // 미소속 보드
    namePlaceholder: '보드 이름 (예: 인물 관계, 1부 사건 흐름)',
    attachAction: '작품/시리즈에 연결',  // 아이디어 보드 나중에 붙이기
    searchPlaceholder: '작품·시리즈·보드 이름으로 검색',
    labelIdea: '아이디어',            // 소속 라벨(미소속)
  },
  view: { fitView: '한눈에 보기', resetViewport: '처음 위치로' },
  actions: { delete: '삭제', disconnect: '연결 끊기' },
} as const;
```

## 6. 머지 전 점검 체크리스트
- [ ] 화면에 `node/edge/메모/인물(구기능명)/viewport` 등 폐기·내부 용어 노출 없음.
- [ ] 도메인 코드(어댑터 제외)에 `node/edge` 식별자 없음.
- [ ] 모든 라벨이 "작가가 답할 수 있는 말"이다.
- [ ] 연결 동사가 "잇기/이어요"로 통일.
- [ ] 새 사용자가 안내 없이 만들기·잇기·종류부여를 시도할 단서가 화면에 보인다.

## 7. 권장 작업 순서
1. **TASK-2 Step 2 (드래그 피드백)** — "되네?"를 만드는 지점, 먼저 프로토타이핑.
2. **TASK-1** (카드 만들기).
3. **TASK-2 나머지** (빈 곳 drop 생성 + 클릭-클릭).
4. **TASK-4** (보드 생성 & 매핑/아이디어) → **TASK-4B** (전역 보드 허브) → **TASK-5** (집필 중 참조).
5. **TASK-3** (종류 부여, progressive disclosure).
6. **TASK-6** (안전망) → **TASK-7** (온보딩).

> 한 줄 요약: **TASK-1·TASK-3** 이 "설정 느낌 → 끄적임 느낌"의 가장 큰 레버. **TASK-2** 드래그 피드백이 도구 전체를 만만하게 만드는 분기점. **TASK-4·5** 가 시리즈/작품 구조와 집필을 잇는다.
