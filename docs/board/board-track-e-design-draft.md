## 1. 왜 "그냥 이관"이 불가인가 (충돌의 본질)

PRD §0 는 "메모·인물 메뉴 폐기 → 보드 Card 흡수"를 말한다. 그러나 세 도메인의 **차원이 서로 다르다**:

1. **소속 차원 충돌** — 메모=M:N(여러 작품), 인물=1:N(한 작품), 카드=1:1(한 보드 전속). 메모 1개가 작품 3개에 연결돼 있으면 어느 보드의 카드가 되는가? (N:N은 PRD 비목표 §3.)
2. **본문 구조 충돌** — 인물의 구조화 필드(age·gender·traits…)가 카드 `body` 평문 한 칸에 안 들어간다(잃거나 직렬화해 뭉개짐).
3. **정체성·생명주기 충돌** — 메모의 iOS 캡처 메타(source·capturedAt)·soft-delete·복구·pin 이 카드에 없다. 카드로 이관하면 **캡처 정체성과 복구 안전망이 소실**된다.

→ 038 이 "보드=메모/인물과 완전 별개"를 택한 것은 **이 충돌의 의도적 회피**였다(roadmap §2 회고 인용). 통합은 메뉴 흡수가 아니라 **도메인 데이터 모델 재설계**다.

## 2. 통합 4 옵션 (roadmap §5-E) — 비파괴 관점 분석

| 옵션 | 무엇 | 데이터 손실 | 캡처/복구 보존 | 되돌리기 | 작업 규모 |
|---|---|---|---|---|---|
| **① 완전 통합** | 메모·인물 메뉴·테이블 폐기 → 전부 Card 흡수 | **큼**(M:N·캡처·구조화 필드·soft-delete 소실) | ✗ | 매우 어려움(파괴적) | 최대 |
| **② 부분 통합** | 인물만 Card(type=character)로, 메모는 유지 | 인물 구조화 필드 손실 | 메모는 보존 | 어려움 | 큼 |
| **③ 참조 통합("가져오기")** | 메모·인물 **데이터 유지**, 보드에서 "가져오기"로 그 내용을 **새 Card 로 복제/링크** | **없음**(원본 보존) | ✓(원본 유지) | 쉬움(추가만) | 중간 |
| **④ 보류(독립 유지)** | 현 상태 — 보드는 별개 | 없음 | ✓ | — | 0 |

### 2-1. 비파괴 권고 (야간 자동 진행의 보수적 기본값)
**③ 참조 통합("가져오기")을 1차 방향으로 권고**한다(단 **사용자 최종 결정 전 미구현**). 근거:
- **비파괴**(핸드오프 §5 가드 3 정합) — 메모·인물 원본 테이블·iOS 캡처·복구·M:N 을 **그대로 두고**, 보드에 "이 작품 인물/메모를 카드로 가져오기" 액션만 **추가**한다. 데이터 손실 0, 되돌리기는 "가져온 카드 삭제"로 끝.
- **PRD 의도 점진 충족** — "보드에서 인물·메모를 펼친다"는 경험을 주되, 차원 충돌(M:N·구조화 필드·캡처 정체성)을 **원본에 남겨** 회피.
- ①②는 파괴적(데이터·캡처 경로 제거)이라 **자동 진행 금지 영역**. 사용자가 명시적으로 ①②를 택하면 그때 별도 spec 으로 범위·마이그레이션·백필·롤백을 설계.

> ③도 "복제(snapshot) vs 링크(live 참조)" 하위 결정이 있다(§3). 본 초안은 방향만 권고하고 하위 결정도 사용자에게 남긴다.

## 3. 데이터 모델 결정 지점 (사용자 결정 필요 — 자동 확정 안 함)

| # | 결정 지점 | 선택지 | 비파괴 기본값(권고) |

## 4. ③ 채택 시 비파괴 구현 스케치 (참고용 — 미구현)

- BE(추가만): `GET /api/projects/{id}/importable`(그 작품 인물 + 연결 메모를 카드化 후보로 반환) — **신규 조회만, 기존 메모/인물 무변경**. `POST /boards/{id}/cards/import`(인물/메모 id 받아 그 내용을 새 Card 로 복제 생성) — **cards 에만 INSERT, 원본 무변경**.
- FE(추가만): 보드 캔버스에 "가져오기" 버튼 → 그 작품 인물/메모 목록 → 선택 → 카드 생성. 042 내부 탭/043 참조와 자연 결합.
- **마이그레이션 0**(복제 방식). 링크 방식(D2 링크) 선택 시에만 `cards` 에 nullable origin 컬럼 1개 — 그 경우 별도 spec.
- 되돌리기: 가져온 카드 삭제 = 원상 복구(원본 메모/인물 무영향).
