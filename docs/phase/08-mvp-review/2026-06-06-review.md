### D3 근거 (원고지 모드)

실사용 기록(MVP + US3~US6 dogfooding)은 009 재디자인 검증에 집중되어 두 후보("원고지 모드" vs "richer memo curation") 중 어느 결핍이 더 컸는지 **직접 마찰 신호는 약했다.** 따라서 이 결정은 dogfooding 마찰보다 **제품 비전 우선순위**(집필 깊이 vs 메모 심화)에 근거한다.

- **원고지 모드(채택):** 집필 *깊이*. `DESIGN.md`가 구상한 A4 페이지뷰 + 원고지 격자(한국어 칸 정렬·매수)는 TipTap 자동 페이지네이션 불안정(`tiptap-pagination-plus` PoC 후 제거)으로 **보류 상태**였고, 현재는 연속 A4("iA Writer식 순도"). 로드맵상 명시 보류 항목이며 한국어 IME 검증(F4)과 연결된다. 집필 경험의 핵심 결핍을 메우는 선택.
- **richer memo curation(보류):** 메모 *심화*(검색/태그/세션노트). 009가 이미 연결·고정·쪽지 책상까지 구현해 추가 심화의 긴급성이 상대적으로 낮다.

### D4 근거 (계속 block)

WEB(001~006)은 paused·코드 보존 상태. `006-phase-3-4-editor-memo`는 자동화 GREEN이나 dogfooding 5건 미완 + develop 미merge. Desktop MVP 집중 중 재개 시 컨텍스트 분산 비용이 크다. 단일 사용자 V1에서 WEB 이슈(017~019)는 실발생 ~0. → 계속 block 유지.
